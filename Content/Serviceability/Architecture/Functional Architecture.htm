Design Objectives

  • Lower Total Cost of Ownership
  • Make it easy to build and maintain the data asset
  • Make it easy to build, deploy and share use cases (“apps”)
  • Integrate seamlessly with other Microsoft tools
  • Enhance connected health experiences, cohort creation and management, reporting and dashboards, unified patient chart visualization
  • Provide an incremental upgrade path that retains the value of existing investments and transformations

Platform Services

  • Topology Web Service - Records everything that is deployed and managed as part of the Amalga Topology
  • Registration Web Service - Used when new Features or Applications are deployed into the environment to record enough data about the feature or app to allow the system to manage it.
  • Discovery Web Service - Provides service configuration and topology registration information.
  • Central Configuration Service - Provides a standardized method to store configuration information for any requesting service.
