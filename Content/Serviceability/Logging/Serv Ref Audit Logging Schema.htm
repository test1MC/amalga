Audit table

 

Auditing data are stored in the AmalgaAudit database in two tables. Following are the column names, null status, and description of the Audit.Audit table. For the auditing details table, see Audit detail logging.

Column name Nullable Description
AuditId False A unique identifier of the audit entry.
Time False A time-zone-aware date/time stamp. The field type is “datetimeoffset”. The time stored is UTC. UTC time is absolute and does not change with daylight saving time.
UserId False The user identification associated with the audit entry.
OperationStatus False The status of the operation being audited.
  • Success
  • Failed
  • Unauthorized
  • ResourceNotFound
  • InvalidRequest
OperationType False The type of the operation being audited.
  • ViewData
  • UpdateData
  • Service
Operation False The name of the operation being audited.
DeviceName True The readable name of the client device.
DeviceIPAddress True The IP address of the client device.
IsFromTrustedSource False A binary field that specifies whether the audited operation originated from a trusted source.
IsFromPlatformService False A binary field that specifies whether the audited operation originated from a platform service.
LinkedAuditId True Points to the master audit entry that relates to this entry
