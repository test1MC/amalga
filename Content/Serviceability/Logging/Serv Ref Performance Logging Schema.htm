Performance logging table

 

Performance data are stored in the AmalgaSystem database in the PerfLog.PerfLogEntry table. Following are the column names, null status, and description.

Column name Nullable Description
Id False A unique identifier that is generated by . This ID will allow to replicate data.
EntryId False A unique identifier that is generated by the logging system. It can be used to uniquely identify a logging message.
TimeStampUTC False A time-zone-aware date/time stamp. The field type is “datetimeoffset”. The time stored is UTC. UTC time is absolute and does not change with daylight saving time.
ActivityId False A unique identifier that identifies a collection of related logging events as defined during the development process. These events may be related by function or usage.
PId True The ID of the process that generated the log entry,
TId True The ID of the thread that generated the log entry.
Level False An integer that reflects the level of performance events that are logged.
  • 0 – No performance logging, nothing is logged. This is the default performance logging level.
  • 1 – Minimal performance logging, some performance events are logged.
  • 2 - Detailed performance logging. All performance events are logged.
ServerName False

The name of the server where the code that created the logging message was executing.

If this code is being executed on a virtual machine, the name that it is logged with will be the name of the virtual machine.
LogicalHostName False The Amalga logical host that contains the server where the application is executing.
PerfMarkerId False A globally unique identifier (GUID) for a specific performance activity.
SequenceNumber False This number, in combination with the ActivityID and the PerfMarkerId, is used by developers when analyzing performance markers to determine the performance marker’s ordering.
Message True Contains the logging message.
