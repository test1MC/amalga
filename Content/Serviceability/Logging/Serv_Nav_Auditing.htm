Auditing

 

This section contains information about the types of data audited by and the audit table schemas.

To access auditing data, use standard database reporting tools.

automatically audits data in the following categories to support regulatory compliance and internal traceability.

Data access

Any read or write activity in the clinical databases or application databases is audited. The following information is captured.

  • User, time, and details of the data query are captured
  • Primary keys of the data rows accessed are captured

Application access

All application access operations are audited. The following information is captured.

  • User and time of hosted service operations
  • User and time of access to third party application service operations

Administration and management

Audits the user and time of the following items.

  • Any administration action, with the exception of the initial setup of platform services
  • Any data access by a user member of an administrative role
  • Calls to web service operations in the following areas.
    • Topology
    • Service discovery
    • Security
    • Authorization management
    • Virtual schema management
    • Query translation
    • Data access control
    • Diagnostic logging
    • Audit

Custom auditing

In addition to standard auditing, developers can add calls to the custom audit application programming interface (API) to create supplemental audit data. The custom audit operations can use a set of predefined audit operation types. For more information, see Serv Ref Audit Custom Audit OperationTypes.
