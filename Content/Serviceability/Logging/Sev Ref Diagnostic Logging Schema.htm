Diagnostic logging table

 

Diagnostic data are stored in the AmalgaSystem database in the DiagLog.DiagLogEntry table. Following are the column names, null status, and description.

Column name Nullable Description
Id False A unique identifier that is generated by . This ID allows to replicate data.
EntryId False A unique identifier that is generated by the logging system. Used to uniquely identify a logging message.
TimeStampUTC False A time-zone-aware date/time stamp. The field type is “datetimeoffset”. The time stored is UTC. UTC time is absolute and does not change with Daylight Saving Time.
ActivityId True A unique identifier that identifies a collection of related logging events as defined during the development process. These events may be related by function or usage.
PId True The ID of the process that generated the log entry.
TId True The ID of the thread that generated the log entry.
Level False An integer that reflects the system impact of the condition that was detected.
  • 16 – Error: An error indicates that this system function was unable to complete its task and had no viable recovery.
  • 17 – Warning: A warning indicates that the system was unable to complete its task; however, the problem was not critical, or could be self-recovered.
  • 18 – Informational: An information log entry is typically used to illustrate program flow; they may also be used for tracking information.
  • 19 – Verbose: Verbose log entries are designed to provide a “noisy” level of logging for system activity, typically used by application developers when monitoring the system during application development.
ServerName False

The name of the server where the code that generated the logging message was running.

If this code is being run within a virtual machine, the name of the virtual machine will be logged with the message.
LogicalHostName False The name of the Amalga Logical Host that contains the server where the application is running.
ParentInstance False The name of the parent application or platform feature that called the application component.
ComponentInstance False The name of the application, service, or platform component where the logging entry was created.
Source True The name of the process (such as myprogram.exe) where the log message originated.
Context True Any pertinent context available to the application when the log entry was generated.
Message True The logging message.
ExceptionName True If an exception occurs, the name will be listed here.
AssemblyName True The name of the assembly (program file) where the logging entry was generated.
ClassName True The name of the class within the application where the logging entry was generated.
MethodName True The name of the method of the class where the logging entry was generated.
FileName True The name of the physical application file that pertains to the logging information. Usually included only for verbose or informational logging.
LineNumber True The line number in the application file where the information was collected. Usually included only for verbose or informational logging.
StackTrace True During an application exception, a stack trace may be generated. This information can be used by personnel familiar with functionality when determining the root cause for an error.
