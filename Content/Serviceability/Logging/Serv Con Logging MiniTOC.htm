Logging

 

This section contains information about logging levels for application services and features, instructions for changing logging levels, logging table schema and sample logging queries.

To access logging data, use standard database reporting tools.

Operational data are stored in the tables described in this section. Also included are several examples of queries that access operational data. Logging data should be accessed by using standard database reporting tools.

To change the logging level of an application's services, see Change application or service logging levels. To change the logging level of an feature, see Change feature logging levels.

The diagnostic and performance logging levels available in are described in the following tables.

Diagnostic logging level Logging result
None No diagnostic logging is performed.
Error Only errors are logged.
Warning Errors and warnings are logged. This is the default.
Information Information, warning, and errors are logged.
Verbose All possible data are logged including information, warning, errors, and debug.

 

Performance logging level Logging result
Off No performance logging is perfomed.
Minimal Only major performance issues are logged.
Detailed All performance issues are logged.
