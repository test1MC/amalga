Hosting runtime events

 

This topic describes the hosting runtime events generated by .

administrators need to monitor the health of the system and take action on system failures. System health monitoring tools rely on failure events and counters generated at runtime by the platform to assess the state of the system.

call duration

On installation of application services, will automatically enable the WCF Call Duration performance counter. Every service will have a dedicated call duration counter that will monitor the average time per service request. For more information, see Call Duration counter.

Unlogged runtime events

The following events are not logged and are generally the client application's responsibility.

Application to application communication failure

The hosting runtime cannot intercept and interpret http communication failures between applications hosted on the hosting runtime. Every application is responsible for reporting any failure to communicate with components and applications it depends on.

WCF events reference

The hosting runtime allows WCF services to be hosted on the platform. Such services can be monitored using WCF default failure events directly.

When installing application services, will enable the health monitoring keyword in the WCF analytic trace. Every hosted WCF service will raise the events contained in the following table.

An administrator must listen to these events in SCOM in order to monitor the Amalga System.

Event 224 allows monitoring if the WCF host server is at threshold.
