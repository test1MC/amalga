Change feature component logging levels

 

You can change the logging levels of feature components. The procedures below can be used to change both the diagnostic and performance logging levels. For information on the logging levels available in see, Operational logging.

Changes made by the Set-AmConfigurationSetting cmdlet will be picked up automatically within a few minutes depending on the system load. There is no need to restart components for the changes to take effect.

For instructions on accessing cmdlets, see PowerShell reference.

For the current version of , setting the logging level of features is controlled at the component level. Changing the logging level of a feature's components is available through only.

To view feature components and their logging levels through the

  • Follow the procedures in View feature information. Components are listed under each feature in the module tree. Selecting a component will display its settings, including logging levels, in the results pane.

To review the current logging levels of an feature component through

To identify the components associated with a feature, use View feature information as described in the prior procedure.

  • Run the Get-AmConfigurationSetting cmdlet with appropriate parameters.

Example: Get-AmConfigurationSetting -InstanceType ComponentInstance -InstanceName IdentityService -SettingName DiagnosticsLogLevel

To change the current logging levels of an feature component

  • Run the Set-AmConfigurationSetting cmdlet with appropriate parameters.

Example: Set-AmConfigurationSetting -InstanceType ComponentInstance -InstanceName IdentityService -SettingName DiagnosticsLogLevel -SettingValue Error
