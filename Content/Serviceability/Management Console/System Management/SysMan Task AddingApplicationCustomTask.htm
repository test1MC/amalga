Add or remove an application custom task

 

An application custom task can be added to the such that it appears in the task pane when the relevant application is selected in the navigation tree of the Applications module.

For instructions on accessing cmdlets, see PowerShell reference.

For the current version of , adding a application custom task can be achieved through only.

To add an application custom task

  1. Copy the XAP file containing the application custom task definition to a physical host within the logical host where the Administration feature (which includes the ) is installed. For example, you might copy SilverlightApplication.xap to c:\temp.

  2. Run the Publish-AmManagementConsolePackage cmdlet with appropriate parameters.

    Example: Publish-AmManagementConsolePackage -Name MyCustomAppTask -Path C:\Temp\SilverLIghtApplication.XAP

  3. Sync the XAP package on each physical host in the logical host where the Administration feature is installed by running the Sync-AmManagementConsolePackage cmdlet with appropriate parameters.

    Example: Sync-AmManagementConsolePackage -Name MyCustomAppTask

To remove an application custom task

  1. Run the Unpublish-AmManagementConsolePackage cmdlet with appropriate parameters.

    Example: Unpublish-AmManagementConsolePackage -Name MyCustomAppTask

  2. Sync each physical host in the logical host where the Administration feature is installed by running the Sync-AmManagementConsolePackage cmdlet with appropriate parameters.

    Example: Sync-AmManagementConsolePackage -Name MyCustomAppTask
