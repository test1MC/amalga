Add or remove a custom module

 

A module is a standalone Silverlight application that takes a dependency on the Microsoft.Amalga.ManagementConsole.dll. With this dependency, the module has access to the 's shell (user interface, dialogs, and themes) and support. The latter is used to make calls to features and hosted services.

For instructions on accessing cmdlets, see PowerShell reference.

For the current version of , adding a custom module can be achieved through only.

To add a custom module

  1. Copy the XAP file containing the custom module definition to a physical host within the logical host where the Administration feature (which includes the ) is installed. For example, you might copy SilverlightApplication.xap to c:\temp.

  2. Run the Publish-AmManagementConsolePackage cmdlet with appropriate parameters.

    Example: Publish-AmManagementConsolePackage -Name MySampleApp -Path C:\Temp\SilverLIghtApplication.XAP

  3. Sync the XAP package on each physical host in the logical host where the Administration feature is installed by running the Sync-AmManagementConsolePackage cmdlet with appropriate parameters.

    Example: Sync-AmManagementConsolePackage -Name MySampleApp

To remove a custom module

  1. Run the Unpublish-AmManagementConsolePackage cmdlet with appropriate parameters.

    Example: Unpublish-AmManagementConsolePackage -Name MySampleApp

  2. Sync each physical host in the logical host where the Administration feature is installed by running the Sync-AmManagementConsolePackage cmdlet with appropriate parameters.

    Example: Sync-AmManagementConsolePackage -Name MySampleApp
