Unregister a data host

 

In the context of , a data host is a instance which is meant to hold Application private databases. You can unregister, or remove, data hosts from the topology by using the cmdlet below. Unregistering a data host simultaneously removes all labels associated with that data host. If you want to unregister a data host label only, see Unregister an data host label.

When the data host has been unregistered, applications can no longer use this instance.

For instructions on accessing cmdlets, see PowerShell reference.

For the current version of , unregistering a data host is possible through only.

To unregister a data host from the topology

  • Run the Unregister-AmDataHost cmdlet with appropriate parameters.

Example: Unregister-AmDataHost –DataHostInstanceName mydatahostinstance
