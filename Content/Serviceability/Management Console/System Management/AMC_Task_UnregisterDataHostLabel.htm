Unregister a data host label

 

In the context of , a data host is a instance which is meant to hold Application private databases. A data host can have multiple labels assigned so that administrators have a more user friendly way to identify the data host. For example, a data host instance might have the operational name "SQLHost91". That same host could have multiple labels such as "AllergyApplicationPrivateData", or "VistingDoctorApplicationData" so that the instance's use would be more apparent to an administrator.

You can unregister, or remove, data host labels from the topology by using the cmdlet below. The cmdlet will only remove a data host label and not the data host itself. To remove a data host from the topology, see Unregister a data host.

For instructions on accessing cmdlets, see PowerShell reference.

For the current version of , unregistering a data host label is possible through only.

To unregister a data host label

  • Run the Unregister-AmDataHostLabel cmdlet with appropriate parameters.

Example: UnRegister-AmDataHostLabel –ServerInstance SQLMachine2 –Label PrivateApplicationData
