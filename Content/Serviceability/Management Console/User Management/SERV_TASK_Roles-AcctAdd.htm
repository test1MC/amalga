Assign accounts to a role

 

This topic describes how to assign accounts to roles.

You can assign user, security group, and computer accounts to roles.

Assigning a computer account to a role of appropriate permissions allows the computer to access data and applications offline, without any user context at run time. Use this configuration to create reports or send scheduled alerts.

Groups must be of the security type; distribution groups cannot be added to roles.

To assign accounts to an role, use the following cmdlets.

To assign this type of account... Use this cmdlet:
User, security group, or computer account Add-AmRoleMembership
The current computer account Add-AmMachineServiceAccount

For instructions on accessing cmdlets, see PowerShell reference.
