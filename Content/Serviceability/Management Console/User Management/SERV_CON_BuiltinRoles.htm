Built-in roles

 

This topic describes built-in roles.

is shipped with the following static, built-in roles, which provide access to predefined platform operations, including all administrative operations, such as publishing applications and granting access to data.

Role title Description
BUILTIN\AmalgaAppAdmin

Administrators authorized to perform the following application-related tasks: 

  • Install and uninstall applications
  • Grant permissions to applications
  • Deploy and remove virtual schemas
  • Modify application settings
  • View diagnostic logs
BUILTIN\AmalgaDataAdmin

Administrators authorized to perform the following data-related tasks: 

  • Create and delete entities
  • Modify binding information for entities
  • Grant permissions to entities (control access to data) 
BUILTIN\AmalgaDataPrivilegedService

Computer accounts running services that need to retrieve resolved queries for users directly rather than using Doors authentication.

Role membership in BUILTIN\AmalgaDataPrivilegedService allows services on the computer account to retrieve users' queries by passing the userid as a parameter instead of using Doors authentication. An example of a service benefiting from membership in this role is an eventing service.

BUILTIN\AmalgaService

All computer accounts running platform services or application services.

Role membership in BUILTIN\AmalgaService is set up at installation; computer accounts for computers on which platform or application services are installed are automatically added to this role.
BUILTIN\AmalgaSysAdmin

Administrators authorized to run all management services in the platform.

Membership in this role is required to deploy and manage an topology, deploy and remove applications, manage entities, manage roles and permissions, and view logging and auditing information.

BUILTIN\AmalgaUsers All  users, including computer accounts as well as user and group accounts, that are members of roles.
BUILTIN\HVPatient patient accounts from the Security Token Service (STS).
