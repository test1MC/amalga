Roles and users

 

This section describes roles and how they connect to users.

An role is a set of access rights for a logical collection of authenticated users. For example, if physicians need a different set of access rights than nurses, then you can create a role for each logical collection: Physicians and Nurses. You can assign user, group, and computer accounts to roles.

Hierarchical roles are not supported; that is, a role cannot be a member of another role.

In the illustration below, the Physicians and Nurses roles each include several group accounts. In the illustration, the Physicians role contains members from Contoso, Eastlake, and Childrens.

To manage Amalga roles, log on to Doors using an account that belongs to the built-in role AmalgaSysAdmin.
