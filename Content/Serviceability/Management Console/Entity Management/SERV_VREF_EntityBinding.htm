Entity binding pane

 

This topic includes several images of the entity binding pane and describes its key elements.

 View and edit binding for entity properties.

Access For access steps, see Configure binding: recommended column.

This topic describes elements visible in the following variations of the entity binding pane.

Initial display

The following image shows the entity binding pane when it is first accessed for the Encounter entity. In this example, the Encounter entity has two binding sources: Contoso Hospital and Eastlake.

Ref# Name Description and tasks
1 (binding source name) User-entered name for a binding source. The position of the binding source indicates its precedence.
2 Activates the editing display for changing the property binding for this entity.
3 When clicked, displays the Transact-SQL SELECT statement, also called the binding SQL statement,  for the entity (all binding sources).
Expanded binding source in the initial display

The following image illustrates the Contoso Hospital binding source for the Encounter entity after the binding source is clicked in the initial display of the entity binding pane. As shown here, an expanded binding source lists the properties in the Encounter entity and the property-specific columns or values from this binding source. This is an informational display. To take any action, click .

Ref# Name Description and tasks
1 (yellow border) Indicates the binding source that was expanded.
2 Lists the columns and values that are bound to the entity properties for the selected binding source.
3 Indicates that the property can be bound to a specified constant value (after you click to activate the editing display). See Configure binding: constant values.
4 In the Entity Property column, indicates an entity property; in the column, indicates a column from a table or view.
5 In the Entity Property column, indicates a primary key entity property; in the column, indicates a primary key column from a table or view.
6 Entity Property Lists the properties of the current entity.
Expanded entity property in the initial display

The following image illustrates the AssignedPatientLocationPointofCare property for the Encounter entity after the property is clicked in the initial display of the entity binding pane. As shown here, an expanded property lists the details for the property and the bound database column. In this example, the bound database column is PV1_3_1 from [ClinicalData].MEDORDER].[v_Collapse_PV1_Default], which is joined to this binding source’s anchor table through the column Visit_PV1_ExternalID. This is an informational display. To take any action, click .

Editing display

The following image illustrates the entity binding pane for the Encounter entity after is clicked (resulting in the editing display of the entity binding pane). In this example, ADT is the first source system in the storage schema as listed in the tab and the entity has two binding sources: Contoso Hospital and Eastlake.

If Switch to Assisted Binding is visible, click it before changing entity binding information.

Ref# Name Description and tasks
1 Clears the binding information for the indicated binding source.
2 Adds an empty binding source for the current entity.
3 Removes all binding of table and view columns from the entity properties; removes all but the default binding source.
4 Save Saves changes to binding.
5 Cancel Cancels changes to binding.

Expanded entity property in the editing display

The following image illustrates the AssignedPatientLocationPointofCare property for the Encounter entity after the property is clicked in the editing display of the entity binding pane. As in the initial display of the entity binding pane, the expanded property lists the details for the property and the bound database column (these details are described earlier in this topic). In the editing display, additional functionality is available for the table join.

Ref# Name Description and tasks
1 Add Displays the dialog box for editing the existing relationship (table join).
2 (drop-down list) Drop-down list of columns in the anchor table (for the current binding source on the current entity) that are eligible for mapping to the primary key column of the related table.
