The following image shows the pane with the BUILTIN\AmalgaUsers role selected for the Result entity. Note the sensitive label "DiagnosisAbn" listed in the pane; the cleared check box indicates that the selected role does not have access to entity properties tagged with that sensitive label.

Ref# Name Description and tasks
1 Allows you to define a rule authorizing the selected role for access to this entity.
2 Removes the selected role from access to this entity.
3

Lists access rules. Rules grant permissions to the selected role for the data underlying the current entity. Each access rule belongs to one of the following groups.

The source entity for each rule is indicated in parentheses; inherited rules are indicated by italic type.

You can right-click an access rule to display options for temporarily disabling or enabling the rule.

4 Allows you to edit the selected rule.
5 Allows you to delete the selected rule.
6 Allows you to add a rule for the selected role.
7 Allows you to copy the selected rule.
8 Allows you to paste the previously copied rule.
9 Lists sensitive labels applicable to the selected entity. A selected check box indicates that the selected role can access data in columns bound to entity properties that are tagged with this sensitive label.
10 Lists the names of roles authorized for access to the selected entity.
