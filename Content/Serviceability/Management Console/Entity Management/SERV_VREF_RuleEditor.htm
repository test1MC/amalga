The Access Level page of the dialog box (not shown here) lists the following rule access levels for selection.

Ref# Name Description and tasks
1

(entity name), its peers, and its authorized children

When selected, the rule is applied to the current entity plus any entities that are configured to inherit access rules from the current entity.
2 Authorized children of (entity name) only

When selected, the rule is applied to entities that are configured to inherit access rules from the current entity under a parent/child relationship, with the current entity being the parent entity; the rule is not applied to the current entity.

The "Authorized children of (entity name) only" rule must be set on the nearest neighbor entity. For example, if the Lab entity is a child of Visit, which itself is a child of Patient, and you want to enable access to Lab, then the "Authorized children of the entity only" rule must be set on Visit.
