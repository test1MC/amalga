Access For access steps, see Debug access rules.

The following image shows the pane for the Patient entity, displaying the SQL SELECT statement corresponding to the underlying data accessible by the BUILTIN\AmalgaUsers role. Note that the RO VIP rule is disabled (gray font), which limits the SQL SELECT statement to the single remaining enabled rule for the BUILTIN\AmalgaUsers role.

Ref# Name Description and tasks
1 Enable Rule Includes the selected access rule in the SQL SELECT statement for the related role.
2 Disable Rule Excludes the selected access rule from the SQL SELECT statement for the related role. Use this option during debugging only.
3 Filter Clause for Selected Role Displays the SQL SELECT statement corresponding to the enabled access rules for the selected role. This statement determines the underlying data for the current entity that the role can access. You can copy the statement by selecting it and pressing CTRL+C.
4 (disabled rule) Disabled rules are indicated by gray font. Rules can be disabled or enabled in the pane or the pane.
5 (enabled rule) Enabled rules are indicated by normal font. Rules can be disabled or enabled in the pane or the pane.
6 Rules Lists all roles that have been granted permissions to the current entity, along with the access rules defined for each role.
