Deploy applications through the

 

This section introduces the topics necessary to install an hosted application through the .

Deploying an application through the requires that the application go through three stages before it can be used in production.

Publish the application

The application is published through the management console Publish an application task. The publishing task parses the application package (cab file) and copies the information to the appropriate databases. During the publishing process, verification routines also determine what additional configuration needs to occur before the application can be moved to production.

After publishing, you can check see the status of the application and the required configuration steps in the Application deployment status screen of the Applications module.

Configure the application

The status of the application is now listed as published, but in a larger context it is in a transient state of being in deployment. The application will remain in deployment until all the required configurations are completed and the application is synchronized to a physical host. Depending on the application's requirements, the required configurations could include some or all of the following configuration tasks.

  • Configure application settings
  • Resolve application dependencies
  • Assign application logical hosts
  • Configure application virtual schema
  • Bind application entities

These configuration steps can be completed in any order, but all the configuration requirements must be satisfied before proceeding to Sync. After each configuration task is completed, the validation routines will refresh the notification area of the Application status page. When the notification area turns green and says that there are no further validation errors, the application can be synchronized.

Synchronize the application

To move the configured application to production, the services must be synchronized to the physical hosts comprising the logical host that is the target environment for the application. This is accomplished through the Sync application task of the . Once this task is complete, the applications status is changed to deployed and it is available for production.
