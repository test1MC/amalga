Publish an application

 

The first step in deploying an application is to publish the application to the topology. Publishing the application takes the information about the application's services, virtual schema, and dependencies and stores it in the AmalgaSystem database. After the application is published, validation routines check to see what additional steps are required before the application can be synchronized onto the physical hosts. The required steps are displayed in the notification area of the Application status window.

To publish an application through , use the Publish-AmApplicationInstance cmdlet. For instructions on accessing PowerShell cmdlets, see PowerShell reference.


Required Role: AmalgaAppAdmin, or AmalgaSysAdmin.

To publish an application

  2. In the module navigation tree, select the Hosted Applications folder and click Publish Application in the tasks pane.

  3. In the Publish Application pane, click Browse to specifiy the location of the application's cab file. Once selected, the Publish Application pane will display the application's specifications for confirmation.

    If there is already an application with the same instance name, will append an underscore and number to create unique application title, instance, and service names. These values can be changed if desired.

  4. Click Publish Application. After the publishing and validation operations have been completed, you will return to a screen similar to the one shown in Application deployment status. The notification area will describe what steps need to be taken before the application is ready to be synchronized to a physical host.
