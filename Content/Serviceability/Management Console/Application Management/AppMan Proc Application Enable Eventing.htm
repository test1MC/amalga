Enable application eventing

 

Data Change Notification (DCN), also known as eventing, provides a way for your applications and services to be notified if data changes. Data Change Notification functionality is provided by the Microsoft.Amalga.Event.Application application, which takes advantage of the Change Data Capture (CDC) functionality in to provide a data-driven callback mechanism that hosted applications can use to support workflow-driven clinical events.

This procedure enables the Change Data Capture (CDC) functionality in on databases and tables which are bound to this application's views. For more information on eventing, see Data Change Notification.


Required Role: AmalgaDataAdmin, or AmalgaSysAdmin.

To enable application eventing

  2. In the module navigation tree, expand the Hosted Applications folder and select the application on which you want to enable eventing.
  3. In the tasks pane, click Enable Eventing.

  4. In the Enable Eventing pane, confirm that you want proceed by clicking Enable Eventing. The results box will display a message if events have been enabled for the application.

  5. After the process is complete, click Close.
