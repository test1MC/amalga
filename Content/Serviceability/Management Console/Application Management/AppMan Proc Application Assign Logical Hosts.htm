Assign application services to logical hosts

 

After publishing an application, the notification area of the results pane will generally call out that logical hosts need to be assigned before the application can be synchronized on a physical host for production. For more information on physical and logical hosts, see Physical architecture. The procedure in this topic assigns the services of an application to a logical host and its component physical hosts.

hosted application services are assigned a cluster mode in their deployment manifests. The assignment is specified by the ClusterMode attribute of either the WcfServiceManifest or DoorsWorkerManifest element. For more information, see Service manifests. The services must be deployed to logical hosts of a type compatible with their cluster mode assignment as follows.

Service ClusterMode Compatible logical host types
NLB NLB, or None.
Failover Failover cluster, or None
None None

Each application service must be assigned to one logical host. services and Doors worker services can both be assigned to logical hosts as follows.

  • A service must be deployed on at least one logical host.
  • An application's component services can all be deployed to one logical host.
  • An application's component services can all be deployed to separate logical hosts so long as the logical hosts are of the type required by the application service.
  • An application's component services can be deployed in any grouping of logical hosts so long as the logical hosts are of the type required by the application service. For example, an application with four services could have one service deployed to LogicalHost1 of type None, one service deployed to LogicalHost2 of type NLB, and 2 services deployed to LogicalHost3 of type FailOver.


Required Role: AmalgaAppAdmin, or AmalgaSysAdmin.

To assign services to logical hosts

  2. In the module navigation tree, expand the Hosted Applications folder and select the application for logical host assignment.
  3. In the tasks pane, click Assign Logical Hosts. The application's services and available logical hosts will be displayed in the Assign logical host pane.

  4. Assign each service to a logical host by selecting one of the available logical hosts listed at the bottom of the service assignment box. In the case of a Failover cluster, you must also select a cluster group.

  5. After selecting a logical host for each service, click Assign.
