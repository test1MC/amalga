Configure application settings

 

This procedure can be used in several scenarios. In the simplest situation, any configurable application settings such as logging levels, cache usage, or retry settings can be modified using this procedure. This type of change is generally made at the operational discretion of administrators.

In the context of application deployment, an application developer can include a setting which must be configured before the application can be synchronized onto a physical host and put into production. In this case, after the application has been published, the notification area will alert the administrator that an application setting requires configuration.

To configure application settings through , use the Get-AmConfigurationSetting cmdlet to review settings and the Set-AmConfigurationSetting cmdlet to change them. For instructions on accessing PowerShell cmdlets, see PowerShell reference.


Required Role: AmalgaAppAdmin, or AmalgaSysAdmin.

To configure an application setting

  2. In the module navigation tree, expand the Hosted Applications folder and select the application you want to configure.
  3. In the tasks pane, click Configure Settings.

  4. In the Configure Application Settings pane, you can configure the desired settings by editing them directly, or selecting an option from the drop down lists.

  5. After you are done configuring the settings, click Save.
