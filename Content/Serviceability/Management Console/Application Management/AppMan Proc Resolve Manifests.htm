Resolve manifests for an application

 

This topic contains information and relevant links to enable resolving the application manifests.

For examples of how to resolve manifests in the context of CTP, see Resolve application manifests in the Installation Guide.

Manifests needing resolution

Following are the manifest types that need resolution. An application will have one application manifest and one or more service manifests.

  1. Application manifest: Only one per application containing application level settings.
  2. Service manifests: There must be at least one of the following service manifest types and there may several. If there is more than one, each service manifest must be resolved.
    • Doors Worker manifest: There can be one or more, and contains DoorsWorker service settings.
    • manifest: There can be one or more, and contains WCF endpoint settings.

Resolution requirements

At a minimum, manifests must be resolve by editing the appropriate XML elements to include the following.

  • Application instance names
  • Logical host targets
  • Dependency instance names
