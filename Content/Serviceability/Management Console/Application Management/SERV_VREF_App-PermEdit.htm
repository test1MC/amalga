Edit Permissions dialog box

 

This topic displays an image of the Edit Permissions dialog box and describes its key elements.

 Adds and removes roles from access levels for the selected application.

Access For access steps, see Grant permissions to an application.

The following image shows the Edit Permissions dialog box for the application TerminologyApplication_Ins1, with the access level TerminologyUser selected and the role BUILTIN\HVPatient being selected for addition to the role.

Ref# Name Description and tasks
1 Access Levels Lists the access levels defined by the application's manifest.
2 Save Saves changes and closes the dialog box.
3 (check box for role) Select to add the role to the selected access level; clear to remove the role from the selected access level.
4 (radio button for access level) Select to add or remove roles from the access level.
