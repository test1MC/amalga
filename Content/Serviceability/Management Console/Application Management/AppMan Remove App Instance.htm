Remove an application with

 

This topic contains the sequence of cmdlets required to remove an application instance from the topology.

All the cmdlets in this procedure need to be run from a command line after the appropriate module is imported. For instructions on opening , importing the module, and links to the cmdlets in this procedure, see reference for .


Required Role: AmalgaSysAdmin.

To remove an application instance

  1. Unregister the application instance.

    This step needs to be executed from one of the physical hosts in the logical host that support the application instance to be removed.

    1. Run the Unregister-AmInstanceWithLogicalHostInstance cmdlet using parameters appropriate for your installation.

    2. Run the Sync-AmPhysicalHost cmdlet to synchronize the physical host.

  2. Synchronize the remaining physical hosts.

    This step needs to be executed on any additional physical hosts in the logical host that support the application being unregistered.

    1. Run the Sync-AmPhysicalHost cmdlet to synchronize each physical host.

  3. Remove the .

    This step need only be executed when removing an application which installed with a (.edmx file), and can be executed from any machine on which the application services feature is installed.

    1. Remove the deployed by running the Remove-AmVirtualSchema cmdlet, using the appropriate parameters. and including a true or false parameter depending on whether you want to drop the application private database.

  4. Remove application instance from the database.

    This step can be executed from any machine on which the application services feature is installed.

    1. Run the UnPublish-AmApplicationInstance cmdlet to remove the application instance from the database.

  5. Verify the application instance's removal.

    1. Open the Applications module in the and confirm that the application instance is no longer displayed. For instructions, see Open the Applications management module.
