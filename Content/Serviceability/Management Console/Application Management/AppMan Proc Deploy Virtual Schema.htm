Deploy the virtual schema

 

You can deploy an application's virtual schema to your topology by using the New-VirtualSchema cmdlet. For more information on the syntax and parameters of this command, see New-VirtualSchema.

For information on how to use this cmdlet in the context of installing applications, see Install applications in the Installation Guide.

To deploy a virtual schema

  1. Start Doors PowerShell as an administrator. Click Start, click All Programs, click Accessories, click Doors PowerShell, and then right-click Doors PowerShell and click Run as Administrator.

  2. Run the following commands to prepare .

    cd 'C:\Program Files\Microsoft Amalga\System 3.0\Platform\PowerShell'

    Import-module .\Microsoft.Amalga.Deployment.RuntimeLib.dll

  3. Run the following command to change a feature setting. Replace the placeholders with variables appropriate for your installation.

    New-AmVirtualSchema -ApplicationInstanceName "AppInstanceName" -DataHostInstanceName "DbServerInstance"

After deploying the application virtual schema you may receive a warning similar to the following warning:

WARNING: Virtual schema created but there are entities that are not bound to the store. Application queries will not work till these are bound.

For information on how to bind the entities, see Entity binding overview.
