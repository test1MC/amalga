Configure application virtual schema

 

After publishing an application, the notification area of the results pane will generally call out that there are virtual schema configuration errors that need resolution before the application can be synchronized on a physical host for production. The procedure in this topic configures the virtual schema and creates a private database, if required, for the application.

If the notification that there are virtual schema configuration errors persists after performing this procedure, proceed to the Bind application entities procedure.

To configure an application virtual schema through , use the New-AmVirtualSchema cmdlet. For instructions on accessing PowerShell cmdlets, see PowerShell reference.


Required Role: AmalgaDataAdmin, or AmalgaSysAdmin.

To assign logical hosts

  2. In the module navigation tree, expand the Hosted Applications folder and select the application for virtual schema configuration.
  3. In the tasks pane, click Configure Virtual Schema.
  4. In the Configure Virtual Schema pane, click the down arrow at the right of the data host assignment box and select a data host from the list presented.
  5. Click Create Virtual Schema. The views making up the application's virtual schema are transferred to the data host, and a database script is generated to create the application's private database.
  6. Create the application's private database.
    1. In the Configure Virtual Schema pane, click Copy Script to Clipboard.
    2. Open and connect to the data host you selected in the previous step.
    3. Paste the script into a query pane.
    4. Execute the script to create the private database.
  7. In the Configure Virtual Schema pane, click Close.
