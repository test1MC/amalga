Sync application

 

An application's services must be synchronized after any of the following activities.

  • Initial publishing followed by the resolution of required configuration activities during deployment. For more information, see Deploy applications.
  • Updating an application. For more information, see Update applications.
  • Rolling back an application. For more information, see Rollback an application.

The procedure in this topic synchronizes the application services to physical hosts within the logical hosts.

If you are synchronizing an update or rollback to an existing application which is hosted on a Network Load Balancing or Failover Cluster logical host, it is best practice to remove one machine at a time from the cluster to synchronize it individually. This will allow for continuous availability of the application being synchronized as the services in the remaining machines are still working.

To synchronize an application through , use the Sync-AmPhysicalHost cmdlet. For instructions on accessing PowerShell cmdlets, see PowerShell reference.


Required Role: AmalgaAppAdmin, or AmalgaSysAdmin.
