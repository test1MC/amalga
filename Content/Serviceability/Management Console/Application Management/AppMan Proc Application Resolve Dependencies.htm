Resolve application dependencies

 

After publishing an application, the notification area of the results pane will generally call out that application dependencies need to be resolved before the application can be synchronized on a physical host for production. For example, if an application developer specifies that the application requires the DataConnector application to be present in the system, the administrator has to resolve the dependency by selecting an instance of dataconnector from the list of dataconnector instances available.The procedure in this topic resolves such dependencies.


Required Role: AmalgaAppAdmin, or AmalgaSysAdmin.

To resolve application dependencies

  2. In the module navigation tree, expand the Hosted Applications folder and select the application for dependency resolution.
  3. In the tasks pane, click Resolve Dependencies. The application's dependencies will be listed in the Resolve Application Dependency pane.
  4. Resolve each dependency.
    1. To the right of each dependency listing, click Edit Selection.
    2. Click the option button next to the application instance you want to use to satisfy the dependency.
  5. After selecting an application instance to resolve all the dependencies, click Save.
