Update applications

 

An application update installs a newer version of an already installed application. The update package is the same as any other application and can be installed as a new instance of the application. The benefit of using the update function of the is that all the application parameters required for a new installation, such as dependency resolution, virtual schema configuration and entity binding are already defined. The update task changes only the application binaries and leaves all operational application parameters intact. Only the administrative versioning parameters are updated to reflect the new values.

An application cab file must meet the following requirements to be used as the source for an application update.

  • Only non-manifest changes
  • No dependency changes or new configuration settings
  • No virtual schema changes
  • No new services, or removal of previous services

To update an application through , use the Update-AmApplicationInstance cmdlet. For instructions on accessing PowerShell cmdlets, see PowerShell reference.
