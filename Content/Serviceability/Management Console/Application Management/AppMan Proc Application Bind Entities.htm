Bind entities used by an application

 

After publishing an application, the notification area of the results pane will generally call out that there are virtual schema configuration errors that need resolution before the application can be synchronized on a physical host for production. The procedures referred to in this topic bind the entities if required. When you bind an entity to storage schema, that binding applies across the installation.

If the notification that there are virtual schema configuration errors persists after performing this procedure, proceed to the Configure application virtual schema procedure.

To bind entities through , use the Set-AmEntityBinding cmdlet. For instructions on accessing cmdlets, see PowerShell reference.
