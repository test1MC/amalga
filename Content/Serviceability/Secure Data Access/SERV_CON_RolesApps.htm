Roles and applications

This topic describes how roles interact with applications.

In , application permissions are managed by granting roles permissions to application access levels.

Each application contains one or more access levels in its application manifest for granular control of access to application-specific resources, including services and application views. Access level definitions are immutable; however, you can manage the role assignments to each access level.

For example, in the illustration below, the LabApp application provides two access levels. The LabReader access level allows the user to call a service that retrieves outstanding lab orders and to view lab orders. The LabWriter access level allows the user to call all services provided by the application and to create, edit, and view lab orders. Andy and Sam have unique access rights in the LabApp application because their roles are assigned to unique access levels. Through user membership in the LabAdmin role, Andy is able to access the LabApp application at the LabReader access level; Sam’s membership in the Radiologist role allows Sam to access the application at the LabWriter access level.

To manage application permissions, log on to Doors using an account that belongs to the built-in role AmalgaAppAdmin or AmalgaSysAdmin.
