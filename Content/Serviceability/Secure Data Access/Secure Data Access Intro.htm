Data access security

This topic provides conceptual information about how permissions for data and applications are managed in terms of roles.

permissions are managed in terms of roles only. Authorization for data and applications is based on user membership within roles specific to the topology. A role is a set of access rights for a logical collection of authenticated users.

For example, in the illustration below, Holly is a member of the role “ICU Nurses”. Based on her membership in this role, Holly is authorized to access the application “PatientNotes”. This role membership also allows her to view data under the entities “Patient” and “Visit”; for “Visit,” she is granted read-write privileges.

This access control framework helps show the right data to the right user at the right time. Access is granted specifically for data (entities) and applications. No access is given by default.

Roles are authorized for data (entities) via access policies. For more information, see Data permissions management.

Roles are authorized for applications via access levels. For more information, see Application permissions management.
