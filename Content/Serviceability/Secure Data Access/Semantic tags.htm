Semantic tags

This topic introduces the concept of semantic tags.

A semantic tag is a standard terminology concept that provides definitive meaning to an entity or one of its properties. When multiple semantic tags are applied to an entity or entity property, the definitive meaning of the entity or property is the logical disjunction of all tags; that is, the tags are treated as a logical OR.

Semantic tags facilitate organization and search, as well as improving performance. The module of uses semantic tags to deterministically establish the relationship between an entity and one to many database tables and views. When a table or view has one or more semantic tags matching those of an entity, the table or view is recommended for access inheritance. When a column from a table or view has one or more semantic tags matching those of a property within an entity, the column is recommended for binding to that property.

Tables, views, and columns typically receive semantic tags from parsers during data ingestion.

Entities and entity properties can have semantic tags applied or removed through the module. For example, if you want additional columns to be recommended for binding to an entity property, you can add the corresponding semantic tags to the property. For more information about applying semantic tags to entities and entity properties, see Edit entities.
