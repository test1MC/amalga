View or update quick criteria

 

This topic describes how to use the QuickCriterionUpdateScript to view or update a quick criterion or criteria.

After a quick criterion is created, you can view its properties including the categories associated with it and update any of the properties using the QuickCriterionUpdateScript.sql script located in the Samples\DataExplorer folder of the installation media.

To view a criterion

  1. Provide a value for @CriterionID or @CriterionName or both. Leave all other variables as empty strings.

    You can use standard SQL matching wildcards to select multiple criteria.

  2. Run the script to retrieve and display data associated with the criterion or criteria.

To modify a criterion

  1. Provide a value for @CriterionID or @CriterionName or both.

    You can use standard SQL pattern matching wildcards to select multiple criteria.

    If both a name and ID pattern are supplied, criteria must match both for the update to occur.

  1. In @UpdateCriterion, enter any single character - for example, 1.
  2. Enter values for any variables that you want to change, including the query itself.

  3. Run the script.

    Basic information about the criteria and properties modified will be displayed.

To retire a criterion

  • Follow the steps for modifying a criterion and set the value for @CriterionStatus to 2.

    Setting the @CriterionStatus to 2 will cause any lists using this criterion to stop functioning. As an alternative, you can remove the criterion from all categories and from the Commonly Used section. It will not appear in the criteria list, but the lists using it will continue to function.

    Retiring a criterion is preferable to deleting it, since it can be restored if needed again.

To restore a criterion

  • Follow the steps for modifying a criterion and set the value for @CriterionStatus to 1.
