Create a value set

 

This topic describes how to use the ValueSetCreationScript to create a value set. You select options from these custom criteria to create a patient list.

To create a value set

  1. In @MyID, enter your user ID.
  2. Do one of the following:

  • If this is a static list, type the values in the @Options variable in the following format: DisplayName1, Value1; DisplayName2, Value2 or Value1; Value2; etc.

    or

  • If the values will be created by an SQL query, copy the query into the @Query variable. The query must return two columns. The values in the first column will be used as the display text and the second as the values for evaluation.

    You can use the QueryTestScript.sql script to test the query.

    does not support the distinct keyword in queries, so you must use "group by" to find distinct values. For example, select DischargeDisposition, DischargeDisposition from Encounter group by DischargeDisposition.

  1. In @TitlesEqualValues, enter a 1 if the display title and actual values are identical. Otherwise, leave it as 0.
  2. In @SetName, enter a name. The name is for internal use only.
  3. In @Organization, enter your organization. The organization is for internal use only.

  4. In @Bindings, enter the custom criteria to which you want to bind the value set in the format entity.attribute. Separate multiple criteria with semicolons.

    Bindings are locale-specific.

  5. Run the script.

    Changes will take effect once the cache is updated on the server.
