Create new criteria

 

This topic describes the general process for creating criteria that will be used for creating patient lists.

Quick criteria are predefined queries that will be used to create a list of patients. These criteria take the form of an SQL query against the application schema that returns a set of primary keys for the Patient entity. quick criteria may be defined by a database administrator or programmer using .

Custom criteria are used to build queries directly against the data model. They can be based on the attributes of any entity possessing a foreign key to the patient entity. All data types supported by the entity model are supported for use as custom criteria except for the Time data type.

Prior to creating criteria or value sets, establish the business requirements for the new criteria and determine the SQL queries that will be needed to obtain the values.

Once the requirements are established, you can use the following tasks to create and update criteria:

  • Create and test a query. See Create new quick criteria query on page 1.
  • Store the query in the database for use by others. See Deploy quick criteria on page 1.
  • Review and modify an existing query. See View or update quick criteria on page 1.
  • Create custom criteria. See Create custom criteria on page 1
  • Create a value set. See Create a value set on page 1.
  • Review and modify a value set. See View or update a value set on page 1.
