Deploy quick criteria

 

This topic describes how to use the QuickCriterionCreationScript to store criteria in the database for use in creating patient lists.

Once a criterion is created, it must be deployed to the database so that is can be used to create a patient list. Use the QuickCriterionCreationScript.sql script located in the Samples\DataExplorer folder of the installation media to deploy the criterion.

To deploy the script

  1. Copy the query definition from the test script over the @Query declaration in this script.
  2. In @Categories, enter the category or categories, separated by commas, under which the criterion will be displayed.

  3. In @FrequentlyUsed, enter a 1 if this criterion will be included in the Most Common section of the criteria.

    You must specifyic at least one category, or indicate that it will be in the Most Common section, before it will be displayed in the Patient list criteria.

  4. Provide a value for @CriterionName. The name is for internal use only.
  5. Provide a value for @CriterionDisplayTitle. This will be the name displayed in the criteria list.
  6. Enter the name of your organization in @Organization.
  7. Specify a different value for @Locale if desired. The criterion will not be available to users in other locales.

  8. Run the script.

    You will receive a message that the criterion was created. Changes will take effect once the cache is updated on the server.It will be available for use once the cache is refreshed on the server.
