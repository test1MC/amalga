Write and test a new quick criteria query

 

This topic describes how a system administrator can create new quick criteria for use in creating patient lists.

Prior to creating a query, review the business requirements for the new criteria. Review the entity model in the Entity Management section of the .

The syntax for the script is limited to what is supported by the . For more information about what is supported, see Insert reference to Dennis' information.

Create your query and test it using the QuickCriterionTestScript.sql script located in the Samples\DataExplorer folder of the installation media.

To write and test a query

  1. Write your query in an SQL editor.

  2. Table names and aliases in FROM statements must be enclosed by a | – for example, select Patient.PersonID from |Patient|.

    If the query results depend on the user, embed |CurrentUser| in the query and enter a user name into the @CurrentUser variable in the form of domain\username.

  3. In the script, enter the text of your query into the @Query variable.

  1. Run the test script and review the results for accuracy.

    If you receive an "Incorrect syntax near '|' error message when running the script, it may indicate that the entity you are trying to query is not valid. Verify that the entity exists and that you have spelled the name correctly.

    This test script will not reflect security restrictions for the assigned user.
