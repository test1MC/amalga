Linking Strategy

The Amalga UA team uses two types of links:

Within a Project: Cross-References

Use a cross-reference to link to a Flare topic. The cross-reference picks up the first heading of the file as the visible text for the file title. If the first heading of the topic is changed, the cross-reference is automatically updated.

Exception: For a link to an API file generated as an automated snippet, you must use a hyperlink. The first heading for all such files is Syntax, so you must use a hyperlink so you can supply appropriate text for the file title.

The default cross-reference style is MadCap:xref. For other styles, see the Amalga UA Style Guide.

To External Resources: Hyperlinks

To link to a website, use hyperlinks. If it is a Microsoft website, request a forward link from the Amalga UA editors. In your email message, include the URL and the name of the file that will use the forward link.

You can use a hyperlink to link directly to a PDF file in a Flare project if you are using Webhelp output, but not PDF output. The hyperlink would have to be conditioned for ScreenOnly.
