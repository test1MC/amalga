Style and Usage Resources

Amalga UA Documentation Standards is not intended to cover every standard you may need to write Amalga UA documentation. In general, the following style guides are complementary. Where they are not, the descriptions identify what each style guide should be used for.

With the exception of the Amalga UA Style Guide, the listed guides and more are available at http://msstyle/.

Amalga UA Documentation Standards

Information in this document describes standards and conventions specific to Amalga UA writing and subject matter; for our writing, this document overrides other standards.

Developer Division User Education (DDUE) Style Guide

The DDUE Style Guide is designed for developers, so we use for the more technical aspects of our writing. Adherence to DDUE standards will position us to more easily put any of our documentation into MSDN.

You can access the DDUE Style Guide by downloading dv_dduestyle.chm at \\docbuild2\Public\Trunk\Main\drop\1033 in Doors Explorer or by using the Contents pane at http://msstyle. (See Other Resources on page 1.)

Microsoft Manual of Style for Technical Publications (MSTP)

MSTP contains standards for computer- and software-related terms suitable for a variety of audience, from end user to developer.

When the same information is covered in both MSTP and DDUE, and there is a conflict, adhere to DDUE.

You can access MSTP at http://mstp/.

Other Resources

There are additional guides to usage, often written with specific products or product types in mind, such as http://winedit, the Doors Server Style Guide.

You can access this and other standards (including DDUE and MSTP) at http://msstyle; the Contents pane lists the individual guides, but you can also search for specific text.
