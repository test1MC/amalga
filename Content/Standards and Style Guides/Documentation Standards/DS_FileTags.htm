File Tag Sets

For specific and current information on available file tag sets and file tag values, see Project Organizer/Advanced in any Flare project and look for this icon, .

File tags are metadata that can be added to a topic or snippet. Amalga UA is using the metadata to help produce reportable statistics (for shiprooms, for instance).

Only one value in a file tag set can be assigned to a topic or snippet.

Reporting

You can export file tag data from Flare to a CSV file using the FlareFileTagReporter.exe located in C:\UIS\UA-SRC\Sandbox\Flare Report Tools\FlareFileTagReporter. Instructions are in readme.txt. Once the report is generated, you can open it using Excel to filter and sort the data.

File Tag Set Descriptions

Editing 

If an editing tag is set, the topic or snippet is either ready for editing, being edited, or being revised in response to edits.

Personas

Roles describing those who participate in workflows related to Shanghai/Beijing. When writing topics, especially procedure topics, keep the appropriate persona for that activity in mind.

If the topic applies to multiple personas, make the hard choice and pick a primary persona, or pick All. Persona may be Microsoft, customer, or ISV personnel.

Our personas are invisible to customers.

For more information on personas, see HSG Persona Models.

Status

An indication of the stage of activity a topic or snippet is in: for example, initial writing, tech review, editing, revision, complete.

TopicOwner

Principal writer for the topic. Do not change without notifying a UA manager.

TopicType

Standard templates to be used when writing a topic. (In process.)
