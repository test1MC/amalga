Folder Organization for TFS and Local Files

To enable maximum flexibility for publication and localization, all content created the UA team will be organized under Content as Features, API documentation, or global content in Team Foundation Server, the source repository:

Features, Applications, Etc.

\Content\

  • Feature Area
    • Topics
    • Images
      • Icons
      • Screen Captures
      • Conceptual Art
    • Snippets
      • Frequently repeated procedures or steps that are unique to this feature
      • Frequently repeated notes, warnings, tips, cautions.
      • Frequently used text
    • Other resources (if any)
      • PDFs
      • Sample code

Cross-Project Material

In addition to folders for feature areas, TFS will contain a parallel folder structure for Cross-Project items such as legal content or content that is common to two or more feature areas. The folders are organized as follows:

\Content\Cross-Project

  • Topics
  • Images
  • Snippets
  • Other Resources

APIs

API documentation folder structure follows the output of the Sandcastle process for generating topics:

\Content\API Name

  • \Namespace
  • \Type
  • \Members
