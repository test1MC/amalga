Request a virtual machine with SharePoint Server 2010 installed

This topic describes how to request a virtual machine with already installed.

To request a virtual machine installation

  1. Go to http://ddweb/techease/TicketCreate.aspx?HSG.

  2. Open a ticket, entering this information:

    1. Asset#: Leave blank.
    2. Affected Machines: Leave blank.
    3. Due Date: input the next business day or whatever date you require.
    4. Category: select Deployment Request.
    5. Urgency: select the appropriate level.
    6. Title: type a name such as Create VM.
    7. Details: Ask for the virtual machine to be built using the image from \\rdlhsgscvmm101.redmond.corp.microsoft.com\hsgvhdlib\GOLD\Amalga\Server-enUS\V3_Beijing_2K8R2SP1_SQL2K8R2SP1_VS2010SP1.

    8. Email Cc: type in hsgteser;your own alias.

      Optional: add Preston Mathis (prestonm); Vince Viggiano (v-vinvig) to Email Cc: to make sure the virtual machine gets installed in the Redmond lab.

    9.  
    10. Which Location is the request for? Select Lab – Redmond, WA ((RDL).
    11. If new machine is it virtual or physical . . . Select Virtual.
    12. If new machine, how much memory: Select 4GB.
    13. If new machine, request which type? Select Other – Please specify in description.
    14. If new machine how much hard drive space? Select 100GB.
    15. Core XT Branch? Leave blank.

  3. Fill in and attach the required document at \\hsfile01\TechEase_Machine_Request\HSG Labs VM Requirements Form Questions.docx

    (See email attachment for an example.) Use the following naming convention for the VM: vm-amuaopsX-v3, where X is the next number in sequence, not already used on https://hsgvmportal/Portal.aspx . Use Health for domain.

  4. In TFS . . . open this document and save it locally (not on TFS): Dir/Create VM. doc . . .
