Remove bad CSS links from template

[Delete this line]

 

This topic describes how to remove bad CSS links from your Amalga templates.

You may encounter a path issue when you create new topics with the Amalga templates. That is, your topics may not reflect the correct formatting in the doc builds.

The issue is that the style sheet links (for AmalgaMain.css) don’t “self-adjust” when you create a topic in a different folder than where the template resides. Instead of being four folders away from the template, for example, the topic might be two folders away. That’s what makes the link go "bad" when you create topics from the templates.

An extra step is required to ensure that the style sheet link stays good. When you create a topic from one of the Amalga templates, remember to re-select the style sheet.

To detect bad CSS paths in topics

  1. Make sure that your primary target does not specify a master style sheet.
  2. In Project Organizer, expand Targets, right-click the desired target, and then select Make Primary.

 

The screenshot below shows Hg-Master-Web as the primary target. Now the topic formatting in the default editor (XML Editor) looks “off” if the CSS link is bad. Note: The primary target is a local setting for each Flare project; it’s not stored in Source Control.

 

To fix bad CSS paths in topics

1. Make sure your primary target is one without a master stylesheet (such as Hg-Master-Web for the AmalgaV3-wking project; see above).

2. For one or more topics:

- Right-click, select Properties, and then select AmalgaMain as the CSS.

3. For the current topic:

a. Right-click the HTML block on the left of the topic and select Stylesheet Links.

 

b. In the dialog box, remove the current link on the left and add the AmalgaMain.css link listed on the right.

 

4. Verify formatting in Flare and in the final build.
