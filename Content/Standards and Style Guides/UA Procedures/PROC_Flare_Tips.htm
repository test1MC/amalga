Preview    
Situation/Objective Action  
To change the output format in Preview Change your primary target (could be a PDF target or WebHelp target or whatever): Right-click the desired target and select Make Primary.  
Note: Do not confuse targets with TOCs. If you attempt to change the primary TOC (on most/all of the team’s projects), then Flare will prompt you to check out the .flprj file, which is not supposed to be edited by anyone other than HG.    
To hide conditioned text in a Preview display 1. In a topic, click the Preview compiled topic icon.
2. In the block containing Include Expression and Exclude Expression, click the down arrow.  
     
3. Select check boxes as needed to exclude (hide) or explicitly display (include) conditions.  
4. Click OK to confirm your selections.  
  In preview, the default is to implicitly include all conditions.  
Conditions are listed as SetName.Condition.    
If you want to see how a condition is used for a specific output type, check the Hg* targets.    
Examples:    
Exclude Default.PrintOnly condition to see how the previewed topic will be displayed in WebHelp.    
Exclude Default.TOCsAll and Manuals.XRefOtherManual to produce a PDF or Word file that displays mref snippets correctly.    
Exclude Default.InternalOnly to hide Internal text and topics.    
Snippets    
Situation/Objective Action  
To find out where snippets are used 1. Go to Content/Resources/Snippets.
2. Right-click on the snippet and select View Links.  
To create a snippet from scratch when a table is in the snippeted text 1. Select Project > Add Snippet and then name the snippet as desired. For example, name the snippet Table.flsnp or Ref-UGModuleList.
2. In the topic with the table, cut (CTRL+X) the table (and any other content).  
3. In the new snippet, paste (CTRL+V) the table (and any other content). Save the snippet.  
4. In the topic in which the table was cut, select Insert > Snippet to insert the saved snippet.  
5. Insert snippet in any other topic as needed.  
To force a snippet to be inline as part of a paragraph or to insert as a separate paragraph 1. Insert the snippet.
2. If it not in the format you want:  
• For an inline snippet, change the XML to <MadCap:snippetText src="../mref-ServerInstallation.flsnp" />  
• For a snippet as a separate paragraph, change the XML to <MadCap:snippetBlock src="../mref-ServerInstallation.flsnp" />  
To add a snippet with a multi-item list as the first items in a non-snippet list. (kluge version, but it works) The normal way of adding the snippet at the top of a list does not come out correctly. Once you have added your snippet, switch to the XML view (called Text Editor) and change the XML so that it looks as follows. You will probably have to add the highlighted tags. The snippet which contains the list is in the snippet Block; the standard list starts at the >\<li> tag:  
<p class="LabelforProcedures">To grant permissions to the users for the resource </p>    
<ol>    
<MadCap:snippetBlock src="../../Resources/Snippets/proc-Open-Amalga-Manager.flsnp" />    
</ol>    
<ol MadCap:continue="true">    
<li>    
<p>In the Use

  1. Right-click on a topic: click on View Links.ndencies: note the file names.

To add <p>-styled paragraph (such as a note) to a numbered or bulleted list with the correct indentation    
  1. Make sure you have the structure blocks turned on. (See “To show or hide span and block bars” in the Unclassified section of this document.) They should be shown at the left of your editing panel.
2. Right-click the <li> block after which you want to insert the non-list paragraph and then select Make Paragraph Item(s).  
     
3. Place your cursor at the end of the content in the <li> block and then press ENTER to add a new line. This line will be unnumbered with a yellow block containing a black arrow. Ignore this. Enter text and apply style.  
4. Place your cursor at the end of the added text and press ENTER. Click the yellow block containing a black arrow on this line to get the next numbered line.

To copy a block of code into Flare using the <pre> style 1. If needed, edit the block of code with tabs or spaces to ensure appropriate spacing.  
2. Copy the block of code to the Doors clipboard.    
3. In a Flare topic (default XML Editor view), create an empty paragraph styled with the <pre> tag.    
The <pre> tag is the last style in the Styles list.    
4. With your cursor in the empty <pre>-styled paragraph, select Edit > Paste (or press CTRL+V) to paste the copied block of code.    
5. In the Paste Text dialog box displayed by Flare, select Paste as inline text and then select Paste. In V7, you must use the little “paste as” option box to select “inline”. The box comes up at the bottom of the text after pasting.    
6. If the topic is intended for use by a PDF or Word target, build those targets and check for long lines that exceed the right-hand margin. If necessary, edit the spacing of the code block using<br> tags or spaces as needed, in Internal Text Editor. Condition any <br> tags as PrintOnly.    
To edit spacing of code blocks in Flare    
Open the topic in Internal Text Editor and review saved changes in XML Editor (default view of topic).    
Tables    
Situation/Objective Action  
To add a paragraph style to a table cell, so that the correct font shows 1. Press ENTER in the cell.
2. If there is text in the cell already, delete the extra paragraph mark.  
  If you have the toggle show blocks on, and look at the blocks on the left, this action changes the cell from having a td block to having td and p blocks.  
To make sure tables do not extend past the edge of the page • Always have at least one column width as auto.
• If you specify a width for one or more columns, the total (including space for the auto column) should not exceed 675 px or 7 inches.

Troubleshooting    
Problem/Symptom Resolution  
Link in PDF does not work; when you hover over the link, you see a hand with a boxed W. The form of the pointer indicates the linked file is not in the PDF build. In at least one case, this problem was resolved by:  
• Making sure the files were tagged to be included in the build  
• Adding the files to the TOC for the build  
Troubleshooting steps were as follows:    
1. Opened the PDF to the topic with the broken links.  
7. Hovered over the broken links and the good links. (Good links did not display any path; broken links displayed a “W,” along with what looked like the path to the .htm source file on my local system. This led me to suspect that those topics were not even in the PDF because the links could not resolve to them.)  
8. Searched for the broken link target topics in the PDF to determine whether they were built. (Result: Topics missing)  
9. Opened the DR TOC in Flare and inspected the Events branch. (Noted that the IADE topics were missing.)  
10. Added the missing topics and regenerated the PDF.  
11. Searched for the missing topics and verified that they were built. (Result: Topics there)  
12. Added the deleted links back to the navigation topic (Amalga Events), regenerated the PDF, and verified that links worked.  
Broken links: Renaming or moving snippets breaks links in topics From Neal at MadCap:  
One possibility is that the Analyzer folder, which maintains a database of the links in the project, is out of date or corrupted. The best bet is to close the project, delete the Analyzer folder, open the project back up and let analyzer rebuild the database for the project.    
If the project is local to your PC, the Analyzer folder will be located in the same folder as the project.    
If the project is on a network, the Analyzer folder for that project will be located in C:\Documents and Settings\[yourname]\Application Data\MadCap Software\Analyzer\name of project. (This is for XP. The path in Vista is a bit different.)    
You'll want to back up that folder just in case, so just renaming it is the easiest way(Projectname.OLD). Open the project and let analyzer rebuild the database. Attempt to move a snippet again. Is the result better?

1. Hyperlinks to Flare bookmarks  
2. Headings within snippets  
Unclassified    
Situation/Objective Action  
To change the appearance of a list such as the Pending Check-ins listing (change column order or width, add color, delete columns) See the help topic “Customizing Column Information.”  
To move a table situated just below a numbered list into the last item of the list 1. Open the file with Internal Text Editor.
2. Find the <table> tag.  
3. Cut the <ol> above <table>.  
4. Paste the <ol> after the next </table>.  
  The table will now be correctly indented when displayed in WebHelp output, but not in a PDF output.  
To insert a space between two variables (or two other non text elements) Press SHIFT+SPACE to create a hard space.  
To paste a block of text as a separate paragraph 1. Highlight and copy the text you want to paste.
2. Right-click the paragraph block that precedes the point at which you want to insert the text.  
3. Select Edit.  
4. Select Paste Below.  
  Paste Above is also an option. If you use this technique you don’t have to add a line, paste, and then delete the extra line.  
To change the name of a folder 1. Open Visual Studio.
2. Expand the tree until you see the name of the folder you want to change.  
3. Right-click the folder name.  
Select Rename, type in the name you want, press ENTER.    
4. Check in the change.  
5. Re-link topics in affected TOCs.  
  If you change the folder name in Flare, you lose the history for all topics in that folder, and you still have to fix the TOCs.  
To show or hide span and block bars To show or hide the span bars, click at the bottom of the editing window. Span bars deal with formatting. They show at the top of the window when your cursor is on a line that has an xref, a span style, a variable, etc.  
To show or hide the block bars, click at the bottom of the editing window. Block bars show at the left side of the editing window, and deal with the HTML structure and tags.    
This and other info about the editing window in Flare is in the Flare help file “Frame Contents Window Pane”.    
To get rid of the formatting represented by a span bar Click the bar and select unbind.  
To insert a mini-TOC 1. In an appropriate topic, click Insert.
2. Click Proxy.  
3. Click Insert Mini-TOC Proxy; do not select a TOC depth.  
4. Condition the mini-TOC as ScreenOnly.  
For more info on mini-TOCs, see the Appendix.    
Variables    
Situation/Objective Action  
When inserting a variable (Insert>Variable) Make sure you have only the word or phrase selected.  
  Do not select the surrounding spaces or they will be added to the variable, and the resulting text will run together. (You can fix run-together text by adding a space as appropriate.)  
To insert a variable and a span tag in the same text Always insert the span tag first. The completed XML should look like:  
Click <span class="UIElementAction"><MadCap:variable name=" UserInterface.SelectAll" /></span>.    
Otherwise, the span does not show up correctly in the output.
