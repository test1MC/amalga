Staff handles link requests

 

This topic describes how patient linking requests are handled by staff.

If manual patient linking is configured, or if automatic linking is specified and an exact, unique match isn't found, link requests are sent to staff for decisions.

Staff process

  1. The staff user logs on to the and selects Patient Portal Management from the solution picker.
  2. The staff user selects the Link request list page from the carousel navigation bar.
  3. The staff user will see a list of open patient link requests and can select from the list.
  4. The staff user either approves, rejects, or escalates it.
  5. The next request is displayed, and the user continues processing until all requests have been handled.

At any time, the user can view the entire list of requests. If they do that before completing an assigned request, that request will be unassigned.

Supervisor role

It is assumed that someone will be assigned to a supervisory role to handle requests escalations and can review requests periodically. The supervisor can make sure that requests assigned to personnel who have left the organization are reassigned and dealt with in a timely manner.
