Site Themes

 

This topic describes the use of style sheets in modifying site themes.

The theme style sheet, mhsdefault.css, contains the styles for modifying site themes. The style sheet main.css contains structural, controlling elements such as page width, navigation location, and banner position. Customizations to main.css are not supported.

Theme styles map directly to the color and font customization guidelines developed by the User Experience (UX) team at Microsoft. Only styles that are designated in the UX Style Guidelines are supported. Unsupported style modifications can make upgrades unworkable.

All portals, end-to-end workflows, and associated controls that are deployed as part of the platform have gone through the rigorous internal product accessibility lifecycle. By following this process and the extensive set of requirements it contains, the platform conforms to most accessibility standards expected by legal and regulatory bodies around the world, and enables use with most available assistive technologies. The degree to which we comply with respect to US regulations is described in the Section 508 VPAT form available on the public Microsoft Accessibility website at http://www.microsoft.com/enable/. Follow the Section 508 link to navigate to the VPAT repository for all Microsoft products.

As a platform product that is intended to enable third parties to rapidly develop healthcare IT solutions, also provides a complete set of baseline components and tools to fully support development of software that meets the accessibility needs of their customers.

Order of style sheet precedence

The core styles (core.css) are applied first, then the structural styles, followed by the theme styles, so the theme style sheet can override all of the other styles.

Location of style sheets

All style sheets are deployed to this folder:

\Program Files\Common Files\Microsoft Shared\Web Server Extensions\14\TEMPLATE\LAYOUTS\STYLES\MHS.
