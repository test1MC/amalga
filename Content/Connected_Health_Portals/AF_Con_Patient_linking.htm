Patient linking

 

This topic explains how patient linking can be configured.

Patient linking is integrated into the patient's experience when signing up for an account or adding a new record. In patient linking, patients identify themselves to the hospital so that they can access their hospital records through the . Linking is the action of connecting an patient record to a record. Linking also gives the hospital access to the patient's data, depending on the application authorization privileges set for the organization.

Automatic and manual linking

Manual linking requires hospital staff to approve or reject all patient link requests. For more information about how staff users handle patient linking requests, see Staff handle link requests.

Automatic linking links the patient's information entered on the with a patient’s hospital record without interaction with hospital staff. The patient enters data that exactly and uniquely matches a hospital patient file. If no exact, unique match is found, manual intervention by hospital staff using the staff site will be required.

If another ID is already matched to that ID, it does not link again, but is sent to the hospital staff.

These are requirements for an exact, unique match:

  • All required patient-entered data and data in the database must match exactly.
  • Data entered by the patient must map to only a single patient in the database (a unique match).

If the automatic linking feature is turned on, the IT administrator must set a maximum number of retry attempts for patients to try to match and link from the patient to their records. This is a setting in the . For any match attempt that is not exact and unique, the count is incremented. For more information about settings for patient linking, see Patient linking configuration.

When the count reaches the allowable limit based on failed exact and unique matches, the application transitions into manual linking mode. generates a patient link request that hospital staff can see on the and process manually.

If the patient has reached the maximum number of attempts, the user interface of the takes this condition into account, and displays an appropriate message.The patient cannot fill out another patient link form until the hospital staff either approves or declines the original request, at which time the autolink request count is reset to zero.
