Internal provider site

 

This topic describes the content and controls available for the .

A basic and set of controls is provided with . A set of controls is available through the , which you can use to add new pages and features to the site. For more information, see Toolbox controls on page 1.

The contains the following items:

  • Provider site header on page 1 
  • Provider site logon control on page 1
  • Provider site footer

Provider site header

For the authenticated provider, the header is the first visible control layer . The primary function of this control is to indicate the context of the provider and provide a framework for site navigation.

 

As shipped, the header includes the institution brand, the provider account context, search, and navigational elements:

1 - SolutionPicker - allows you to navigate through the top-level sites.

2 - EnterpriseAccountContext - displays the authenticated user’s identity, and provides a link to sign off.

3 - NotificationIndicator - not available in this release (for placement only).

4 - PatientSearch - not available in this release

CarouselNavigationBar (not displayed) - allows you to navigate the pages (depends on the solution chosen).

As shipped, the does not include any pre-set pages, so the CarouselNavigationBar does not show.

Footer information (for example, legal content) should be added to the master page.

Provider site logon control

The site logon control includes a button to initiate the logon process for providers who have already signed up for a provider account.
