Staff Site

 

This topic describes the pages that make up the staff site and the primary controls that exist on the page.

A basic and set of control is provided with . A set of controls is available through the , which can be used to add new pages and features to the site.

Staff site header

For the authenticated staff user, the header is the first control layer visible.

As shipped with , the header includes the institution brand, the staff account context, search, and navigational elements:

1 - SolutionPicker - allows the user to navigate through the top-level sites

2 - EnterpriseAccountContext - displays the authenticated user’s identity, and provides a link for signing off.

3 - CarouselNavigationBar - allows the user to navigate the pages, depending on the solution chosen.

4 - PatientSearch - potential future functionality.

The contains the following pages:

  • Patient Link Request (PatientMatchStaffPage.aspx) - displays the information about the patient requesting the link and the possible candidates that match the entered data. The main controls on this page are:
  • PersonDataCard (red in the following image) - displays the patient demographics data as entered by the user in the input form. The list of fields displayed is read from the XML payload that is part of the link request task. If the fields in the payload change, the control will automatically adjust. If the number of fields is more than can be displayed inside the pane, scrollbars are added to allow the user to scroll to any content outsidethe view. The displayed labels are read from a resource file.
  • PotentialLinkCard (blue in the following image) - displays the patient demographics data from the database. Once a user selects a patient from the PotentialMatchList, the data for that patient is displayed in this section of the screen. The staff user can decide whether the information entered by the patient (on the left) matches the information in the system ( on the right). If yes, they can click Link to this patient file.
  • PotentialMatchList (green in the following image) - displays a list of candidates that partially match the entered patient data. The control is data bound in code to the result of a search performed by the patient linking service.

  • Link request list (Linkrequestlist.aspx) - lists the link-requests. This is the only page that is displayed under the Patient Portal Management solution in the CarouselNavigationBar control. The main component on the page is:
  • PatientLinkRequestList - lists the requests in the queue. It can indicate which items are marked for escalation and which are currently assigned to other users.
