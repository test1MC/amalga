Framework style sheets

This topic describes the style sheets in and their use.

There are three types of style sheets that control the look and feel of the provider sites.

  1. Structural- controls the basic layout of pages

    • Base.css – contains all common styles applicable to all portals. Use these styles whenever possible to keep site styling consistent.

    • Public.css – contains styles applicable only to public portal pages

    • Patient.css – contains styles applicable only to patient portal pages

    • InternalProvider.css – contains styles applicable only to internal provider portal pages

    • ExternalProvider.css – contains styles applicable only to external portal pages

    • Staff.css – contains styles applicable only to staff portal pages

  1. Themable - controls the default look and feel of pages when no theme is applied

    • Theme.css – contains all common styles applicable to all portals

    • Public.css – contains styles applicable only to public portal pages

    • Patient.css – contains styles applicable only to patient portal pages

    • InternalProvider.css – contains styles applicable only to internal provider portal pages

    • ExternalProvider.css – contains styles applicable only to external portal pages

    • Staff.css – contains styles applicable only to staff portal pages

Theme styles map directly to the color and font customization guidelines developed by the User Experience (UX) team at Microsoft. Only styles that are designated in the UX Style Guidelines are supported. Unsupported style modifications to the controls shipped with the base package can make upgrades unworkable.

  1. Customizable style sheets – can be used to customize the layout, or look and feel of a page. To minimize the number of style sheets a customer has to manage, these style sheets contain styles to support layout and look and feel.

    • Common.custom.css

    • Public.custom.css

    • Patient.custom.css

    • InternalProvider.custom.css

    • ExternalProvider.custom.css

    • Staff.custom.css

Style-sheet precedence

Style sheets are loaded for a public portal page in the following order.

1. Corev4.css

2. Amalga\Base.css

3. Amalga\Public.css

4. Themable\Amalga\Theme.css

5. Themable\Amalga\Public.css

6. Themable\Amalga\Common.custom.css

7. Themable\Amalga\Public.custom.css

By default, any master page with the following element “<SharePoint:CssLink runat="server" Version="4" />” in a environment will load corev4.css.

Location of style sheets

All style sheets are deployed to this folder: \Program Files\Common Files\Microsoft Shared\Web Server Extensions\14\TEMPLATE\LAYOUTS\1033\STYLES.
