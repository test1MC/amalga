Customize your sites

 

This topic describes a typical scenario for creating your own solutions using foundational site definitions.

The Foundational site definitions provide a starting point for the development of solutions that can be deployed for patient, provider, and staff access. Using the foundation sites provisioned with baseline master pages available in , you can create a customized experience for your customers.

The following steps represent a typical, but not comprehensive, scenario for customization:

  • Customize the look and feel of the portal header (logo, background color).
  • Create a new solution site in .

  • Link that site to the correct parent site so that it will appear in the solution picker.

    If you use the Staff Portal site template, the linking will be done automatically. If you use a different site template, contact your engineering partner for instructions.

  • Create a new empty Amalga project in and create pages for deployment to your sites.

  • Add controls to your pages. These can be any of the existing controls (see controls and Creating a new control) or you can create your own using the templates and base control classes accessed by clicking Add Item in .

  • Deploy the project to the site so that the pages will display in the Carousel Navigation bar

For more information about creating a customized experience, contact your engineering partner.
