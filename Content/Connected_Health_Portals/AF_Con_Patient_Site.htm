Patient site

 

This topic describes the content of the and what can be customized.

A basic and set of controls is provided with . A set of controls is available through the which can be used to add new pages and features to the site. For more information on controls, see Toolbox controls on page 1.

The patient site will contain both unauthenticated and authenticated pages. The authenticated pages will only be accessible by patients who have logged on to the site using their credentials. The unauthenticated logon page is provided for them to get authenticated.

Different master pages are applied to authenticated and unauthenticated pages. For more information about master pages, see Master pages on page 1.

Unauthenticated pages

Unauthenticated pages will contain:

  • Patient portal header
  • Sign-up control on page 1
  • Sign in control on page 1
  • Patient portal footer on page 1

header

For the unauthenticated patient portal, the header will include the organization branding area only. The look and content of the header is controlled by the master page.

The master page for the unauthenticated patient page is physically defined in the public site, even though it is presented as part of the patient site.

Authenticated pages

Authenticated pages will contain:

  • Patient site header
  • Patient portal footer on page 1

For the authenticated patient, the header is the first control layer visible. The primary function of this control is to indicate the context of the patient and provide a framework for site navigation. As shipped, it includes the institution brand, patient record selector, account context, and site navigation. As shipped, these controls are already authored into the header on the authenticated master page. Each control is also independently available from the Toolbox and can be used to create a custom master page. Each control also has a set of available customizations. For more information, see Toolbox controls on page 1.

The header and its child controls will remain visible whenever there is an active logged on session.

1 -Logo - displays your organization's logo.

2 - HealthVaultRecordSelector - enables access to other records for which the authenticated user has gone through the application authorization process. This authorization process enables the organization to view data from the user’s record and push data into their record. Data pushed into the record can be viewed in or through other authorized applications. records that the user created must also go through the application authorization process before they can be displayed in the record selector.

3 - EnterpriseAccountContext - shows the authenticated user’s identity, and provides a link to sign off.

The Help link is not functional in the release.

4 - CarouselNavigationBar - enables access to all other pages and workflow that complete the patient site functionality for your organization.

5 - Search - enables a patient to quickly search for information on the site.
