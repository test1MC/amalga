Site customization

 

This topic describes the ways that the sites can be customized.

All the elements of an site can be customized. Some customizations (like branding and legal content) are required to meet the specific needs of your hospital environment, while others (like fonts and colors) are optional.

This topic contains the following subsections:

  • Theming
  • Styling

Ways to customize a site

 

The provides several ways to customize the appearance and behavior of a site. They are listed below in order of increasing capability and complexity.

Theming

supports Office 2010 themes, which provide a quick and easy way to apply colors and different fonts to the content of a site. Any site owner or user with designer rights can customize themes by modifying the site settings in the browser.

For more information on modifying themes, see Themes overview.

Styling

The Framework portal sites use one style sheet (Base.css) to control the appearance and layout of every element in the page. The base style sheet defines the core size, spacing, and layout styles used by the site. Use these styles whenever possible to keep site styling consistent. For more information, see Base structural styles.

Customization

"Site customization" refers to changing or replacing any of the specific resources on the site. This includes, but is not limited to, the following:

Image files – logos, backgrounds, and other graphical page elements.

Resource files (.resx) – localizable text used by the page elements.

User control files (.ascx) – configuration for Web Parts and server controls.

Master pages (.master) - common structure for the pages of a site. For more information about the master pages, see Master pages on page 1.

Site pages associated with patient matching work flows should not be replaced, as this will break the work flow. Similarly, removing individual items from a patient matching work flow page will break the reference in the code-behind. It is safe to add new page items that reference the code-behind. For more information, see Customize patient matching work flows.

Control extension

The preceding tiers of customization allow changes to the structure and appearance of the sites. To change the functionality of the site, you may need to replace one or more of the controls that implement the site's behavior.

Control extension is the replacement of one site control with another, where the new control is derived from one of the controls shipped with the . Control extension allows the new control to inherit support for the page messaging system, application identity discovery, and other features without any additional programming.

For more information on extending controls, see controls overview and Creating a new control.

Control replacement

Control replacement is the process of writing a new control entirely from scratch, including implementation of the interfaces required to interact with other controls and services.
