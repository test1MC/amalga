Unpack CAB files

 

This topic describes how to locate and isolate pipeline application manifest files by feed.

Each pipeline application is compressed into a Doors CAB file within a specific folder structure called a package. An operational feed consists of the resolved manifest files for all the

To locate and unpack all pipeline manifest XML files

  1. Create a staging folder with a nested temporary folder. For example, create C:\Pipeline Manifest to hold the CAB files and C:\Pipeline Manifest\MasterConfig to hold the extracted XML files.
  2. Navigate to root\v3\Rel\Applications\Release and locate FeedAcquisition.CAB, MessageShredder.CAB, and DataParser.CAB.
  3. Copy the set of CAB files to the staging folder.
  4. For each CAB file, extract the XML files to the \MasterConfig subfolder. Keep the extracted XML files as master configuration files.
  5. Create a subfolder for each new feed. For example, create C:\Pipeline Manifest\ADT for ADT feed.
  6. Copy a full set of extracted XML configuration files to the feed folder. For example, copy the XML files from the master folder (C:\Pipeline Manifest\MasterConfig) to the feed folder (C:\Pipeline Manifest\ADT).
