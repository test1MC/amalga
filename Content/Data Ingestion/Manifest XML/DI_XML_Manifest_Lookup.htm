Lookup element

 

Declares a lookup function and the associated parameters for the specified segment element.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <DataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <Lookup>
                           <Function>...</Function>
                           <InputParameters />						
                        </Lookup>
                     </Element>
                  </Elements>
               </Segment>
            </DataParserOutput>
					
            <ExtensibleDataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <Lookup>
                           <Function>...</Function>
                           <InputParameters />	
                        </Lookup>
                     </Element>
                  </Elements>
               </Segment>
            </ExtensibleDataParserOutput>
					
            <ExpandableDataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <Lookup>
                           <Function>...</Function>
                           <InputParameters />					
                        </Lookup>
                     </Element>
                  </Elements>
              </Segment>
            </ExpandableDataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements Element
Child elements Function, InputParameters

Remarks

The Lookup element is mandatory in the data-parser service manifest to look up data that is external to message. Do not declare this element in the manifests of other pipeline applications.

Example

The following example converts a terminology concept code to descriptive text for the DiagnosisCleansedText column.

<Segment Name="DG1" SourceName="DG1">
   <Elements>
     <Element Name="DiagnosisCleansedText" Type="System.String" MaxLength="400">
        <ConceptId />
        <Lookup>
           <Function>Lookups.usf_Terminology_ConvertCodeToText</Function>
           <InputParameters>
              <Parameter Name="selectedTerminologyId" Type="System.String" SourceType="Constant" SourceName="ICD9ContentSetId" />
              <Parameter Name="codeTypeId" Type="System.String" SourceType="Constant" SourceName="ICD9CodingSystemTypeId" />
              <Parameter Name="incomingConceptCode" Type="System.String" SourceType="ShreddedInput" SourceTable="DG1" SourceName="DG1_3_1" />
           </InputParameters>
        </Lookup>
      </Element>
   </Elements>
</Segment>
