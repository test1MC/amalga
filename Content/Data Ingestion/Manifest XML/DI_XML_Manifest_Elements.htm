Elements element

 

Declares a one or more elements of a segment.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <MessageShredderInput>
               <Segment>					
                  <Elements>
                    <Element>...</Element>
                  </Elements>
               </Segment>
            </MessageShredderInput>				

            <DataParserOutput>
               <Segment>					
                  <Elements>
                     <Element />
                  </Elements>
               </Segment>
            </DataParserOutput>
					
            <ExtensibleDataParserOutput>
               <Segment>					
                  <Elements>
                     <Element />
                  </Elements>
               </Segment>
            </ExtensibleDataParserOutput>
					
            <ExpandableDataParserOutput>
               <Segment>					
                  <Elements>
                     <Element />
                  </Elements>
               </Segment>
            </ExpandableDataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length Array
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements Segment
Child elements Element

Remarks

A segment element represents a shredded table column in a data-parser service manifest. Every column of source type "ShreddedInput" that you intend to modify within a data parser must be declared explicitly in the MessageShredderInput element section of the service manifest.

The Elements element can include one or more Element element. You must declare an element for a second time as data-parser output to apply a transform action, EnterpriseId action, or lookup function to the element. Do not declare this element in the manifests of other pipeline applications.

Example

The following example first shows the format to use in the MessageShredderInput element section.

<MessageShredderInput>
    <Segment Name="PID" SourceName="PID">
        <Elements>
            <Element Name="PID_5_1" />
            <Element Name="PID_5_2" />
        </Elements>
    </Segment>
</MessageShredderInput>

Next, the example applies a transform action to segment element PID_5_1 in the DataParserOutput element section. 

<DataParserOutput>
   <Segment>
      <Elements>
         <Element Name="FamilyName" Type="System.String" MaxLength="100">
            <ConceptId>1uJSHAEy5J0eLAWfCsgBAA</ConceptId>
            <Transform>
               <Action>ToTitleCase</Action>
               <InputParameters>
                     <Parameter SourceType="ShreddedInput" SourceTable="PID" SourceName="PID_5_1" />
               </InputParameters>
            </Transform>
         </Element>
      </Elements>
   </Segment>
<DataParserOutput>

Use the same format to specify a transform action or lookup function in the ExtensibleDataParserOutput element and ExpandableDataParserOutput element sections.
