DefaultAck element

 

Declares one or more acknowledgement bytes.

SyntaxErrorCodes element

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <Feeds>
             <Feed Name="String">
                <ReceiveAdapter> Name="String"
                    <DefaultAck>...</DefaultAck>
                </ReceiveAdapter>
              </Feed>	
          </Feeds>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length String
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements ReceiveAdapter
Child elements None

Remarks

The DefaultAck element specifies the acknowledgement bytes to be used in case the AckCapture element fails to match the message and is not able to create the ACK messages. If the AckCapture element fails to match the incoming message, and the message verification is successful, the adapter returns the default byte or bytes. Use a comma to separate different byte settings when you configure multiple settings.

You can use the DefaultAck element to configure an instance of the when the message transport format is MLLP. Before applying this element tag, set the Name attribute value of the ReceiveAdapter element to "Microsoft.Amalga.Pipeline.AcquisitionAdapters.MLLPServer". Do not declare this element in the manifests of other pipeline applications.

Example

<defaultack>0x0B,0x15,0x1C,0x0D</defaultack>
