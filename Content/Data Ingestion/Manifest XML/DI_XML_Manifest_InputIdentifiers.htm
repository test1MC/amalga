InputIdentifiers element

 

Declares one or more source identifiers for enterprise identifier mapping.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <DataParserConfiguration>
            <EnterpriseId>
                <EnterpriseIdAction> Action=String AnchorTable=String >
                   <InputIdentifiers>
                      <EncounterIdentifier>...</EncounterIdentifier>
                      <EncounterSetIdentifier>...</EncounterSetIdentifier>
                      <PatientRecordIdentifier>...</PatientRecordIdentifier>
                      <PersonIdentifier>...</PersonIdentifier>
                      <ProviderIdentifier>...</ProviderIdentifier>
                   </InputIdentifiers>
                </EnterpriseIdAction>
             </EnterpriseId>
          </DataParserConfiguration>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-n: Optional element that can occur one or more times.

Element relationships

Relationship Element
Parent elements EnterpriseIdAction
Child elements EncounterIdentifier, EncounterSetIdentifier, PatientRecordIdentifier, PersonIdentifier, ProviderIdentifier

Remarks

The InputIdentifier element contains one or more source identifier parameters. A source identifier parameter can represent an encounter, encounter set, patient record, person, or provider. The InputIdentifiers element must include at least one source identifier, but it is not mandatory to define every source identifier.

Declare the InputIdentifiers element in the service manifest of the . Do not declare this element in the manifests of other pipeline applications.
