PrimaryKeyColumn element

 

Declares the primary key for the resulting table.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <DataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <PrimaryKeyColumn>...</PrimaryKeyColumn>
                  </CollapseConfiguration>
               </Segment>
            </DataParserOutput>
					
            <ExtensibleDataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <PrimaryKeyColumn>...</PrimaryKeyColumn>
                  </CollapseConfiguration>
               </Segment>
            </ExtensibleDataParserOutput>
					
            <ExpandableDataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <PrimaryKeyColumn>...</PrimaryKeyColumn>
                 </CollapseConfiguration>
              </Segment>
            </ExpandableDataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length String
Default value None
Cardinality 0-1: Optional element that can occur once and only once

Element relationships

Relationship Element
Parent elements CollapseConfiguration
Child elements None

Remarks

The PrimaryKeyColumn element, when set on a column of a segment table, enables you to refine a collapsed view that is established by the ContextKey attribute of the Element element. Define only one PrimaryKeyColumn element for each segment that you define in a data-parser service manifest. Do not declare this element in the manifests of other pipeline applications.

Always set the value of the ContextKey attribute, which is a column, to equal the value of the PrimaryKeyColumn element. For example, if you declare the Insurance_IN1_ContextKey column to be the context key for the table, you must set the same value as the primary key column for the table.

Example

<PrimaryKeyColumn>Insurance_IN1_ContextKey</PrimaryKeyColumn>
