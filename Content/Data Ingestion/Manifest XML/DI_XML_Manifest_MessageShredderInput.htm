MessageShredderInput element

 

Declares segment elements to be used by other sections of the data-parser service manifest.

Syntax

[ <Setting Name="SettingNameXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <MessageShredderInput>
               <Segment />			
            </MessageShredderInput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length Array
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DataParserConfiguration
Child elements Segment

Remarks

The MessageShredderInput element section is mandatory in all data-parser service manifests when data parser output requires shredded input. Do not declare this element in the manifests of other pipeline applications.

Example

The following example lists two segment elements that can be used as input to other data-parser operations. 

<MessageShredderInput>
    <Segment Name="PID" SourceName="PID">
        <Elements>
            <Element Name="PID_5_1" />
            <Element Name="PID_5_2" />
        </Elements>
    </Segment>
</MessageShredderInput>
