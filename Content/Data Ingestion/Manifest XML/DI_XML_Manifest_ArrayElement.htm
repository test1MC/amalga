Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-n: Optional element that can occur one or more times.

Element relationships

Relationship Element
Parent elements Array
Child elements None (or Array when used with source identifier elements).

Attributes

Attribute Description
Name

(Optional string.) Name of the array element.

SourceType

(Required string.) Defines the input source. The following values are allowed.

Value Description
ShreddedInput

Reads input from a shredded message. You must also specify values for the SourceTable and SourceName attributes in the array element.

Always explicitly declare SourceTable and SourceName attribute values in the parser service manifest using the MessageShredderInput element before adding them to an array element.
DerivedField Reads a field that you derive in the parser service manifest. You must also specify values for the SourceTable and SourceName attributes in the array element.
Inline Reads a value that you specify in the parameter using the Value attribute.
Constant Reads a variable list of arguments that you declare with the Array element. Use the Type attribute to indicate the array type.
Array Reads a global value. You must declare the constant first in the DataConstants element and then specify the named constant with the SourceName attribute.
SourceTable (Optional string) Specifies the table from which to read the data. For example, if you want to define a transform for table PID, set the SourceTable attribute to "PID" or any of its parents (MSH).
SourceName (Optional string) The name of the table column or constant as indicated by the source type.
Type (Optional string) Data type such as System.String.
Value (Optional string) Specifies an explicit inline value, such as "|" (pipe).

Remarks

The ArrayElement element specifies a variable list of array elements. When used with a source identifier like the PatientRecordIdentifier element, you can specify another variable list of array elements.

Declare the ArrayElement element in the service manifest of the . Do not declare this element in the manifests of other pipeline applications.

Example

 <Parameter Name="Arguments" SourceType="Array" Type="System.String">
    <Array>
       <ArrayElement SourceType="ShreddedInput" SourceTable="OBR" SourceName="OBR_4" />
       <ArrayElement SourceType="ShreddedInput" SourceTable="OBX" SourceName="OBX_1" />
    </Array>
</Parameter>
