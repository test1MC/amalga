KeepOnTemporaryStorage element

 

Controls the movement of messages from temporary storage to the database.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <Feeds>
             <Feed Name="String">
                 <KeepOnTemporaryStorage>...</KeepOnTemporaryStorage> 
             </Feed>	
          </Feeds>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length Boolean
Default value False
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements Feed
Child elements None

Remarks

If you set the KeepOnTemporaryStorage element value to true, messages remain in temporary storage and do not move to the database. While this behavior can be helpful during internal testing, you should not set the value to true for production application instances.

Use the KeepOnTemporaryStorage element to configure an instance of the when the message transport format is MLLP. Before applying this element tag, set the Name attribute value of the ReceiveAdapter element to "Microsoft.Amalga.Pipeline.AcquisitionAdapters.MLLPServer". Do not declare this element in the manifests of other pipeline applications.

Example

<KeepOnTemporaryStorage>true<KeepOnTemporaryStorage>
