Domain element

 

Declares a set of source identity parameters.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <DataParserConfiguration>
            <EnterpriseId>
                <EnterpriseIdAction> TransformName=String AnchorTable=String >
                   <InputIdentifiers>
                      <PatientRecordIdentifier>
                         <Domain><Array/></Domain>
                      <PersonIdentifier>
                         <Domain><Array/></Domain>
                      <ProviderIdentifier>
                         <Domain><Array/></Domain>
                      <EncounterSetIdentifier>
                         <Domain><Array/></Domain>
                      <EncounterIdentifier>
                         <Domain><Array/></Domain>
                   </InputIdentifiers>>
                </EnterpriseIdAction>
             </EnterpriseId>
          </DataParserConfiguration>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements EncounterIdentifier, EncounterSetIdentifier, PatientRecordIdentifier, PersonIdentifier, ProviderIdentifier
Child elements Array

Attributes

Attribute Description
Name

(Optional string.) Name of the domain.

SourceType

(Required string.) Defines the input source. The following values are allowed.

Value Description
ShreddedInput

Reads input from a shredded message. You must also specify values for the SourceTable and SourceName attributes in the domain.

Always explicitly declare SourceTable and SourceName attribute values in the parser service manifest using the MessageShredderInput element before adding them to a domain.
DerivedField Reads a field that you derive in the parser service manifest. You must also specify values for the SourceTable and SourceName attributes in the domain.
Inline Reads a value that you specify in the parameter using the Value attribute.
Constant Reads a global value. You must declare the constant first in the DataConstants element and then specify the named constant with the SourceName attribute.
Array Reads a variable list of arguments that you declare with the Array element. Use the Type attribute to indicate the array type.
SourceTable (Optional string) Specifies the table from which to read the data. For example, if you want to define a transform for table PID, set the SourceTable attribute to "PID" or any of its parents (MSH).
SourceName (Optional string) The name of the table column or constant as indicated by the source type.
Type (Optional string) Data type such as System.String.
Value (Optional string) Specifies an explicit inline value, such as "|" (pipe).

Remarks

The Domain element value when concatenated with the Id element value fully identifies an encounter, encounter set, patient record, person, or provider source. You must configure at least one source identifier.

Constraints

  • Declare the Domain element only within a service manifest of the .
  • Specify the Domain element only once for each source identifier that you define.

Example

<Domain Name="Domain" Type="System.String" SourceType="ShreddedInput" SourceTable="PID" SourceName="PID_3_5" />
