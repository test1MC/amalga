NackCapture element

 

Declares a regular expression capture string for different elements.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <Feeds>
             <Feed Name="String">
                <ReceiveAdapter> Name="String"
                   <NackCapture>...</NackCapture>
                </ReceiveAdapter>
             </Feed>	
          </Feeds>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length String
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements ReceiveAdapter
Child elements None

Remarks

The NackCapture element defines the regular expression capture string used by the application instance to acquire the different HL7 elements within each message. HL7 elements appear as substrings between separator characters (“|”). For more information about defining the capture string of a regular expression, see Regular Expression Classes at http://go.microsoft.com/fwlink/?LinkID=230529 or How to match a pattern by using regular expressions and Visual C# at http://go.microsoft.com/fwlink/?LinkID=230532.

You can use the NackCapture element to configure an instance of the when the message transport format is MLLP. Before applying this element tag, set the Name attribute value of the ReceiveAdapter element to "Microsoft.Amalga.Pipeline.AcquisitionAdapters.MLLPServer". Do not declare this element in the manifests of other pipeline applications.

Example

In the following example, (?&lt;tok0&gt;[^|]*) defines a capture element of (?<tok0>[^|]*). If the incoming message is “MSH|...”, the application sets the value of <tok0> equal to “MSH” in the database.

<nackcapture>
(?&lt;tok0&gt;[^|]*)\|(?&lt;tok1&gt;[^|]*)\|(?&lt;tok2&gt;[^|]*)\|(?&lt;tok3&gt;[^|]*)\|(?&lt;tok4&gt;[^|]*)\|(?&lt;tok5&gt;[^|]*)\|(?&lt;tok6&gt;[^|]*)\|(?&lt;tok7&gt;[^|]*)\|(?&lt;tok8&gt;[^|]*)\|(?&lt;tok9&gt;[^|]*)\|(?&lt;tok10&gt;[^|]*)\|(?&lt;tok11&gt;[^|]*)\|(?&lt;tok12&gt;[^|]*)\|(?&lt;tok13&gt;[^|]*)</nackcapture><nackcapture>(?&lt;tok0&gt;[^|]*)\|(?&lt;tok1&gt;[^|]*)\|(?&lt;tok2&gt;[^|]*)\|(?&lt;tok3&gt;[^|]*)\|(?&lt;tok4&gt;[^|]*)\|(?&lt;tok5&gt;[^|]*)\|(?&lt;tok6&gt;[^|]*)\|(?&lt;tok7&gt;[^|]*)\|(?&lt;tok8&gt;[^|]*)\|(?&lt;tok9&gt;[^|]*)\|(?&lt;tok10&gt;[^|]*)\|(?&lt;tok11&gt;[^|]*)\|(?&lt;tok12&gt;[^|]*)\|(?&lt;tok13&gt;[^|]*)
</nackcapture>
