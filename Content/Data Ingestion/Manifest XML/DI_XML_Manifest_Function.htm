Function element

 

Declares a named function for a lookup operation.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <DataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <Lookup>
                           <Function>...</Function>			
                        </Lookup>
                     </Element>
                  </Elements>
               </Segment>
            </DataParserOutput>
					
            <ExtensibleDataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <Lookup>
                           <Function>...</Function>
                        </Lookup>
                     </Element>
                  </Elements>
               </Segment>
            </ExtensibleDataParserOutput>
					
            <ExpandableDataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <Lookup>
                           <Function>...</Function>						
                        </Lookup>
                     </Element>
                  </Elements>
              </Segment>
            </ExpandableDataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements Lookup
Child elements None

Remarks

The Function element is mandatory in a data-parser service manifest to perform a lookup function involving the specified segment element. Use the InputParameter element to pass in parameters to the function. Do not declare this element in the manifests of other pipeline applications.

Example

The following example converts the terminology code in the message to a different code from the terminology database.

<Lookup>
   <Function>Lookups.usf_Terminology_ConvertCodeToCode</Function>
   <InputParameters>
      <Parameter Name="incomingConceptCode" Type="System.String" SourceType="ShreddedInput" SourceTable="DG1" SourceName="DG1_3_1" />
      <Parameter Name="conversionTerminologyId" Type="System.String" SourceType="Constant" SourceName="ICD9ToSNOMEDConversionTerminologyId" />
   </InputParameters>
</Lookup>
