FinalAction element

 

Declares the action that an application executes after reaching the maximum retry time.

Syntax

[ <Setting Name="ErrorHandlingConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <ErrorHandlingConfiguration>
             <ErrorHandlingPolicies>
                <ErrorHandlingPolicy>
                   <FinalAction>...</FinalAction>
                </ErrorHandlingPolicy>	
             </ErrorHandlingPolicies>
          </ErrorHandlingConfiguration> 
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length String
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements ErrorHandlingPolicy
Child elements None

Remarks

You can configure the FinalAction element to stop or continue. In the former, the process stops entirely. In the latter, continue ignores the error and continues processing the same message. You can declare a FinalAction element only when the Action element value equals Retry for the specified exception.

Declare the FinalAction element only within a service manifest of the , , , and applications.

Example

<Action>Retry</Action>
<MaximumRetry>10</MaximumRetry>
<FinalAction>Continue</FinalAction>
