LogSetting element

 

Declares a log entry for the specified exception.

Syntax

[ <Setting Name="ErrorHandlingConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <ErrorHandlingConfiguration>
             <ErrorHandlingPolicies>
               <ErrorHandlingPolicy>
                   <LogSetting>
                      <EventId>...</EventId>
                      <ErrorTitle>...</ErrorTitle>
                      <ErrorDescription>...</ErrorDescription>
                   </LogSetting>
                </ErrorHandlingPolicy>	
             </ErrorHandlingPolicies>
          </ErrorHandlingConfiguration>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements ErrorHandlingPolicy
Child elements EventId, ErrorTitle, ErrorDescription

Remarks

The LogSetting element provides a consistent format for logging the private errors that you define explicitly in a manifest. You must declare a log setting for each policy.

Declare the LogSetting element only within a service manifest of the , , , and applications.

Example

 <LogSetting>
        <EventId>33001</EventId>
        <ErrorTitle>DbServerNotResponding</ErrorTitle>
        <ErrorDescription>The component has attempted to connect to the database server; however, the connection has timed out.</ErrorDescription>
</LogSetting>
