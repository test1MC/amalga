CollapseConfiguration element

 

Declares the characteristics of a collapsed view.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <DataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <PrimaryKeyColumn>...</PrimaryKeyColumn>
                     <ResultType>...</ResultType>
                     <ResultTypeSortField>...</ResultTypeSortField>		
                     <ResultRowsFilter>...</ResultRowsFilter>
                  </CollapseConfiguration>
               </Segment>
            </DataParserOutput>
					
            <ExtensibleDataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <PrimaryKeyColumn>...</PrimaryKeyColumn>
                     <ResultType>...</ResultType>
                     <ResultTypeSortField>...</ResultTypeSortField>				
                     <ResultRowsFilter>...</ResultRowsFilter>
                  </CollapseConfiguration>
               </Segment>
            </ExtensibleDataParserOutput>
					
            <ExpandableDataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <PrimaryKeyColumn>...</PrimaryKeyColumn>
                     <ResultType>...</ResultType>
                     <ResultTypeSortField>...</ResultTypeSortField>
                     <ResultRowsFilter>...</ResultRowsFilter>
                 </CollapseConfiguration>
              </Segment>
            </ExpandableDataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements Segment
Child elements PrimaryKeyColumn, ResultRowsFilter, ResultType, ResultTypeSortField

Remarks

A context key specified for one column of a segment table produces a collapsed view of the data by default. In addition to a context key, you can declare the CollapseConfiguration element to refine the default view results for a better assisted-binding experience. First set the ContextKey attribute of the Element element and then use the element name (column name) as the value to the (child) PrimaryKeyColumn element.

Define both the context key and collapse configuration (one each for a table) in the data-parser service manifest. Do not declare this element in the manifests of other pipeline applications.

Example

<CollapseConfiguration>
      <PrimaryKeyColumn>Insurance_IN1_ContextKey</PrimaryKeyColumn>
      <ResultRowsFilter>Insurance_IN1_ContextKey is not null and Insurance_IN1_ContextKey &lt;&gt; ''</ResultRowsFilter>
</CollapseConfiguration>
