SearchPattern element

 

Declares a pattern used to search a subdirectory for files to acquire.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <Feeds>
             <Feed Name="String">
                <ReceiveAdapter> Name="String"
                   <SearchPattern>...</SearchPattern>
                </ReceiveAdapter>
             </Feed>	
          </Feeds>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length String
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements ReceiveAdapter
Child elements None

Remarks

The SearchPattern element is mandatory when the ReceiveAdapter element Name attribute is "Microsoft.Amalga.Pipeline.AcquisitionAdapters.FileAdapter" and you have provided a valid Directory element value. Do not declare this element in the manifests of other pipeline applications.

Your search pattern can include wildcard characters. For example, to search for all LAB messages in the folder, specify *.lab as the element value.

Example

The following search pattern returns all files from the specified directory folder and its subdirectory folders.

<searchpattern>*.*</searchpattern>
