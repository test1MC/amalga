EnterpriseIdAction element

 

Declares an enterprise identifier transform action.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <DataParserConfiguration>
            <EnterpriseId>
                <EnterpriseIdAction> Action=String AnchorTable=String >
                   <Condition>...</Condition>
                   <InputIdentifiers />
                   <InputParameter />
                </EnterpriseIdAction>
             </EnterpriseId> 
          </DataParserConfiguration>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-n: Optional element that can occur one or more times.

Element relationships

Relationship Element
Parent elements EnterpriseId
Child elements Condition, InputIdentifiers, InputParameters

Attributes

Attribute Description
Action

(Required string.) The name of the transform action. The following values are allowed.

Value Description
Merge Explicitly merges an identifier at the specified tier.
MergePatient Explicitly merges an identifier at the SOID tier.
StandardMapping Creates a source group and adds each source identifier parameter value as a mapping for the corresponding tier in the source group.
Reparent Moves identifiers under a specified parent identifier.
AnchorTable (Required string.) The table that provides shredded message data.

Remarks

Use the StandardMapping attribute to configure person, patient record, encounter set, and encounter identifiers. Configure at least one feed for every source identifier that you need to match using the . In addition, any feed that requests explicit merge and move operations must engage the . Typically, you should includes all ADT feeds.

Declare the EnterpriseIdAction element in the service manifest of the . Do not declare this element in the manifests of other pipeline applications.

Example

<EnterpriseIdAction Action="StandardMapping" AnchorTable="PID">
   <InputIdentifiers>
      <PatientRecordIdentifier Name="Patient">
         <Id Name="Id" Type="System.String" SourceType="ShreddedInput" SourceTable="PID" SourceName="MRN" />
         <Domain Name="Domain" Type="System.String" SourceType="ShreddedInput" SourceTable="PID" SourceName="PID_3_5" />
      </PatientRecordIdentifier>
      <EncounterIdentifier Name="Encounter">
         <Id Name="Id" Type="System.String" SourceType="ShreddedInput" SourceTable="PID" SourceName="PID_18_1" />
         <Domain Name="Domain" Type="System.String" SourceType="Constant" SourceName="Domain" />
     </EncounterIdentifier>
   </InputIdentifiers>
</EnterpriseIdAction>
