Element element

 

Declares an element of a segment.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <MessageShredderInput>
               <Segment>					
                  <Elements>
                     <Element>...</Element>
                  </Elements>
               </Segment>
            </MessageShredderInput>				
 
            <DataParserOutput>
               <Segment>					
                  <Elements />
                     <Element>
                        <ConceptId>...<ConceptId>
                        <Lookup />
                        <Transform />
                     </Element>
                  </Elements>
               </Segment>
            </DataParserOutput>
					
            <ExtensibleDataParserOutput>
               <Segment>					
                  <Elements />
                     <Element>
                        <ConceptId>...<ConceptId>
                        <Lookup />
                        <Transform />
                     </Element>
                  </Elements>
               </Segment>
            </ExtensibleDataParserOutput>
					
            <ExpandableDataParserOutput>
               <Segment>					
                  <Elements />
                     <Element>
                        <ConceptId>...<ConceptId>
                        <Lookup />
                        <Transform />
                     </Element>
                  </Elements>
               </Segment>
            </ExpandableDataParserOutput>
					
            <DataConstants>
               <Element>...</Element>					
            </DataConstants>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-n: Optional element that can occur one or more times.

Element relationships

Relationship Element
Parent elements Elements, DataConstants
Child elements ConceptId, Lookup, Transform

Attributes

Attribute Description
Name

(Required string.) Name of a shredded or derived element.
SourceName (Optional string) Name of segment and element from which the data for the table derives.
Type (Optional string) Data type such as System.String.
MaxLength (Optional string) Maximum length of the derived column.
Persisted

(Optional boolean) True to persist the segment element to the database; otherwise, False. True by default.

Do not set this attribute to False.
ContextKey

(Optional boolean) True to establish the element as a logical record identifier (collapsing key); otherwise, False. The default value is False.

You can only define one context key for a table.
ReferenceKey (Optional boolean) True to set a non-clustered index on the column and to specify the column as the context key to another table for use in assisted binding; otherwise, False. The default is False.
Reparse (Optional boolean) True to back fill the column represented by the element; otherwise, False. The default value is False.

Use this attribute only in a Backfill Parser service manifest.

Remarks

The Element element is mandatory in the FeedConfigurationXML setting of all data-parser service manifests when one or more table columns are derived, transformed, or are used in a lookup function or plug-in logic. Do not declare this element in the manifests of other pipeline applications.

Always declare an element in the MessageShredderInput element section of the service manifest if you intend to use it as input to an action, function, or to specify a context key. For an input declaration, you can only specify the Name, SourceName, and Type attributes. The other attributes are invalid. Child elements are also invalid for input declarations. The same restrictions apply to an element when you declare it within the DataConstants element.

In contrast, output declarations (DataParserOutput element, ExtensibleDataParserOutput element, and ExpandableDataParserOutput element) accept all attributes and child elements. Never set the Reparse attribute to True in your data-parser service manifest. For more information about the Reparse attribute, see Enable backfill parsing.

Example

The following example first shows the format to use in the MessageShredderInput element section.

<MessageShredderInput>
    <Segment Name="PID" SourceName="PID">
        <Elements>
            <Element Name="PID_5_1" />
            <Element Name="PID_5_2" />
        </Elements>
    </Segment>
</MessageShredderInput>

Next, the example applies a transform action to segment element PID_5_1 in the DataParserOutput element section. 

<DataParserOutput>
   <Segment>
      <Elements>
         <Element Name="FamilyName" Type="System.String" MaxLength="100">
            <ConceptId>1uJSHAEy5J0eLAWfCsgBAA</ConceptId>
            <Transform>
               <Action>ToTitleCase</Action>
               <InputParameters>
                     <Parameter SourceType="ShreddedInput" SourceTable="PID" SourceName="PID_5_1" />
               </InputParameters>
            </Transform>
         </Element>
      </Elements>
   </Segment>
<DataParserOutput>

Use the same format to specify a transform action or lookup function in the ExtensibleDataParserOutput element and ExpandableDataParserOutput element sections.
