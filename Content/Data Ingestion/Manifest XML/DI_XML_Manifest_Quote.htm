Quote element

 

Declares the character used for delineating entire strings.

Syntax

[ <Setting Name="CustomizedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <CSVConfiguration>
            <Quote />
         </CSVConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value Double quote (")
Cardinality 0-1: Optional element that can occur once and only once

Element relationships

Relationship Element
Parent elements CSVConfiguration
Child elements None

Remarks

Message fields with embedded double-quote characters must be enclosed within a second set of double-quote characters, and the entire string must be enclosed in double-quote characters. For instance, "luxurious" in the following incoming message example requires a second set of quotes for the original set to appear in the column name.

1997,Ford,E350,"Super, ""luxurious"" truck"

Declare the Quote element in a CSV-shredder service manifest as an empty element set to use the default double quote (“) character. Otherwise, specify an alternative value to use as the quote character in the XML element. Do not declare this element in the manifests of other pipeline applications.

 

Example

The following example specifies the default double-quote characters.

<Quote></Quote>
