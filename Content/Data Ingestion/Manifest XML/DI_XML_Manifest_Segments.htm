Segments element

 

Declares one or more message segments.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <MessageShredderConfiguration>
            <Segments>
               <Segment>...</Segment>					
            </Segments>
         </MessageShredderConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements MessageShredderConfiguration
Child elements Segment

Remarks

The Segments element is mandatory when you declare one or more Segment elements in the FeedConfigurationXML setting of a service manifest. Declare a segment only when you want to specify the parent of the specified segment.

Do not declare this element in the manifests of other pipeline applications. In particular, do not declare a Segments element in a data parser service manifest when you specify a segment for derivation.

Example

The following example specifies a parent segment for each segment in the set. The MSH segment has no parent segment.

<MessageShredderConfiguration>
    <Segments>
        <Segment Name="MSH" SourceName="MSH" Description="Message Header" />
        <Segment Name="PID" SourceName="PID" ParentName="MSH" Description="Person Identification" />
        <Segment Name="NK1" SourceName="NK1" ParentName="PID" Description="Next of Kin" />
        <Segment Name="PV1" SourceName="PV1" ParentName="PID" Description="Patient Visit" />
        <Segment Name="PV2" SourceName="PV2" ParentName="PV1" Description="Patient Visit" />
        <Segment Name="IN1" SourceName="IN1" ParentName="PV1" Description="Insurance Information" />
        <Segment Name="GT1" SourceName="GT1" ParentName="PV1" Description="Guarantor" />
        <Segment Name="DG1" SourceName="DG1" ParentName="PV1" Description="Diagnosis" />
    </Segments>
</MessageShredderConfiguration>
