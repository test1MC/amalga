NackFormular element

 

Declares the negative acknowledgement string that the adapter returns to the source system.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <Feeds>
             <Feed Name="String">
                <ReceiveAdapter> Name="String"
                   <NackFormular>...</NackFormular>
                </ReceiveAdapter>
             </Feed>	
          </Feeds>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length String
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements ReceiveAdapter
Child elements None

Remarks

The NackFormular element defines the negative acknowledgement string that the adapter returns to the source system. You can use the NackFormular element to configure an instance of the when the message transport format is MLLP. Before applying this element tag, set the Name attribute value of the ReceiveAdapter element to "Microsoft.Amalga.Pipeline.AcquisitionAdapters.MLLPServer". Do not declare this element in the manifests of other pipeline applications.

Example

In this example, the adapter replaces <tok0>, <tok1>, <tok2>, and so on with the actual capture string obtained from the NackCapture element.

<nackformular>
MSH|&lt;tok1&gt;|&lt;tok4&gt;|&lt;tok5&gt;|&lt;tok2&gt;|&lt;tok3&gt;|$date||ACK^&lt;tok8&gt;|&lt;tok9&gt;|&lt;tok10&gt;|2.1|&lt;tok12&gt;\rMSA|AA|&lt;tok9&gt;||&lt;tok12&gt;
</nackformular>
