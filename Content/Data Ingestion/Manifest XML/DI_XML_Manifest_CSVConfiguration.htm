CSVConfiguration element

 

Declares a collection element to define the characteristics of a comma-separated values (CSV) shredder.

Syntax

[ <Setting Name="CustomizedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <CSVConfiguration>
            <RowDelimiter />
            <FieldDelimiter />
            <Quote />
            <SkipFirstRow />
         </CSVConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DefaultValue
Child elements None

Remarks

CSV is a delimited data format with fields (or columns) separated by the comma character and records (or rows) terminated by new lines. Fields that contain a special character, such as a comma, new line, or double quote, must be enclosed in double quotes.

The CSVConfiguration element in the message-shredder service manifest redefines the default values used to shred CSV messages. Do not declare this element in the manifests of other pipeline applications.

CSV shredder characteristic Default value
Row delimiter New line (\r\n)
Field delimiter Comma (,)
Quote Double quote (")
Skip first row False

Example

<CSVConfiguration>
   <RowDelimiter>|</RowDelimiter>
   <FieldDelimiter>~</FieldDelimiter>
   <Quote></Quote>
   <SkipFirstRow>true</SkipFirstRow>
</CSVConfiguration>
