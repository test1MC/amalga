Lookups.usf_Terminology_ConvertCodeToCode function

Maps an incoming terminology code to a destination terminology code.

Syntax

[<Lookup>			
   <Function>Lookups.usf_Terminology_ConvertCodeToCode</Function>
   <InputParameters>
      <Parameter>Name="incomingConceptCode"...</Parameter>
      <Parameter>Name="conversionTerminologyId"...</Parameter>
   </InputParameters>
</Lookup>]

Parameters

Parameter Data type Description
incomingConceptCode System.String

A terminology code string. For example, the ICD code string is 480.1.
conversionTerminologyId System.String HLI GUID for terminology mapping content set. For example, the ICD9 to SNOMED conversion mapping content-set GUID is ABjfrwDwaR0MBAI62I5XRA.

Return value

VARCHAR(400)

If there are multiple matches, only one will be returned.

Remarks

For a list of valid attributes to use with this function, see the Parameter element. Apply this terminology function to any segment element (or column) that you declare in the data-parser service manifest. Do not declare this element in the manifests of other pipeline applications.
