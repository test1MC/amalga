Array element

 

Declares a list of arguments as a parameter.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>

            <DataParserOutput>(Note: Element levels removed.)
                  <InputParameters>

                        <Array />
                     </Parameter>
                  </InputParameters>
            </DataParserOutput>

            <EnterpriseId>
               <EnterpriseIdAction> TransformName=String AnchorTable=String >
                  <InputIdentifiers>
                     <PatientRecordIdentifier>
                        <Id>

                        </Id>
                        <Domain>

                        </Domain>
                     <PersonIdentifier>
                        <Id>

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements Parameter
Child elements ArrayElement

Remarks

Declare the Array element in the service manifest of the . Do not declare this element in the manifests of other pipeline applications.

Example

<Element Name="Result_OBX_ContextKey" Type="System.String" MaxLength="200" ContextKey="true">
    <ConceptId>f/xuQgEy71iABYBTCrJmYw</ConceptId>
    <Transform>
        <Action>ConcatenateStringList</Action>
        <InputParameters>
            <Parameter Name="Arguments" SourceType="Array" Type="System.String">
                <Array>
                    <ArrayElement SourceType="ShreddedInput" SourceTable="OBR" SourceName="OBR_4" />
                    <ArrayElement SourceType="ShreddedInput" SourceTable="OBX" SourceName="OBX_1" />
                </Array>
            </Parameter>
        </InputParameters>
    </Transform>
</Element>
