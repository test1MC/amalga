Plugin element

 

Declares the characteristics of plug-in code.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <Plugins>
               <Plugin>
                  <PluginData>...</PluginData>
                  <URL>...</URL>
                  <SourceType>...</SourceType>
                  <NameSpace>...</NameSpace>
                  <Dependencies />				
               </Plugin>			
            </Plugins>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]
[ <Setting Name="PluginModelConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <Plugin>

            <SourceType>...</SourceType>
            <NameSpace>...</NameSpace>
            <Dependencies />				
         </Plugin>		
      ]]>
   </DefaultValue>
</Setting> 
[ <Setting Name="ShredderPluginConfigurationXml" Type="System.String">
   <DefaultValue>

Element characteristics

Characteristic Description
Data type and length String
Default value None
Cardinality 0-1: Optional element that can occur once and only once

Element relationships

Relationship Element
Parent elements DefaultValue, Plugins
Child elements Dependencies, NameSpace, PluginData, SourceType, URL

Attributes

Attribute Description
Name

(Required string.) Unique name of the plug-in.
Assembly (Required string) Name of the assembly (include the version, culture, and a PublicKeyToken if needed).
ClassName (Required string) Name of the class that contains the member or members of interest.

Remarks

The Plugin element extends a pipeline application instance by defining a mechanism to plug custom code into a service manifest. The following table shows the ways you can declare this element within a setting.

Setting Description of use Parent element
FeedConfigurationXML Add a transform library for use with derived columns. Declare within the DataParserOutput element of a data-parser service manifest only. Plugins
PluginModelConfigurationXML Add a custom assembly to the service manifest of a or application. DefaultValue
ShredderPluginConfigurationXML Add a custom assembly to the service manifest of a . DefaultValue

For more information about plug-ins, including examples, see Extend Pipeline applications.
