DefaultNack element

 

Declares a negative acknowledgement bytes.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <Feeds>
             <Feed Name="String">
                <ReceiveAdapter> Name="String"
                    <DefaultNack>...</DefaultNack>
                </ReceiveAdapter>
             </Feed>	
          </Feeds>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length String
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements ReceiveAdapter
Child elements None

Remarks

The DefaultNack element specifies the acknowledgement bytes in case the NackCapture element fails to match the message and is not able to create the NACK messages. If the NackCapture element fails to match the incoming message, and the message verification is unsuccessful, the adapter returns the default byte. Use a comma to separate different bytes settings.

You can use the DefaultNack element to configure an instance of the when the message transport format is MLLP. Before applying this element tag, set the Name attribute value of the ReceiveAdapter element to "Microsoft.Amalga.Pipeline.AcquisitionAdapters.MLLPServer". Do not declare this element in the manifests of other pipeline applications.

Example

<defaultnack>0x0B,0x06,0x1C,0x0D</defaultnack>
