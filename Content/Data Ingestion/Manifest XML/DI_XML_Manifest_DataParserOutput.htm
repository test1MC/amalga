DataParserOutput element

 

Declares data-parser output tables and columns.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <DataParserOutput>
               <Segment />
            <DataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length Array
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DataParserConfiguration
Child elements Segment

Remarks

The DataParserOutput element is mandatory only if you need to transform shredded message data, perform a lookup, or derive a new column that was not automatically generated during shredding. If you use shredded input as the source type of an operation, then you must first declare the segments and elements in the MessageShredderInput element section of your data-parser service manifest. Do not declare this element in the manifests of other pipeline applications.

In the data-parser output section, you can only derived one segment from a source segment and the Name attribute of segment must be the same as the SourceName attribute.

Correct usage

<DataParserOutput>
      <Segment Name="PID " SourceName="PID">
</DataParserOutput>

Incorrect usage

<DataParserOutput>
      <Segment Name="PID " SourceName="PID">
      <Segment Name="PID2" SourceName="PID">
</DataParserOutput>
