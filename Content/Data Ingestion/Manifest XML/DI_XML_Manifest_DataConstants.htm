DataConstants element

 

Declares a set of global constants for use in data-parser service manifests.

Syntax

[ <Setting Name="SettingNameXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <DataConstants>
               <Element>...</Element>
            </DataConstants>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length Array
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DataParserConfiguration
Child elements Element

Attributes

Attribute Description
Name

(Required string.) Name of the constant.
Type (Required string) Data type of the constant.
Value (Required string) Value of the constant.

Remarks

The DataConstants element defines a set of constant values that you can use as the source type in data output declarations. After these have been defined in a data-parser service manifest, you can use the named constant repeatedly throughout the manifest. Do not declare this element in the manifests of other pipeline applications.

Example

First the Pipe constant is defined.

<DataConstants>
    <Element Name="Pipe" Type="System.String" Value="|" />
</DataConstants>

Next, the Pipe constant is specified as the SourceType and SourceName attributes of the first parameter of the following transform.

<Element Name="Diagnosis_DG1_ContextKey" Type="System.String" MaxLength="200" ContextKey="true">
    <ConceptId>f/xuQgEy71iABYBuCrJmYw</ConceptId>
    <Transform>
        <Action>Concatenate</Action>
        <InputParameters>
            <Parameter Name="Pipe" SourceType="Constant" SourceName="Pipe" />
            <Parameter Name="Visit" SourceType="ShreddedInput" SourceTable="PV1" SourceName="PV1_19" />
            <Parameter Name="SetId" SourceType="DerivedField" SourceTable="DG1" SourceName="SetId" />
        </InputParameters>
    </Transform>
</Element>
