Lookups.usf_Terminology_ConvertTextToCode function

Returns the first matching HLI concept GUID for the specified text (terminology meaning).

Syntax

[<Lookup>			
   <Function>Lookups.usf_Terminology_ConvertTextToCode</Function>
   <InputParameters>
      <Parameter>Name="selectedTerminologyId"...</Parameter>
      <Parameter>Name="codeTypeId"...</Parameter>
      <Parameter>Name="incomingConceptMeaningText"...</Parameter>
      <Parameter>Name="prefixSearch"...</Parameter>
   </InputParameters>
</Lookup>]

Parameters

Parameter Data type Description
selectedTerminologyId

System.String

 

HLI-determined GUID for the text string (terminology meaning) specified. For example, the string for ICD-9 is ABjfrwDwaR0MBAI82I5XRA.

Use the following query to display the complete list of terminology GUIDs:

SELECT  
      [ContentSetGuid]
      ,[ContentSetSearchText]
FROM [AmalgaSourceData].[Terminology].[ContentSet]
codeTypeId System.String

A terminology code-type GUID. For example, the GUID for the ICD-9 CM code is AAE/rADjg0+O7oAH2I5XVw.

Use the following query to display the complete list of terminology code-type GUIDs:

SELECT distinct 
      [FacetTypeGuid]
      ,[FacetTypeSearchText]
FROM [ClinicalData].[Terminology].[ConceptFacet]
incomingConceptMeaningText System.String

Text (terminology meaning) from the incoming feed.
prefixSearch Boolean Whether to match by the complete meaning string or to treat the string as a prefix.

Return value

VARCHAR(400)

Remarks

For a list of valid attributes to use with this function, see the Parameter element. Apply this terminology function to any segment element (or column) that you declare in the data-parser service manifest. Do not declare this element in the manifests of other pipeline applications.
