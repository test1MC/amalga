EnterpriseId element

 

Declares a collection element to configure identifiers.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <DataParserConfiguration>
              <EnterpriseId>
                <EnterpriseIdAction />
             </EnterpriseId> 
          </DataParserConfiguration>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length Array
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DataParserConfiguration
Child elements EnterpriseIdAction

Remarks

The EnterpriseId element is one of the root elements for FeedConfigurationXML setting. Configure at least one feed for every source identifier that you need to match using the . In addition, any feed that requests explicit merge and move operations must engage the Correlation Engine. Typically, that includes all ADT feeds.

Declare the EnterpriseId element in the service manifest of the . Do not declare this element in the manifests of other pipeline applications.

Example

The following example maps two source identifiers (patient-record and encounter level).

    <EnterpriseId>
        <EnterpriseIdAction Action="StandardMapping" AnchorTable="PID">
            <InputIdentifiers>
                <PatientRecordIdentifier Name="Patient">
                    <Id Name="Id" Type="System.String" SourceType="ShreddedInput" SourceTable="PID" SourceName="MRN" />
                    <Domain Name="Domain" Type="System.String" SourceType="ShreddedInput" SourceTable="PID" SourceName="PID_3_5" />
                </PatientRecordIdentifier>
                <EncounterIdentifier Name="Encounter">
                    <Id Name="Id" Type="System.String" SourceType="ShreddedInput" SourceTable="PID" SourceName="PID_18_1" />
                    <Domain Name="Domain" Type="System.String" SourceType="Constant" SourceName="Domain" />
                </EncounterIdentifier>
            </InputIdentifiers>
        </EnterpriseIdAction>
    </EnterpriseId>
