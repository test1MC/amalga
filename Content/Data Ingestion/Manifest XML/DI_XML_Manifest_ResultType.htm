ResultType element

 

Declares how to display a resulting collapsed view.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <DataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <ResultType>...</ResultType>
                  </CollapseConfiguration>
               </Segment>
            </DataParserOutput>
					
            <ExtensibleDataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <ResultType>...</ResultType>
                  </CollapseConfiguration>
               </Segment>
            </ExtensibleDataParserOutput>
					
            <ExpandableDataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <ResultType>...</ResultType>
                 </CollapseConfiguration>
              </Segment>
            </ExpandableDataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value Latest
Cardinality 0-1: Optional element that can occur once and only once

Element relationships

Relationship Element
Parent elements CollapseConfiguration
Child elements None

Remarks

The ResultType element displays collapse-view result rows by showing either the earliest results only or the latest results only. To return the earliest results, insert Earliest as the value to the element.

Declare the result-type only after you declare a ContextKey attribute of the Element element and a matching PrimaryKeyColumn element in the data-parser service manifest. Do not declare this element in the manifests of other pipeline applications.

Example

<ResultType>Earliest</ResultType>
