DataParserConfiguration element

 

Declares a collection element in a data-parser service manifest.

Syntax

[ <Setting Name="SettingNameXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <MessageShredderInput />
            <DataParserOutput />
            <ExtensibleDataParserOutput />
            <ExpandableDataParserOutput />
            <DataConstants />
            <Plugins />
            <EnterpriseId />
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DefaultValue
Child elements DataConstants, DataParserOutput, EnterpriseId, ExpandableDataParserOutput, ExtensibleDataParserOutput, MessageShredderInput, Plugins

Remarks

The DataParserConfiguration element is mandatory if you need to clean up shredded message data, derive new columns, or expand existing segments. Each child element represents a distinct section within your data-parser service manifest. Do not declare this element in the manifests of other pipeline applications.

If an operation requires shredded input, you must declare each segment and element to be used in the MessageShredderInput element section.
