Feed element

 

Declares an adapter by which a instance acquires data from a source system.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <Feeds>
             <Feed Name="String">
                   <ReceiveAdapter>...</ReceiveAdapter>
                   <SendAdapter>...</SendAdapter>
                   <TempStoragedAdapter>...</TempStoragedAdapter>
                   <MsgType>...</MsgType>
                   <Verifier>...</Verifier>
                   <KeepOnTemporaryStorage>...</KeepOnTemporaryStorage> 
             </Feed>	
          </Feeds>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 1-1: Required element that can occur once and only once.

Element relationships

Relationship Element
Parent elements Feeds
Child elements KeepOnTemporaryStorage, MsgType, ReceiveAdapter, SendAdapter, TempStoragedAdapter, Verifier

Attributes

Attribute Description
Name

(Required string.) The unique name of the feed.

Remarks

The Feed element specifies the service configuration of one instance. Each message source must have a unique service manifest that declares this element.

Constraints

  • Use the Feed element to configure an instance of the when the message transport format is file or MLLP. The transport format determines the child elements that you declare in a service manifest.
  • KeepOnTemporaryStorage, TempStoragedAdapter, and Verifier child elements only apply to MLLP transport format.
