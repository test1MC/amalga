Segment element

 

Declares a message segment.

Syntax

Use within the service manifest only.

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <MessageShredderConfiguration>
            <Segments>
               <Segment>...</Segment>					
            </Segments>
         </MessageShredderConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Use within the service manifest only.

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <MessageShredderInput>
               <Segment>					
                  <Elements />
               </Segment>
         </MessageShredderInput>				

         <DataParserOutput>
            <Segment>					
               <Elements />
            </Segment>
         </DataParserOutput>
					
         <ExtensibleDataParserOutput>
            <Segment>					
               <Elements />
            </Segment>
         </ExtensibleDataParserOutput>
					
         <ExpandableDataParserOutput>
            <Segment>					
               <Elements />
            </Segment>
         </ExpandableDataParserOutput>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-n: Optional element that can occur one or more times.

Element relationships

Relationship Element
Parent elements Segments, DataParserOutput, ExpandableDataParserOutput ExtensibleDataParserOutput
Child elements Elements

Attributes

Attribute Description
Name

(Required string) Segment name used to generate a table.
SourceName (Required string) Name of segment from which the data for the table derives.
ParentName (Optional string) Parent segment name or segment table.
Description (Optional string) Description of the segment.

Remarks

You can declare the Segment element in the service manifest of both and applications. Constraints on this element vary, depending on the hosting manifest and the location within the service manifest. Do not declare this element in the manifests of other pipeline applications.

service manifest

Segment declaration is optional within the MessageShredderConfiguration element unless you need to specify a parent segment for one or more segments in a message. Always declare all the segments within a parent Segments element. Do not declare child elements for segments within a shredder service manifest.

service manifest

The Segment element has two distinct purposes within a parser service manifest. The following table describes each purpose.

Usage Related elements
Identify a segment that requires further activity, such as a transform action or lookup function. Inclusion is mandatory. MessageShredderInput
Apply a transform action or lookup function. DataParserOutput, ExpandableDataParserOutput, or ExtensibleDataParserOutput
