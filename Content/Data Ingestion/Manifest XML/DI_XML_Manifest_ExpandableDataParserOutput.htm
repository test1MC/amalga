ExpandableDataParserOutput element

 

Declares extra rows for expandable segments.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <ExpandableDataParserOutput>
               <Segment />
            <ExpandableDataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length Array
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DataParserConfiguration
Child elements Segment

Remarks

The ExpandableDataParserOutput element specifies the segment elements in the data-parser service manifest on which row expansion operates. Row expansion moves multiple values located in one column to a separate, related table in which the values are distributed in separate rows. Do not declare this element in the manifests of other pipeline applications.

Unlike regular data-parser output, expanded data-parser output can have multiple segments that derive from the same shredded table, which you specify with the SourceName attribute. For example, all three named segments in the following configuration expand from the same PID source segment. 

<ExpandableDataParserOutput>
   <Segment Name="PID_PatientIdentifier_1" SourceName="PID">
   <Segment Name="PID_PatientIdentifier_2" SourceName="PID">
   <Segment Name="PID_OtherInfo _" SourceName="PID">
</ExpandableDataParserOutput>

Example

The following example shows an expanded PID_PatientIdentifer segment (from the PID segment source) with several elements. In the resulting table, the PatientIdentifier_ContextKey column is the primary key for the PID_PatientIdentifier table.

    <ExpandableDataParserOutput>
        <Segment Name="PID_PatientIdentifier" SourceName="PID">
            <Elements>
                <Element Name="PatientIdentifier_ContextKey" Type="System.String" MaxLength="200" ContextKey="true">
                    <ConceptId>f/xuQgEy71iABYA4CrJmYw</ConceptId>
                </Element>
                <Element Name="Patient_PID_ContextKey" Type="System.String" MaxLength="200" ReferenceKey="true">
                    <ConceptId>f/xuQgEy71iABYADCrJmYw</ConceptId>
                </Element>
                <Element Name="IDNumber" Type="System.String" MaxLength="200">
                    <ConceptId>1uJSHAEy5J0eLAU3CsgBAA</ConceptId>
                </Element>
                <Element Name="IDType" Type="System.String" MaxLength="200">
                    <ConceptId>1uJSHAEy5J0eLAVTCsgBAA</ConceptId>
                </Element>
              </Elements>
        </Segment>
    </ExpandableDataParserOutput>
