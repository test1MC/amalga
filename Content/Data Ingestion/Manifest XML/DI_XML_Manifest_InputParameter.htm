InputParameters element

 

Declares a set of parameters for a transform action, lookup function, or condition action.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <DataParserConfiguration>
					
             <Transform>
                <Action>...</Action>
                <InputParameters>
                   <Parameter>...</Parameter>
                </InputParameters>
             </Transform>
					
             <Lookup>
                <Function>...</Function>
                <InputParameters>
                   <Parameter>...</Parameter>
                </InputParameters>
             </Lookup>
					
             <EnterpriseIdAction> Action=String AnchorTable=String >
                <InputParameters>
                   <Parameter>...</Parameter>
                </InputParameters>
					
                <Condition>
                   <Action>...</Action>
                   <InputParameters>
                      <Parameter>...</Parameter>
                   </InputParameters>
                </Condition>>					
             </EnterpriseIdAction>

          </DataParserConfiguration>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once

Element relationships

Relationship Element
Parent elements Condition, EnterpriseIdAction, Lookup, Transform
Child elements Parameter

Remarks

The InputParameters element provides one or more parameters as input to a transform action, a lookup function, or a condition action. The number and order of parameters depends on the requirements of the specified action or function. For example, because the ToTitleCase transform has only one parameter, the corresponding InputParameters element will have only one child Parameter element.

Declare the InputParameters element in the service manifest of the . Do not declare this element in the manifests of other pipeline applications.

Example

<Element Name="FamilyName" Type="System.String" MaxLength="100">
    <ConceptId>1uJSHAEy5J0eLAWfCsgBAA</ConceptId>
    <Transform>
        <Action>ToTitleCase</Action>
        <InputParameters>
            <Parameter SourceType="ShreddedInput" SourceTable="PID" SourceName="PID_5_1" />
        </InputParameters>
    </Transform>
</Element>
