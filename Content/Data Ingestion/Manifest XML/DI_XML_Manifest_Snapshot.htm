Snapshot element

 

Declares that the message sending the segment uses HL7 snapshot mode.

Syntax

[ <Setting Name="FeedConfigurationNameXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <DataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <ResultRowsFilter>...</ResultRowsFilter>
                  </CollapseConfiguration>
               </Segment>
            </DataParserOutput>
					
            <ExtensibleDataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <ResultRowsFilter>...</ResultRowsFilter>
                  </CollapseConfiguration>
               </Segment>
            </ExtensibleDataParserOutput>
					
            <ExpandableDataParserOutput>
               <Segment>
                  <CollapseConfiguration>
                     <ResultRowsFilter>...</ResultRowsFilter>
                 </CollapseConfiguration>
              </Segment>
            </ExpandableDataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length Boolean
Default value False
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements CollapseConfiguration
Child elements None

Remarks

Declare the Snapshot element in the data-parser service manifest when the sending message uses HL7 snapshot mode. Do not declare this element in the manifests of other pipeline applications.

Example

<Snapshot>true</Snapshot>
