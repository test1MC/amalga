TerminologyName element

 

Declares the terminology database that provides semantic tags for shredded and derived columns.

Syntax

[ <Setting Name="FeedInfoXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <FeedInfo> 
             <TerminologyName>...</TerminologyName>
          </FeedInfo>
       ]]> 
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length String
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements FeedInfo
Child elements None

Remarks

The terminology database provides the terminology concept codes used to tag (or label) shredded and derived columns stored in the database. Concept codes enable entities to bind to storage schema.

Always declare the TerminologyName element in the service manifest of the and applications. Do not declare this element in the manifests of other pipeline applications.

The name must match the terminology database name defined during installation.

Example

<TerminologyName>Microsoft Beta HL7 V2.X Code</TerminologyName>
