Overwritable element

 

Declares whether the default error action can be overwritten in the service manifest.

Syntax

[ <Setting Name="ErrorHandlingConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <ErrorHandlingConfiguration>
            <ErrorHandlingPolicies>
               <ErrorHandlingPolicy>
                  <Overwritable>...</Overwritable>	
               </ErrorHandlingPolicy>
            </ErrorHandlingPolicies>
         </ErrorHandlingConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length Boolean
Default value False
Cardinality 1-1: Required element that can occur once and only once.

0-1: Optional element that can occur once and only once.

1-n: Required element that can occur one or more times.

0-n: Optional element that can occur one or more times.

1-2: Required element that can occur at least once and no more than twice.

0-2: Optional element that can occur at least once and no more than twice.

Element relationships

Relationship Element
Parent elements ErrorHandlingPolicy
Child elements None

Remarks

The Overwritable element restricts or allows a change to a user-defined error action. For a list of predefined error codes, see Predefined error codes for the data pipeline.

Constraints

Declare the Overwritable element element only within a service manifest of the , , , and applications.

Example

<Overwritable>true</Overwritable>
