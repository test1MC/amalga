Transform element

 

Declares a transform action and the associated parameters for the specified segment element.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <DataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <Transform>
                           <Action>...</Action>
                           <InputParameters />						
                        </Transform>
                     </Element>
                  </Elements>
               </Segment>
            </DataParserOutput>
					
            <ExtensibleDataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <Transform>
                           <Action>...</Action>
                           <InputParameters />		
                        </Transform>
                     </Element>
                  </Elements>
               </Segment>
            </ExtensibleDataParserOutput>
					
            <ExpandableDataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <Transform>
                           <Action>...</Action>
                           <InputParameters />								
                        </Transform>
                     </Element>
                  </Elements>
              </Segment>
            </ExpandableDataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once

Element relationships

Relationship Element
Parent elements Element
Child elements Action, InputParameters

Remarks

The Transform element is mandatory in the data-parser service manifest to transform the format of data from the message. Do not declare this element in the manifests of other pipeline applications.

Example

The following example generates a context key by concatenating a shredded segment element and a derived column.

<Element Name="Diagnosis_DG1_ContextKey" Type="System.String" MaxLength="200" ContextKey="true">
    <ConceptId>f/xuQgEy71iABYBuCrJmYw</ConceptId>
    <Transform>
        <Action>Concatenate</Action>
        <InputParameters>
            <Parameter Name="Pipe" SourceType="Constant" SourceName="Pipe" />
            <Parameter Name="Visit" SourceType="ShreddedInput" SourceTable="PV1" SourceName="PV1_19" />
            <Parameter Name="SetId" SourceType="DerivedField" SourceTable="DG1" SourceName="SetId" />
        </InputParameters>
    </Transform>
</Element>
