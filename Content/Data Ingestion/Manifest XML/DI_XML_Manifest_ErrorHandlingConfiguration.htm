ErrorHandlingConfiguration element

 

Declares a collection element to identify error-handling policies.

Syntax

[ <Setting Name="ErrorHandlingConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <ErrorHandlingConfiguration>
             <ErrorHandlingPolicies />
          </ErrorHandlingConfiguration>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DefaultValue
Child elements ErrorHandlingPolicies

Remarks

The ErrorHandlingConfiguration element is a root element used to modify the behavior of predefined exceptions that can be overwritten. You can also define your own exception plug-in and declare its use in the service manifest of an application instance. For a list of predefined error codes, see Predefined error codes for the data pipeline.

Declare the ErrorHandlingConfiguration element only within a service manifest of the , , , and applications.

Example

The following example defines a new policy for the private exception type Microsoft.Amalga.Pipeline.Shredders.ExceptionA.

<ErrorHandlingConfiguration>
  <ErrorHandlingPolicies>    
    <ErrorHandlingPolicy>
      <ExceptionType>System.Data.SqlClient.SqlException</ExceptionType>
      <ErrorCodes>
        <ErrorCode>7</ErrorCode>
        <ErrorCode>9</ErrorCode>
      </ErrorCodes>     
      <Action>Stop</Action>
      <LogSetting>
        <EventId>33001</EventId>
        <ErrorTitle>DbServerNotResponding</ErrorTitle>
        <ErrorDescription>The component has attempted to connect to the Database Server; however, the connection has timed out.</ErrorDescription>
      </LogSetting>
    </ErrorHandlingPolicy>
  </ErrorHandlingPolicies>    
</ErrorHandlingConfiguration>
