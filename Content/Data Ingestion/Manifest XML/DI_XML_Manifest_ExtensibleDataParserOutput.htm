ExtensibleDataParserOutput element

 

Declares the segment elements to be modified by a custom plug-in assembly that runs within the instance at runtime.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <ExtensibleDataParserOutput>
               <Segment />
            <ExtensibleDataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length Array
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DataParserConfiguration
Child elements Segment

Remarks

The ExtensibleDataParserOutput enables you to optionally introduce custom transform or configuration behavior to your data-parser service manifest. Do not declare this element in the manifests of other pipeline applications.

For instructions on how to extend a data parser, see Custom assembly parser extension.
