Directory element

 

Declares a directory to be monitored for file acquisition or used as temporary storage when the database becomes unavailable.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <Feeds>
             <Feed Name="String">
                <ReceiveAdapter> Name="String"
                     <Directory>...</Directory> 
                </ReceiveAdapter>
 
                <TempStorageAdapter; Name="String"
                    <Directory>...</Directory>
                </TempStorageAdapter;
             </Feed>	
          </Feeds>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length String
Default value None
Cardinality

0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements ReceiveAdapter, TempStorageAdapter
Child elements None

Remarks

The Directory element is mandatory for the following two distinct purposes:

  • When nested within the ReceiveAdapter element, the Directory element value provides a destination directory for incoming message files. An application instance configured to monitor the directory folder detects and acquires new files as they arrive in the folder.
  • When nested within the TempStorageAdapter element, the Directory element provides a temporary directory folder that stores MLLP-based messages when the database is not present.

The application instance requires access permission to the directory folder.

Use the Directory element to configure an instance of the when the message transport format is file or MLLP. Before applying this element tag, set the Name attribute value of the ReceiveAdapter element or TempStorageAdapter element, respectively, to the file adapter or MLLP adapter. Do not declare this element in the manifests of other pipeline applications.

Examples

File adapter example:

<receiveadapter name="Microsoft.Amalga.Pipeline.AcquisitionAdapters.FileAdapter">
   <directory>L:\Temp\FileReceive\LAB</directory>
   <searchpattern>*.*</searchpattern>
</receiveadapter>

MLLP adapter example:

<tempstorageadapter name="Microsoft.Amalga.Pipeline.AcquisitionAdapters.MLLPServer">
   <directory>C:\Temp\SecondaryStorage</directory>
</tempstorageadapter>
