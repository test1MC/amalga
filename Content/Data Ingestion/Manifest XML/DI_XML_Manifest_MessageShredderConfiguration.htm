MessageShredderConfiguration element

 

Declares a collection element for message segments.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <MessageShredderConfiguration>
            <Segments />
         </MessageShredderConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DefaultValue
Child elements Segment

Remarks

The MessageShredderConfiguration element is mandatory as the root element of the FeedConfigurationXML setting for all service manifests. Do not declare this element in the manifests of other pipeline applications.

Message shredding is automatic and the resulting shredded table structure is predictable. The shredder creates one table for each message segment and one table for a flat file. If the segment has additional elements, the elements are shredded into table columns.

Segment declaration is optional unless you intend to specify a parent segment for one or more segments in the message. A segment (or segment table) can have only one parent.

Example

The following example establishes segment PID as parent to segment PV1. Always declare the parent segment first.

<MessageShredderConfiguration>
    <Segments >
       <Segment Name="PID" SourceName="PID" Description="Personal Identification">
       <Segment Name="PV1" SourceName="PV1" ParentName="PID" Description="Patient Visit">
   <Segments >
</MessageShredderConfiguration>
