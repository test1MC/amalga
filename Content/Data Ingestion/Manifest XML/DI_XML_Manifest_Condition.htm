Condition element

 

Declares a condition under which the specified transform action is applied.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <DataParserConfiguration>
             <EnterpriseId>
                <EnterpriseIdAction Action="String" ArchorTable="String">
                   <Condition>
                      <Action>...</Action>
                      <InputParameters />
                   </Condition>
                </EnterpriseIdAction>	
             </EnterpriseId>
          </DataParserConfiguration>
       ]]>
    </DefaultValue>
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements EnterpriseIdAction
Child elements Action, InputParameters

Remarks

The Condition element performs a specified action with the input parameters that you supply. If the condition is met, the parameter values are included in the root action. Specify only one action for each condition. If you intend to perform multiple actions, declare multiple conditions.

Declare the Condition element in the service manifest of the . Do not declare this element in the manifests of other pipeline applications.

Example

The following example determines if values for the EventLocation and EventCode parameters are equal.

<Condition>
        <Action>Equals</Action>
        <InputParameters>
            <Parameter Name="EventLocation" Label="MSH_9_2" ReferenceTable="MSH" Source="ShreddedInput"/>
            <Parameter Name="EventCode" Label="A43" Source="Constant"/>
        </InputParameters>
 </Condition>
