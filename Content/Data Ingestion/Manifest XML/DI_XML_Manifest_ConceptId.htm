ConceptId element

 

Declares a concept identifier for a derived column to enable assisted binding.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
   <DefaultValue>
      <![CDATA[<?xml version="1.0" encoding="utf-8"?>
         <DataParserConfiguration>
            <DataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <ConceptId>...</ConceptId>
                     </Element>
                  </Elements>
               </Segment>
            </DataParserOutput>
					
            <ExtensibleDataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <ConceptId>...</ConceptId>
                     </Element>
                  </Elements>
               </Segment>
            </ExtensibleDataParserOutput>
					
            <ExpandableDataParserOutput>
               <Segment>
                  <Elements>
                     <Element>
                        <ConceptId>...</ConceptId>
                     </Element>
                  </Elements>
              </Segment>
            </ExpandableDataParserOutput>
         </DataParserConfiguration>
      ]]>
   </DefaultValue>
</Setting> ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements Element
Child elements None

Remarks

A concept identifier is the globally unique identifier (GUID) of the semantic tag that describes relational-schema components, such as tables and entities. Each semantic tag is a is a fully qualified string representation of a concept in a clinical terminology included with the Terminology platform application. Use the following query to display the complete list of terminology GUIDs from the HLI database:

SELECT
   [ContentSetGuid]
   ,[ContentSetSearchText]
FROM [AmalgaSourceData].[Terminology].[ContentSet]

The ConceptId element is mandatory when you want a derived column to be displayed during assisted binding. Unlike derived columns, shredded columns receive concept identifiers automatically when the message type supports semantic tags. You declare this element once for each derived column specified in the data-parser service manifest. Do not declare this element in the manifests of other pipeline applications.

The following figure shows both shredded and derived columns in the PID table during ingestion. In this example, Derived Col1 and Derived Col2 are assigned explicit concepts that connect the columns to entity properties defined in the entity registry. When you map multiple columns to the same concept identifier, each tagged column is visible during the binding process.

Example

<Element Name="FamilyName" Type="System.String" MaxLength="100">
    <ConceptId>1uJSHAEy5J0eLAWfCsgBAA</ConceptId>
    <Transform>
        <Action>ToTitleCase</Action>
        <InputParameters>
            <Parameter SourceType="ShreddedInput" SourceTable="PID" SourceName="PID_5_1" />
        </InputParameters>
    </Transform>
</Element>
