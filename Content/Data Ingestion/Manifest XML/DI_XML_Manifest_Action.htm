Action element

 

Declares a transform action, a condition action to evaluate, or an action in response to an error.

Syntax

FeedConfigurationXml (See the Transform and Condition elements for complete syntax.)

[ <Transform>>
   <Action>...</Action>
</Transform>
                   
<Condition>
   <Action>...</Action>
</Condition> ]

ErrorHandlingConfigurationXml (See the ErrorHandlingPolicy element for complete syntax.)

[ <ErrorHandlingPolicy>
   <Action>...</Action>
</ErrorHandlingPolicy> ]

Element characteristics

Characteristic Description
Data type and length String
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements Condition, ErrorHandlingPolicy, Transform
Child elements None

Remarks

The Action element is mandatory to define a transform or condition action. It is also mandatory to handle errors.

Transform and Condition actions

  • Declare a transform (method) and specify a predefined transform action. Use the following list to see the parameters for each method:
    • ApplyXPath
    • Base64ToBinary
    • Concatenate
    • ConcatenateChildren
    • ConcatenateStringList
    • Copy
    • GetCurrentUTCTime
    • HexToBinary
    • StringEqualsIgnoreCase
    • ToDateTime
    • ToTitleCase
    • Trim
  • Declare a condition that uses Common Language Specification (CLS) compliant actions.

Constraints for transform and condition actions are:

  • Declare a transform or condition action in the service manifest of the only. Do not declare transform or condition actions the manifests of other pipeline applications.
  • Apply one action to a transform or condition. If you intend to perform multiple actions, declare multiple transforms or conditions.

Example for a transform action:

<Transform>
   <Action>ToTitleCase</Action>
   <InputParameters>
      <Parameter Name="LowCaseString" SourceType="ShreddedInput" SourceTable="PID" SourceName="PID_5_1" />
   </InputParameters>
</Transform>

Example for a condition action:

<Condition>
   <Action>StringEqualsIgnoreCase</Action>
   <InputParameters>
     <Parameter Name="left" Type="System.String" SourceTable="MSH" SourceType="ShreddedInput" SourceName="MSH_9_2"/>
     <Parameter Name="right" Type="System.String" SourceType="Inline" Value="A40" />
   </InputParameters>
</Condition>

Error-handling actions

The Action element is mandatory if you want to handle errors in your manifest. Declare one of the following error-handling actions for a specified exception:

  • Stop: Stops the process entirely.
  • Continue: Ignores the error and continues processing the same message.
  • Retry: Retries the failed message.

Constraints for error-handling actions are:

  • Declare the Action element only within a service manifest of the , , , and applications. Do not declare this element in the feed-acquisition service manifest.
  • You must first specify the retry action if you want to define a retry interval, maximum retries, or a final action.

Example for an error-handling action:

<Action>Retry</Action>
