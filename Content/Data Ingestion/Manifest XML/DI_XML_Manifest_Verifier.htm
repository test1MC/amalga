Verifier element

 

Declares a regular expression to verify the pattern of the received message.

Syntax

[ <Setting Name="FeedConfigurationXml" Type="System.String">
    <DefaultValue>
       <![CDATA[<?xml version="1.0" encoding="utf-8"?>
          <Feeds>
             <Feed Name="String">
                <Verifier>...</Verifier>
             </Feed>	
          </Feeds>
       ]]>
    </DefaultValue> 
 </Setting> ]

Element characteristics

Characteristic Description
Data type and length None.
Default value None.
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements Feed.
Child elements None.

Remarks

The Verifier element is mandatory in the service manifest of the when the message transport format is MLLP. Do not declare this element in the manifests of other pipeline applications.

If the specified regular expression successfully matches the message content, and you include the AckCapture and AckFormular element values, then the subsystem creates an appropriate return acknowledgement message.

Example

<verifier>^MSH\|([^|\r\n]*\|){13,}.*\r\n</verifier>
