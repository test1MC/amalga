Resolve manifests

 

This topic describes how to resolve application and service manifests.

For an application profile, see Message Shredder application.

Application manifest

To resolve an application manifest

  1. Copy, rename, and edit an example file.
  2. Replace the deployment-manifest instance name with a new name that uniquely identifies the application instance. For help, see DeploymentManifest element.
  3. Save the file.

Service manifest

To resolve a service manifest

  1. Copy, rename, and edit an example file.
  2. Replace the deployment-manifest instance name with a new name that uniquely identifies the service instance. For help, see DeploymentManifest element.

  3. Replace the logical host name with the local name. For help, see DoorsWorkerManifest element.

    Get the local logical-host name from the Amalga Management Console.

  4. Set the idle connection timeout if the default value is insufficient. For help, see IdleConnectionTimeout setting.
  5. Set the connection timeout (in seconds) if the default value is insufficient. For help, see ConnectionTimeoutSeconds setting.

  6. In the FeedConfigurationXML setting block, configure the following element values for either a file format or an MLLP format feed:

    For file format

    • Feed element (Specify a feed name for the Name attribute.)
    • ReceiveAdapter element (Specify the following child elements: Directory element and SearchPattern element.)
    • MsgType element (Specify a message type value; use the same value for the Type element in the related shredder and parser service manifests.)

    For MLLP format

    • Feed element (Specify a feed name for the Name attribute.)
    • ReceiveAdapter element (Specify child element: Port element.)
    • TempStorageAdapter element (Specify the child element: Directory element.)
    • MsgType element (Specify a message type value; use the same value for the Type element in the related shredder and parser service manifests.)

  1. (Optional) Declare a custom plug-in application. For help, see Add an Acquisition plug-in.
  2. Save the file.
