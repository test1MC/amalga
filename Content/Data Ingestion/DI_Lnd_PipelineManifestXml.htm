Pipeline manifest XML

 

This topic introduces the settings, elements, functions, and transforms available to pipeline applications.

Use the items in this section to declare the runtime metadata required to run an application or service instance in the data pipeline.

Configuration settings

Pipeline settings accept either simple values or complex configurations. A setting that accepts a complex configuration also supports a hierarchy of child elements and the setting name is appended with XML. You can declare zero or more settings in the service manifest. The is unusual, because it requires a setting in the application manifest. For instructions, see Enable backfill parsing.

XML elements

Each complex setting contains a hierarchy of Extensible Markup Language (XML) elements that combine to produce a valid configuration for each pipeline application instance. To determine whether to include an element, use one or both of the following approaches to locate an element topic:

  • Open a setting hierarchy (see the previous table).
  • Display an alphabetic list of elements (see Pipeline XML elements).

Lookup functions

Lookup functions are restricted to data-parser service manifests. provides three terminology functions that help you convert terminology codes into strings, strings into codes, or convert one code to another code. For an overview, see Terminology lookup functions.

MessageHelper transform library

The MessageHelpers class provides a library of data transformations that can be declared as transform actions in data-parser service manifests to improve the usability of ingested messages. Each method of this class represents a transform action.

For help to declare and use a transform, see Transform library class extension.
