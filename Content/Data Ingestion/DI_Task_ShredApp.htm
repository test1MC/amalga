Resolve manifests

 

This topic describes how to resolve application and service manifests.

For an application profile, see Message Shredder application.

Application manifest

To resolve an application manifest

  1. Copy, rename, and edit an example file.
  2. Replace the deployment-manifest instance name with a new name that uniquely identifies the application instance. For help, see DeploymentManifest element.

  3. Replace the dependency instance name with the instance name of local Terminology application. 

    <Dependencies>
        <Dependency Name="TerminologyAppDependency" InstanceName="TerminologyApplication_Ins1" 
                    Type="Application" Id="c031033a-49be-4c8f-a6a8-cb95bddaac0a" MinVersion="1.0.0.0" />
</Dependencies>

    Get the local terminology instance name from the Amalga Management Console.

  4. Save the file.

Service manifest

To resolve a service manifest

  1. Copy, rename, and edit an example file.
  2. Replace the deployment-manifest instance name with a new name that uniquely identifies the service instance. For help, see DeploymentManifest element.

  3. Replace the logical host name with the local name. For help, see DoorsWorkerManifest element.

    Get the local logical-host name from the Amalga Management Console.

  4. Replace the Type element value with the value used in the related feed-acquisition service manifest to define the MsgType element. For example, if the feed's message-type value is ABC, then the shredder's feed-information type must be ABC. For more feed-related options, see FeedInfoXML setting.
  5. (Optional) Assign a parent segment. For help, see MessageShredderConfiguration element and Segments element.
  6. (Optional) Define comma-separated values (CSV) data characteristics, such as row and field delimiters. For help, see CSVConfiguration element.
  7. (Optional) Update the attributes of an existing error policy. For help, see ErrorHandlingConfiguration element.
  8. (Optional) Declare a custom plug-in application. For help, see Add a shredder plug-in.
  9. Save the file.
