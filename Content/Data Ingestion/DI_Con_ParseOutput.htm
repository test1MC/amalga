Parser output

 

The structure of parsed output depends on the input and the derivations performed during parsing. By default, the Data Parser subsystem creates one read-only table for each output segment, which includes a RowId column and an additional column for each derived field that you define in the parser. For example:

 

Column example: DataParser.ADT.PID_Derived (HL7)

RowID FirstName LastName FullName

Use the following structure to specify data parser output that includes concept identifiers and transforms.

   <DataParserOutput>
        <Segment>
            <Elements>
                <Element>
                    <ConceptId></ConceptId>
                    <Transform>
                        <Action></Action>
                        <InputParameters>
                            <Parameter/>
                        </InputParameters>
                    </Transform>
                  </Element>
            </Elements>            
        </Segment>
    </DataParserOutput>

Both input and output tables reside in the ClinicalData database. Use the Data Parser subsystem name to distinguish output tables from Message Shredder subsystem input tables and views. All parser output tables use the following format:

[database].[feedname].[Segment_Derived]
