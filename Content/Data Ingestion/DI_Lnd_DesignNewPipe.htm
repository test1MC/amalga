Design a new data pipe

 

 

A data pipe is the set of coordinated application instances that together represent a single feed in the Amalga data pipeline. In production, there should be one data pipe for each message type.

Getting started

Good data-pipe design begins with a deep understanding of each individual message type and of the entity or entities that will bind to the resulting storage schema. Entities and storage schema are independent constructs that, when bound , support the creation of virtual schemas. Ingested data is visible to hosted applications at design time or run time only after an operator has bound entity properties to the schema provisioned by each data pipe to hold message data. For a list of entities to investigate, see entity namespace.

Required applications

  • Feed Acquisition application
  • Data Parser application
  • Message Shredder application
