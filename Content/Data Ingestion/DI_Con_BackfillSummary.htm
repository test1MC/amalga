When to use this application

Deploy one instance of the Backfill Data-Parser application to parse some or all historic messages associated with a single feed that would otherwise produce blank records for a newly added derived column. When processing finishes, stop the application instance and remove it from production.

Extensibility

None.

The Backfill cannot be extended; however, all extensions that apply to the updated columns in the data parser must also appear in the backfill service manifest.

 Deployment

Deployment duration

Operates continuously in production until the message-reparse job is finished and the service logs error with an Event ID of 11001.

Special instructions

You can deploy a backfill instance to an idle server to reduce the impact of backfill parsing on forward parsing.

Remove the backfill instance when all of the rows are populated.

Remarks

When you derive a new column for an existing data parser and redeploy the updated parser service, forward paring begins with the first new message. As the following figure shows, forward parsing begins for derived “Column C” at row 4-C. All of the previous rows are blank.

Backfilling the new column (shown in the next figure) with past messages is optional. If you decide to populate previous rows, enable backfill parsing to run until the backfill completes, stop the service, and then remove the Backfill Data-Parser instance. For setup instructions, see Enable backfill parsing.

 Restrictions
  • The backfilled column will be visible to entity binding only when the backfill operation completes.
  • Only one backfill parser can run for a given data parser.
  • You must stop the backfill service manually.
  • If you must amend a deployed backfill parser with another derived column before current backfill parsing is complete, always do the following:
    1. Stop the backfill service.
    2. Remove the backfill application instance.
    3. Add the new column.
    4. Rename the backfill instance to ensure that the newest derived column will be fully populated.
  • When changing row expansion definition, the updated row expansion must be configured with a new row-expansion table name.
