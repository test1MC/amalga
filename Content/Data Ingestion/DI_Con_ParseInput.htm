Parser input

 

Parsers require shredded columns as input. Each parser instance receives a batch of messages as soon as message shredding is completed for the associated message type.

You must identify all of the shredded input segments and elements needed by the parser. This group is generally a small subset of the segments in a feed. If a column that you specify as input does not exist, the subsystem throws an exception and the parser stops.

Use the following structure to specify message shredder input to the parser:

<DataParserConfiguration>
    <MessageShredderInput>
        <Segment>
          <Elements>
              <Element/>
              <Element/>
          </Elements>
        </Segment>
     </MessageShredderInput>
 </DataParserConfiguration>

Segment elements within <MessageShredderInput> and <DataParserOutput> element tags do not have the same behavior. Specifically, you must not include transform methods in segments within the <MessageShredderInput> section. You can specify transform methods only in segments within the <DataParserOutput> section.
