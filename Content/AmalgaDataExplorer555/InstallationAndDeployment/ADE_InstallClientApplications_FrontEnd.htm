Complete the front end installation

The following steps complete the installation. You will modify the XML configuration files, upload the Patient Chart configuration files, and grant permissions.

To modify the XML configuration files for your installation

  1. Locate the following files in c:\DataExplorerInstallation (the location to which you copied the installation files) and open each of them in a text editor (ignore the number of instances until step 2):

    • allergies_logicalconfig.xml (1 instance)

    • clinicalsummary_logicalconfig.xml (6 instances)

    • labs_logicalconfig.xml (3 instances)

    • problemlist_logicalconfig.xml (1 instance)

    • vitalsigns_logicalconfig.xml (1 instance)

  2. Locate all occurrences of https URLs. The preceding list indicates how many https URLs are in each XML file.
  3. Update each URL to reflect the fully qualified domain name (FQDN) for your installation. Be sure to also update the port number in the URL for your existing installation.

    Search for 'https' and then modify the server name and domain to be the specific fully qualified domain name of your server.

To upload the configuration files for the Patient Chart

  1. Locate the following files in c:\DataExplorerInstallation\ and copy them to C:\Program Files\Common Files\Microsoft Shared\Web Server Extensions\14\TEMPLATE\FEATURES\ModuleLauncher_Feature1\:

    • allergies_logicalconfig.xml

    • allergies_presentationconfig.xml

    • clinicalsummary_logicalconfig.xml

    • clinicalsummary_presentationconfig.xml

    • labs_logicalconfig.xml

    • labs_presentationconfig.xml

    • problemlist_logicalconfig.xml

    • problemlist_presentationconfig.xml

    • vitalsigns_logicalconfig.xml

    • vitalsigns_presentationconfig.xml

  2. Locate the modulelauncher-dummyconfig.txt in c:\DataExplorerInstallation and copy to c:\Program Files\Common Files\Microsoft Shared\Web Server Extensions\14\TEMPLATE\FEATURES\ModuleLauncher_Feature1\ModuleLauncher\.

  3. Open a command prompt with administrator level permissions.

  4. Type the following command and press enter:

    sqlcmd /i c:\DataExplorerInstallation\Packages\SampleQuickRules.sql

To grant permission for a user to visit the

  1. Navigate to the site permissions. These can be found at https://fqdn:port/sites/amalga/_layouts/user.aspx, where fqdn is the fully qualified domain name for your installation.
  2. Click Grant Permissions.
  3. Click People Picker (found near the Users/Groups box) and perform the following:
    1. Select Amalga ADFS.
    2. Type the domain and user alias in the find input box. For example: mydomain\user1
    3. Click the search button in the find box.
    4. Select the appropriate result from the panel.
    5. Click Add.

    6. Click OK.

      The user will now be added to the Select User text box.

  4. Check Full Control.
  5. Click OK.

If you are prompted for the account to sign in with, use Amalga ADFS.
