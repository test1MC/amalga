Configure Clinical Summary

 

This topic describes the configuration options for a module for the Clinical Summary.

Presentation configuration

Element Children Mandatory? Comments
ColumnCount     Number of columns in the Clinical Summary; positive integer
Columns ColumnConfiguration Yes Configuration for the columns; Is there a maximum number allowed?
ColumnConfiguration Index, Width, Widgets Yes Configuration for the column; repeat for each column
Index   Yes Index of the column; zero-based integer
Width   Yes Percentage of the window devoted to the column
Widgets WidgetLayoutConfiguration   Configuration for all widgets

WidgetLayoutConfiguration

 Index, ID, Height, MinimumHeight, DisplayName   Configuration for each widget; repeat for each widget
Index   Yes Index of the widget in the column; zero-based integer
ID   Yes Widget ID; must match the ID in logical configuration
Height   Yes Widget border height
MinimumHeight   Yes Minimum height of the widget border; when resizing, this is the minimum height.
DisplayName   Yes Widget title. Use resource key in Microsoft.Health.PatientChart.ClinicalSummary.resx if globalization is required.

Logical configuration

Element Children Mandatory? Comments
WidgetIdentityConfiguration     Logical configuration of a single widget; repeat for all widgets
ID     Widget ID; must match the ID in presentation configuration
RelatedModuleID    

When clicking the widget title, the related module will open;

can be empty if there is no related module
WebPartName     Name of the webpart
ContentPageUri    

The Uri of the content page that contains the webpart;

relative path to the site root Is this used instead of the name? Is it only used for a Beijing webpart?
