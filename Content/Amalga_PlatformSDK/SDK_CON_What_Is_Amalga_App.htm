applications

An Amalga application is an application that uses the Amalga platform to provide functionality for building hospital applications, including security, data modeling, application configuration management, logging, auditing, and topology/discovery. Amalga applications consist of several components, starting with data and ending with user interfaces.

In addition to data from various hospital systems, an application may have its own data. For example, an application that reads lab reports may also allow users to write and save notes about each lab report. As described below, you can easily use Amalga tools to create a database for your application data only.

Virtual Schema

The Virtual Schema provides a layer of abstraction so that application developers don't need to code to specific databases, but rather can code to the Virtual Schema, which specifies which data entities the application can use. A Virtual Schema tool is available that allows you to specify which entities your application can uses, giving read-only access to data in any number of databases with hospital data. In addition, by specifying writable entities in the Virtual Schema, you can automatically create an application database with data that is accessible through the Virtual Schema tool.

The Amalga Fabric API

The Amalga Fabric API is a collection of .NET libraries that make it easier to create applications for hospital data. Some of the features provided by Amalga Fabric are:

  • Security
  • Role Management
  • Authorization Management
  • Connection to the Virtual Schema
  • Logging
  • Auditing
  • Topology and Discovery
  • Application Configuration Management

Amalga Hosted Services and Applications

Amalga Hosted Services and Applications have several advantages over their equivalent services and applications without Amalga. The Amalga platform provides features such as security and logging, as well as management through the Amalga Console.

Doors Communication Foundation Services

By using Visual Studio templates provided by the Amalga SDK, you can easily build WCF services that can access data through the Virtual Schema. These WCF services can then be used by Amalga hosted applications, or by any application so that end users can interact with your data. Like any other WCF service, your Amalga WCF service can take advantage of technologies such as clustering in order to achieve high performance.

Doors Worker Services

Again, Visual Studio templates provided by the Amalga SDK make it easy to build Doors Worker Services that run on your server. These services are good for running background tasks, such as sending out automated reports at regularly scheduled intervals.

In addition to just hosting, you can also build applications using the Amalga Application Framework. This allows you to easily create applications using Sharepoint that take advantage of several Amalga features, such as context passing so that controls can respond to user reactions on other controls, even on separate web pages.

Other Applications

You aren't restricted to creating your applications using Amalga technology, or even Microsoft technology. By calling WCF services using whatever technology you choose, you can still access your Virtual Schema in a secure fashion.
