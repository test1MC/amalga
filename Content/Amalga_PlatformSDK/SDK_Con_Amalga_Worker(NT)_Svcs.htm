Hosted s

This topic describes how to build your own Amalga Doors service.

Amalga hosted Doors Services (sometimes called Doors Worker Services) allow you to create processes that run in the background on your server, and which take advantage of Amalga features, such as security and management. Examples of these services include automated, scheduled generation of reports and automated sending of email notifications.

There can be only one class in a given assembly that inherits from the AmalgaDoorsWorker class, and that class must be a direct child of AmalgaDoorsWorker.

To build your own hosted Doors service

  1. Open . From the File menu, select New, then Project.
  2. Select the Amalga Doors Worker Project template, name your project, and click OK.
  3. If your service requires a virtual schema, then create one. For instructions, see Virtual Schema Designer.
  4. Add code to the Program.cs and Service.cs classes to complete your Doors service. Use Amalga ADO.NET to connect to the virtual schema.
  5. Configure the service by modifying the AmalgaService.xml file.
  6. Package and deploy the service.
