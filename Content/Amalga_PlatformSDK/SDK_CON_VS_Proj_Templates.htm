project templates

 

This topic describes the project templates included with for use with .

The  includes project templates to make it easier to develop both hosted applications and applications.

Hosted applications

Three project templates are provided for developing hosted applications, including hosted services. Use these project templates as the starting point for your hosted applications. Each template provides the appropriate deployment manifest, references, and sample code for developing a hosted application or hosted service.

  • Amalga Application Project

    This template provides an application manifest, which provides the metadata for a hosted application or hosted service. A Visual Studio solution that contains hosted applications, including hosted services, must contain at least one Visual Studio project created from this template. Other projects, such as an Amalga e project, must contain a reference to a project created from this template.

  • Amalga WCF Service Project

    This template provides a service manifest and sample classes for developing a hosted , including a class that inherits from AmalgaWcfService. This template also provides references to Microsoft.Amalga.Data.DataClient and Microsoft.Amalga.Hosting.Runtime namespaces, as well as references to the namespaces most commonly used while developing a .

  • Amalga Doors Worker Project

    This template provides a service manifest and sample classes for developing a hosted , including a class that inherits from AmalgaDoorsWorker. This template also provides references to Microsoft.Amalga.Data.DataClient and Microsoft.Amalga.Hosting.Runtime namespaces, as well as references for the namespaces most commonly used while developing a .

Display the project templates for hosted applications

After you install , the project templates are integrated with , and can be used like any other installed template included with .

  2. From the File menu, click New, and then click Project.
  3. In Installed Templates, click Visual C#.

applications

A template is provided to support applications that incorporate features and functionality. The project template, named Amalga Empty SP Project, supports both sandboxed and farm solutions.

This template provides a Features collection, a Solution Package deployment item, and a key file with an exportable public key. The project template also provides references to the most commonly used namespaces, as well as the namespaces most commonly used while developing a application.

Display the project templates for applications

After installing , the project templates are integrated with . However, the project template cannot be used unless is also installed. If you attempt to create a project based on the Amalga Empty SP Project template without first installing the required components, an error occurs.

To display the project templates in for applications

  2. From the File menu, click New, and then click Project.
  3. In Installed Templates, expand Visual C#, and then click Amalga.
