Data Connector

Whenever your application uses a virtual schema, it uses the data connector to retrieve data from the clinical database and the private database for the application. Data Connector functionality is provided by a hosted application, named DataConnectorApplication. Unlike most Amalga platform applications, you do not use the DataConnectorApplication application directly. Instead, you use the client, Microsoft.Amalga.Data.DataClient, to connect to the application views represented by the virtual schema for your hosted application.

The DataConnectorApplication application must be installed on the server that hosts the data. If your application accesses data, you must specify a dependency to the DataConnectorApplication application in the application manifest at design time. When installing your application, you can choose the DataConnectorApplication instance that will be used by your application to access data, supporting geographic scaling of applications on an topology.

For more information about using Data Connector to access data, see Data access on page 1.

Data Change Notification

Data Change Notification (DCN), also known as eventing, provides a way for your applications and services to be notified if data changes. Data Change Notification is provided by a hosted application, Microsoft.Amalga.Event.Application, containing multiple hosted services that you can use to define, subscribe to, and receive DCN messages. For more information about Data Change Notification, see Data-driven events.

Terminology

The terminology application contains a service that allows you to navigate healthcare concepts that often have more than one clinical term to describe them. The terminology service can be accessed using either Terminology entities or the Terminology API.
