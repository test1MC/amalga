Application dependencies

 

This topic describes application dependencies and requirements related to application dependencies.

The application manifest for Amalga hosted applications requires that all application dependencies are declared so that required applications are installed during deployment and are discoverable by the dependent application at runtime.

Your Amalga hosted application may be dependent on other applications. Examples of dependencies include:

  • Responding to events raised by another application
  • Requiring a library from another application
  • Requiring a user interface component from another application
  • Requiring specific data that is created by another application

Application dependencies are declared in hosted applications by adding Dependency elements to the Dependencies element in the application manifest for a hosted application.

If one or more application dependencies are declared, then :

  • During deployment of the dependent application, the deployment tools included with verify whether all required applications were previously installed. If the required applications are not deployed, an error occurs during deployment.
  • During runtime, the required applications are discoverable by the dependent application, so it can make use of the hosted services, settings, etc., of the dependent application.
