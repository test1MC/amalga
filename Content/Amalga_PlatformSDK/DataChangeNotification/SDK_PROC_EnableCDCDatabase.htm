Enable Change Data Capture for a database

 

This topic describes how to use a SQL Server script to enable Change Data Capture (CDC) for a database in .

The following procedure uses the sp_cdc_enable_db system stored procedure to enable Change Data Capture (CDC) for a database. When this system stored procedure is executed, it adds new objects, including functions, stored procedures, tables, and constraints, used by CDC to the database. These objects are needed by CDC to track the data changes made to CDC-enabled tables. For more information about the sp_cdc_enable_db system stored procedure, see sys.sp_cdc_enable_db (Transact-SQL), on Microsoft Developer Network at http://go.microsoft.com/fwlink/?LinkId=241586.

By default, no tables are enabled for CDC when you enable a database for CDC. You must explicitly enable CDC on the tables to be tracked within the CDC-enabled database.

You must enable CDC for a database before you can enable or disable CDC on tables contained within that database.

You can enable CDC for a database, as well as for specific tables, within the same SQL Server script.

For more information about enabling CDC for tables, see Enable Change Data Capture for a table on page 1.

You must be a member of the sysadmin fixed database role to perform the following procedure.

To enable CDC for a database

  3. Click New Query.

  4. Paste the following SQL Server script into the query window, replacing DatabaseName with the name of the database to be enabled for CDC.

    USE [DatabaseName]
GO
EXEC sys.sp_cdc_enable_db
GO
  5. Click Execute.
