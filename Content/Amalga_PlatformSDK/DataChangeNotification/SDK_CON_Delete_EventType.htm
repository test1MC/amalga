Deleting a DCN event type

 

This topic describes the process involved in deleting an existing Data Change Notification (DCN) event type by using the DeleteEventType operation.

A hosted application can delete a DCN event type, which also deletes the DCN event subscriptions associated with the specified DCN event type, by calling the DeleteEventType operation in the IEventManagementService contract for the EventingManagerService service.

Only the user account that created the DCN event type can delete that DCN event type. The user account that calls the DeleteEventType operation must match the value of the Owner property for the EventType object that represents the DCN event type to be deleted. Otherwise, an error occurs.

The DCN event type isn't actually deleted, however. Instead, the DCN event type is retired - the EventingManagerService sets the value of the State property for the EventType object that represents the DCN event type to ResourceState.Retired. Retired DCN event types are ignored for the purposes of tracking, queuing, dispatching, or sending DCN event messages.

The DeleteEventType operation performs the following tasks when deleting a DCN event type:

  • Deletes existing relationships between the DCN event type and existing DCN subscriptions.
  • Deletes existing DCN subscriptions for the DCN event type.
  • After all dependent objects are deleted, retires the DCN event type.

DCN event messages in the DCN message queue are not deleted when a DCN event type is retired. The DCN event subscriptions for the retired DCN event type are, however, deleted, so DCN event messages can become orphaned in the DCN message queue.
