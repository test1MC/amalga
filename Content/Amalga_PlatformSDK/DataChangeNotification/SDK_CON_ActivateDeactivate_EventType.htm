Activating and deactivating a DCN event type

 

HElP

 

 

This topic describes how to activate and deactivate an existing Data Change Notification (DCN) event type by using, respectively, the ActivateEventType and DeactivateEventType operations.

 

HELLO

Hosted applications can activate and deactivate a DCN event type, which either includes or excludes a DCN event type, respectively, from consideration when tracking, queuing, dispatching, or sending DCN error messages.

The ActivateEventType and DeactivateEventType operations, in the IEventManagementService service contract for the EventingManagerService service, respectively activate and deactivate the DCN event types identified by an array of event identifiers.

Only the user account that created the DCN event type can delete that DCN event type. The user account that calls the ActivateEventType or DeactivateEventType operation must match the value of the Owner property for the EventType object that represents the DCN event type to be deleted. Otherwise, an error occurs.

Calling the ActivateEventType operation for a DCN event type sets the value of the State property for the EventType object that represents the DCN event type to ResourceState.Active.

Correspondingly, calling the DeactivateEventType operation for a DCN event type sets the value of the State property to ResourceState.Inactive.
