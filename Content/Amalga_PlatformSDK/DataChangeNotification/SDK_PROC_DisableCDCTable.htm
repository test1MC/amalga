Disable Change Data Capture for a table

 

This topic describes how to use a SQL Server script to disable Change Data Capture (CDC) for a table in .

The following procedure uses the sp_cdc_disable_table system stored procedure to disable Change Data Capture (CDC) for a table. When this system stored procedure is executed, it attempts to remove data used by CDC to track the data changes made to the specified table. For more information about the sp_cdc_disable_table system stored procedure, see sys.sp_cdc_disable_table (Transact-SQL), on Microsoft Developer Network at http://go.microsoft.com/fwlink/?LinkId=241585.

You must enable CDC for a database before you can enable or disable CDC on tables contained within that database. If you disable CDC for a database, you cannot disable CDC for the tables contained in that database.

You can disable CDC for a database, as well as for specific tables, within the same SQL Server script. CDC should be disabled for tables first, before disabling CDC for the database.

For more information about disabling CDC for databases, see Disable Change Data Capture for a database on page 1.

You must be a member of the fixed server sysadmin role or the fixed database db_owner role to perform the following procedure.

To disable CDC for a table

  3. Click New Query.

  4. Paste the following SQL Server script into the query window, replacing DatabaseName with the name of the database containing the table, SchemaName, with the name of the schema containing the table, and TableName with the name of the table to be disabled for CDC.

    USE [DatabaseName]
GO
EXEC sys.sp_cdc_disable_table
	@source_schema = N'SchemaName',
	@source_name = N'TableName',
	@capture_instance = 'all'
GO
  5. Click Execute.
