Unsubscribing from and deleting DCN event types

 

This topic describes the processes involved in unsubscribing from and deleting existing DCN event types.

Hosted applications can unsubscribe from existing Data Change Notification (DCN) event types and, if the existing types are no longer needed, the host application can delete existing DCN event types.

Unsubscribing from a DCN event type

You can unsubscribe from an existing DCN event type by deleting a specified DCN event subscription, leaving the associated DCN event type in place for other subscribers.

Only the user account that created the DCN event subscription can delete that DCN event subscription.

For more information about unsubscribing from a DCN event type, see Deleting a DCN event subscription.

Deleting a DCN event type

You can also delete a DCN event type, which also deletes all associated DCN event subscriptions, regardless of the subscriber.

Only the user account that created the DCN event type can delete that DCN event type.

For more information about deleting a DCN event type, see Deleting a DCN event type on page 1.
