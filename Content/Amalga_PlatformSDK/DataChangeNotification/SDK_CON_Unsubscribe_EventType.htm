Deleting a DCN event subscription

 

This topic describes how to delete a DCN event subscription, without deleting the associated DCN event type, by using the Unsubscribe operation.

A hosted application can unsubscribe from a Data Change Notification (DCN) event type by deleting the DCN event subscription associated with that DCN event type. Deleting the DCN event subscription does not delete the associated DCN event type.

 

 

 

 

asdfasdfsadf

The Unsubscribe operation, in the IEventManagmentService contract for the EventingManagerService service, is used to delete a DCN event subscription.

Only the user account that created the DCN event subscription can delete that DCN event subscription. The user account that calls the Unsubscribe operation must match the value of the Owner property for the EventSubscription object that represents the DCN event subscription to be deleted. Otherwise, an error occurs.

The Unsubscribe operation performs the following tasks when deleting a DCN event type:

  • Deletes existing relationships between the DCN event subscription and the associated DCN event type.
  • Deletes the DCN event subscription.

Unlike deleting a DCN event type, in which the DCN event type is actually retired, but still present in the database for the Microsoft.Amalga.Event.Application application, the DCN event subscription is actually deleted from the database.
