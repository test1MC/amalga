Updating a DCN event type

 

This topic describes how to change an existing DCN definition by using the UpdateEventType operation.

Hosted applications can update existing DCN event types by retrieving an EventType object for an existing DCN event type, updating the desired properties, and then specifying the updated EventType object when calling the UpdateEventType operation in the IEventManagementService contract for the EventingManagerService service.

For more information about finding an existing DCN event type, see Finding a DCN event type on page 1.

The UpdateEventType operation performs the following actions when updating a DCN event type:

  • Confirms that a DCN event type with the event identifier specified in the EventId property of the updated EventType object exists.

    You cannot create a new DCN event type by calling the UpdateEventType operation. For more information about creating a new DCN event type, see Creating a DCN event type on page 1.

  • If the NeverExpire property of the updated EventType object is set to True, ensures that the ExpiresOn property is also set to DateTime.MaxValue. For more information about expiration dates, see Creating a DCN event type on page 1.

  • Ensures that the user account calling the UpdateEventType operation matches the Owner property of the existing DCN event type to be updated.

    Only the user account that created the DCN event type can update that DCN event type. Otherwise, an error occurs.

  • Confirms that the user account calling the UpdateEventType operation is authorized to access the eventable resources specified in the updated EventType object. Otherwise, an error occurs.
  • Validates existing DCN event subscriptions for the updated EventType object.
  • Updates the existing DCN event type using the updated EventType object.
