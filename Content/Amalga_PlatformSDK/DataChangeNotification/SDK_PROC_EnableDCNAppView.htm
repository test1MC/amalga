Enable or disable DCN on an

 

This topic describes how to enable or disable Data Change Notification for an application view in .

You can automatically enable or disable DCN on the databases and underlying data structures for an application view used by a hosted application by setting the Enable for Eventing property for an application view in .

When the application is deployed, the application deployment tool is then be used to automatically generate the SQL Server script needed to enable Change Data Capture (CDC) for the databases and tables referenced by the DCN-enabled application views for that hosted application.

To enable or disable DCN on an application view in

  1. In Solution Explorer, right-click the data manifest (.edmx) file for the hosted application, and then click Open.

  2. Select the appropriate application view in the visual design surface of .

    The entity type information is displayed in the Properties window.

  3. In the Properties window, navigate to and select the Enable for Eventing property.
  4. To enable DCN for the application view, set the Enable for Eventing property to True. Otherwise, set the property to False.
