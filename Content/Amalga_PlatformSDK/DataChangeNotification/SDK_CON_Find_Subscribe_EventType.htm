Finding and subscribing to DCN event types

 

This topic describes the processes involved in finding and subscribing to existing Data Change Notification (DCN) event types.

In some cases, a hosted application does not have to define new DCN event types, but instead must locate existing DCN event types defined by other hosted applications, and then subscribe to them. Multiple hosted applications can subscribe to an existing DCN event type, allowing flexible design patterns involving multiple hosted applications.

Finding an existing DCN event type

You can find and retrieve EventType objects for existing DCN event types by using either the FindEvents operation to retrieve all available DCN event types owned by the current user, or by using the FindEventTypeForId operation to retrieve a specific event type owned by the current user, by specifying an event identifier. Both operations are provided by the IEventManagementService contract for the EventingManagerService service.

For more information about finding a DCN event type, see Finding a DCN event type on page 1.

Subscribing to an existing DCN event type

You can subscribe to an existing DCN event type by defining a new DCN event subscription, assigning it to an existing DCN event type, and then calling the Subscribe method to create a new DCN event subscription.

For more information about subscribing to a DCN event type, see Subscribing to a DCN event type on page 1.
