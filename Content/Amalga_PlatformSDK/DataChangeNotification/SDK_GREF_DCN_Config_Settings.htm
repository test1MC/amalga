Data Change Notification configuration settings

 

This topic describes the configuration settings provided for the Data Change Notification hosted application and its associated hosted services.

The following configuration settings can specified by using the to configure the settings for an instance of Data Change Notification (DCN). These configuration settings can also be specified manually, in the application manifest, for application settings, or the appropriate service manifest, for service settings, of an instance of DCN.

Application settings

Name Data Type Description
IncludeResults String

Determines whether changed records are included in the payload of a DCN event message. The default value of this property is set to False. For more information about payloads in DCN event messages, see Processing a DCN event message on page 1.

The value of this property is converted to a Boolean value. If the value of this property is set to a string that cannot be converted to a Boolean value, then the property is set to False.

ChangeTrackerService service settings

Name Data Type Description
BatchSize Integer

The maximum size of the batch that is sent to the server for writing event data logs.

The value of this setting can be set to between 1 and 1000. The default value of this property is set to 100.
BatchTimeout Integer The time, in milliseconds, the service will wait before sending a batch of event logs to the server. The service automatically sends a batch of event logs to the server when either the maximum batch size specified in the BatchSize property is reached, or after waiting for the interval specified in this property, whichever comes first.
The value of this setting can be set to between 1 and 120000. The default value of this property is set to 3000.
CacheTimeoutInterval Integer

The interval, in seconds, for which a resolved Transact-SQL statement is cached.

The service keeps a copy of resolved Transact-SQL statements to compute data changes. These statements reflect the current user's entity binding and authorization on an entity.

The value of this setting can be set to between 120 and 1440. The default value for this setting is set to 120.
CommandTimeout Integer

The time, in seconds, to wait for a command issued to Change Data Capture by the service, before terminating the command.

The value of this setting can be set to between 1 and 360. The default value for this setting is set to 30.
DiagnosticsLogLevel String

The minimum log level at which the service writes log entries into the diagnostics log.

The value of this property can be set to the name of a valid diagnostics log level. The default value of this property is set to "Warning".
MaxAvailableEvents Integer

The maximum number of event types that can be processed by a single instance of Data Change Notification at any given time, to avoid overloading the database server.

The value of this setting can be set to between 100 and 2147483647. The default value for this setting is set to 1000.
MaxCdcRecords Integer

The maximum number of CDC changes per table.

The value of this setting can be set to between 1 and 2147483647. The default value of this property is set to 3000.
MaxChangedRecords Integer The maximum number of changed records retrieved by a single query from the service to Change Data Capture. The value of this setting can be set to between 1 and 2147483647. The default value of this property is set to 2000.
MaxChangeProcessorThreads Integer

The maximum number of requests to the change processor thread pool that can be active concurrently, per processor. Additional requests remain queued until a thread from the change processor thread pool becomes available.

The total number of threads in the thread pool is calculated by multiplying the value of this setting with the number of processors from the physical host running the service.

The value of this setting can be set to between 1 and 200. The default value of this property is set to 5.
MaxConcurrentQueries Integer

The maximum number of Transact-SQL queries, issued by the service, that can be active concurrently.

The value of this setting can be set to between 0 and 2147483647. The default value of this property is set to 25.
MaxEventMesageSize Integer

The maximum size, in bytes, allowed for a single event message.

The value of this setting can be set to between 256000 and 3000000. The default value of this property is set to 2000000.
MaxExecutionTime Integer

The maximum number of seconds to attempt processing a DCN event type. If this number is exceeded, processing times out and execution is terminated. Depending on the value of the MaxTimeoutRetries configuration setting, the service attempts to reprocess the DCN event type.

The value of this setting can be set to between 5 and 2147483647. The default value of this property is set to 240.
MaxTimeoutRetries Integer

The maximum number of times the service attempts to reprocess a DCN event type that has exceeded the executiont time set by the MaxExecutionTime configuration setting.

If a DCN event type cannot be processed after exceeding the number of timeouts set by this configuration property, the state of the DCN event type is set to ResourceState.Suspended. For more information about the state of a DCN event type, see Creating a DCN event type on page 1.

The value of this setting can be set to between 1 and 100. The default value of this property is set to 3.
MetadataReaderPollingInterval Integer

The interval, in seconds, the service will wait before polling for updates to DCN event types and subscriptions.

The value of this setting can be set to between 1 and 60. The default value of this property is set to 5.
PerformanceLogLevel String

The minimum log level at which the service writes log entries into the performance log.

The value of this property can be set to the name of a valid performance log level. The default value of this property is set to "Off".
PollingInterval Integer

The interval, in seconds, the service will wait before polling the database server for data changes.

The value of this setting can be set to between 0 and 2147483647. The default value for this setting is set to 10.
StaleEventBuffer Integer

The interval, in seconds, before the service resets a stale event for processing.

A stale event can occur if the service encounters an exception while processing an event in the message queue. In such cases, the event is marked in the message queue as processed, but the event has not actually been processed and, while so marked, is ignored by all instances of the service. This property indicates how long the service should wait before resetting the status of such events, so that they can be reprocessed by an instance of the service.

The value of this setting can be set to between 10 and 1200. The default value for this setting is set to 30.

EventingDispatcherService service settings

Name Data Type Description
ContinueOnError Boolean

Specifies how the service should respond if it fails to call the WCF event handler for an event subscription. If this property is set to True, the service logs the information about the error and continues to process the current batch of event messages. If this property is set to False, the service stops processing the current batch of event messages and rolls back the processing status for all of the event messages in the current batch.

If the value of this property is set to False, duplicate event messages may be sent to the subscriber.

The default value of this property is set to True.
DiagnosticsLogLevel String

The minimum log level at which the service writes log entries into the diagnostics log.

The value of this property can be set to the name of a valid diagnostics log level. The default value of this property is set to "Warning".
MaxConcurrentActions Integer

The maximum number of action invocations that can be performed by the service.

The value of this setting can be set to between 0 and 2147483647. The default value of this property is set to 10.
MaxDispatcherThreads  

The maximum number of requests to the dispatcher thread pool that can be active concurrently, per processor. Additional requests remain queued until a thread from the dispatcher thread pool becomes available.

The total number of threads in the thread pool is calculated by multiplying the value of this setting with the number of processors from the physical host running the service.

The value of this setting can be set to between 1 and 200. The default value of this property is set to 5.
PerformanceLogLevel String

The minimum log level at which the service writes log entries into the performance log.

The value of this property can be set to the name of a valid performance log level. The default value of this property is set to "Off".
