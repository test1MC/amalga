Create a new read-write application view

This topic describes how to create a new read-write application view using .

In some situations, you may want to create an application view that contains writable data. For example, you could create a view to store notes that users have written about data that has been ingested. When the virtual schema is created, will automatically create a database for the application instance, as well as the necessary tables and types in the database to hold the writable data.

The following procedure describes how to define a read-write application view in the virtual schema, using .

To create a new read-write application view using

  1. In Solution Explorer, right-click the data manifest (.edmx) file for the hosted application, and then click Open.
  2. Right-click an empty area in the visual design surface of , click Add, and then click Entity.
  3. In Solution Explorer, right-click the data manifest (.edmx) file Right-click in an empty portion of the designer window and select Add and then Entity.

  4. In Entity name, specify the name of the new read-write application view.

    The name must be unique within the virtual schema.

  5. In Base type, select (None).
  6. In Entity Set, specify a name for the entity set.
  7. In Key Property, select Create key property.
  8. In Property name, specify the name of the primary key property for the read-write application view.
  9. In Property type, specify the data type of the primary key property for the read-write application view.
  10. Click OK.

  11. In the Add Entity dialog box, fill out information about the entity you are creating and click OK.

The new read-write application view is now added to the visual design surface of . You can confirm that the application view is read-write by checking the value of the CanWrite property for the EntityType object that represents the read-write application view in the Properties window. If the property is set to True, it is a read-write application view; otherwise, it is a read-only applicaiton view.

You can now create new properties, such as scalar properties, for the new read-write application view.

To create a scalar property for a read-write application view

  1. In the visual design surface of , right-click the application view, click Add, and then click Scalar Property.

    A new scalar property is added to the entity with the following default property values:

    Property name Default value
    Type Int16
    Nullable True
    Entity Key

    False

    The new scalar property is not part of the entity key.
    Name "Property", followed by a number to make the property name unique in the entity and its hierarchy.

    The new scalar property is added in edit mode, so that the name can be changed.

  2. Replace the default property name with the intended name of the new scalar property.
