Data entity overview

 

This topic describes the function of data entities. .

data entities represent data in physical databases, but they provide an abstraction layer so that applications can reference that data without needing to interact directly with the physical database. When is installed, a large number of entities are provided that cover common healthcare concepts. However, your application may require entities that need to be created for this purpose.

You can create entities by using the . For instructions on how to create entities, see Create entities. Note that in addition to defining the new entities, you need to use the to bind the entities to databases.

When your application is deployed, the new entities that it uses will be imported. The deployment process ensures that everything in the data manifest for the application can resolve to a physical database. If an entity is notcannot resolved, then you will be prompted for a database to bind the entity to. Note that once an entity has been imported, it can be used by any application.

Entities are read-only. If your application requires private, writable data, then you can specify that data in the virtual schema. The data created in this fashion will be accessible only by the application instance. See Virtual Schema Designer for information on how to create writable data in the virtual schema.
