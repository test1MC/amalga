Validating a virtual schema

 

This topic describes how to validate a virtual schema using the VirtualSchema platform feature in .

You can invoke the Validate operation, from the IVirtualSchemaManagementService service contract for the VirtualSchemaService service, to validate and optionally activate a virtual schema instance. When invoking Validate, you must provide the instance name of the virtual schema to be activated and a Boolean value indicating whether to optionally activate the virtual schema instance if successfully validated.

This operation differs from the ActivateVirtualSchema operation in that this operation returns a Boolean value indicating whether the virtual schema instance was successfully validated, while the ActivateVirtualSchema operation returns a VirtualSchemaInfo object containing information about the virtual schema instance itself. If you need additional information about the virtual schema instance, use the ActivateVirtualSchema operation.

Invoking the operation

When invoked, the Validate operation performs the following actions:

  1. Ensures that the specified virtual schema instance exists; otherwise, an exception is thrown.

  2. Validates the specified virtual schema instance, to ensure that the following conditions are met. If the following conditions are not met, an exception is thrown:

    1. The virtual schema instance contains at least one entity.
    2. For each entity representing an application view, ensure that all entity bindings have a defining query.
  3. Activates the virtual schema instance.
  4. Returns a Boolean value indicating whether the virtual schema instance is valid.
