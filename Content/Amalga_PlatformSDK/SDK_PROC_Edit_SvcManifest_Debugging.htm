Edit a service manifest for debugging

 

This topic provides a procedure for editing the elements and attributes in a service manifest that are used when publishing a hosted application for debugging purposes.

Before a hosted application that contains one or more hosted services can be published for debugging by using the publishing feature in , several elements and attributes must be properly set in the service manifest for each hosted service in the hosted application. All of the elements and attributes that must be set prior to debugging are contained in the Test section of the Manifest Editor.

To publish your hosted application, each service manifest must have these names:

  • A valid instance name specified for the TestInstanceName attribute of the DeploymentManifest element, to identify the test instance of the hosted service. For more information about the DeploymentManifest element, see DeploymentManifest Element on page 1. This test instance name is also used to remove any existing test instances that may have been previously published for debugging.
  • A valid logical host name specified for the TestLogicalHostName attribute of the WcfServiceManifest element (for a ) or DoorsWorkerManifest element (for a ), to identify the logical host which will host and expose the hosted service for debugging purposes. For more information about the WcfServiceManifest or DoorsWorkerManifest elements, see, respectively, WcfServiceManifest Element on page 1 or DoorsWorkerManifest Element on page 1.

To edit a service manifest in for debugging

  1. In Solution Explorer, right-click the service manifest of a hosted service in the hosted application to be published, and then click Edit Manifest.
  2. If needed, expand Test.
  3. In TestInstanceName, type the instance name that will be used by this hosted service when the hosted application is published for debugging.

  4. In TestLogicalHostName, type the name of the logical host that will host and expose the hosted service when the hosted application is published for debugging.

  5. Click OK.
