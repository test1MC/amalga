Use WCF services to connect to virtual schemas

An hosted application can be any kind of application, such as an ASP.NET or Silverlight web application. Because Amalga hosted applications may use a virtual schema, it is possible for a web application to connect to a virtual schema directly. However, this affects performance negatively.However, this is not ideal for performance. Instead, if you build a WCF service that is an hosted application that uses the virtual schema, and then have your web application call the WCF service to access data, then you can take advantage of all of the performance-enhancing technology available to service-oriented architecture, such as clustering.

Best practice is to build a ( in an hosted application) that uses the virtual schema, and then set up your web application to call the to access data. This allows you to take advantage of the performance-enhancing technology available to service-oriented architecture, such as clustering.
