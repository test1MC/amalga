Reading event log entries

 

This topic describes how to read event log entries from the Doors event logs for .

You can use the Event Viewer, included with Doors, to examine and analyze the Doors event logs used by hosted applications, features, and applications in . maintains three custom event logs, into which events are written depending on the source of the event:

  • Amalga Applications

    Events from hosted applications, including hosted services owned by hosted applications, are written to this custom event log.

  • Amalga Features

    Events from platform features, including hosted services owned by platform features, are written to this custom event log. Events from hosted applications, including hosted services owned by hosted applications, may also be written to this custom event log if errors occur when writing events to the Amalga Applications event log.

  • AmalgaAppFramework

    Events from applications are written to this custom event log.

The following procedure describes how to use Event Viewer to read the Amalga Applications event log, used to store event log entries for hosted applications.

To use Event Viewer to read the Amalga Applications event log

  2. In the tree view, expand Applications and Services Logs.

  3. Click Amalga Applications.

Once you have selected an event log to examine and analyze, you can use the actions provided in the Actions pane of Event Viewer to create custom views, filter the event log, export event log entries, or clear the event log, as needed.
