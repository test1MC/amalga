Obtaining information about services

 

This topic describes how to obtain information from services.

In , services and clients model contracts using managed attributes, interfaces, and methods. To connect to a service using a WCF client, you need to first obtain the service contract from a service endpoint.

First, you must identify the service endpoint for the service to be used by your hosted application. For more information about identifying service endpoints, see Service Endpoint Reference on page 1.

Once you have a service endpoint, you can then use one of the following methods to obtain the necessary information and define a WCF client that you can incorporate into your hosted application.

  • You can generate a WCF client from a running WCF service, including platform services, by using the ServiceModel Metadata Utility Tool (svcutil.exe). This is the preferred method for generating a WCF client for an platform service. For more information about the ServiceModel Metadata Utility Tool, see "ServiceModel Metadata Utility Tool (Svcutil.exe)", on Microsoft Developer Network at http://go.microsoft.com/fwlink/?LinkId=234805.

    Once it is generated, you can add the WCF client to your hosted application. For more information about generating a WCF client, see Generate a WCF client on page 1.

  • You can also add a service reference within to generate a WCF client. For more information about adding a service reference, see Add a service reference on page 1.

The resulting WCF client can be used to create a WCF client channel and interact with a service endpoint. For more information about creating a WCF client channel, see Creating a WCF client channel on page 1.
