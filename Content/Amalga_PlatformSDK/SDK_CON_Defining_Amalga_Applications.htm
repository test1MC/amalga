Creating and configuring hosted applications

This topic describes how to create new projects in using project templates for hosted applications, and how to configure hosted applications by modifying the application manifest and service manifests.

To create a hosted application, including hosted services, in , you must have, at a minimum, a solution containing a project based on the Amalga Application Project template. This application project provides the metadata needed to identify and configure an application.

If you are developing a hosted service, the solution must also contain a project based on either the Amalga WCF Service Project or Amalga Doors Worker Project template, and that project must have a reference to an application project in the same solution.

This combination of projects provides a flexible, yet resilient framework for creating and configuring a hosted application or hosted service in with .
