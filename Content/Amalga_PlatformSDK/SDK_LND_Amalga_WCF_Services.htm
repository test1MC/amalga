Hosted s

 

This topic discusses the advantages of using s in an Amalga hosted application, and provides an overview of how to build a .

Amalga hosted s are an important building block of applications. Build your own by using a template, the , and . Finally, configure, package, and deploy the service.

Amalga hosted s allow you to rapidly build service-oriented applications that communicate across the web and the enterprise, taking advantage of features, such as security and management. Although you can build applications that use Amalga s using a variety of technologies, there are many advantages to using Amalga hosted applications.

The exposed service contracts for a hosted must be contained in a single assembly.

To build your own hosted WCF service

  1. Open . From the File menu, select New, then Project.
  2. Select the Amalga WCF Service Project template, name your project, and click OK.
  3. If your service uses a virtual schema, then create one by following the instructions in Createing a vVirtual sSchema.
  4. Add code to the IService.cs and Service.cs classes to complete your WCF service. Use Amalga ADO.NET.
  5. Configure the service by modifying the AmalgaService.xml file.
  6. Package and deploy the service.
