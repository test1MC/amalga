Publish a hosted application for debugging

 

This topic describes how to publish a hosted application from for debugging purposes, using the publishing feature included in .

The publishing feature included with allows you to use test configuration and instance information included in the application manifest and service manifests of a hosted application to quickly publish a hosted application from within to a target Amalga topology for debugging purposes.

Before you can use the publishing feature, however, you must ensure that the appropriate elements and attributes are added to the application manifest and service manifests, and then set to the values appropriate to your target Amalga topology and debugging requirements.

You must also successfully build the hosted application in before you can publish it, to ensure that the appropriate files are present prior to publication. The publishing feature will rebuild all of the projects required for the hosted application in during the publishing process.

To publish a hosted application in for debugging

  1. Edit the application manifest for debugging.

    For more information about editing an application manifest for debugging, see Edit an application manifest for debugging on page 1.

  2. Edit each service manifest for debugging.

    You must edit the service manifest for each hosted service that contains a reference to the hosted application to be published. For more information about editing a service manifest for debugging, see Edit a service manifest for debugging on page 1.

  3. Ensure that all projects required for the hosted application have been built in prior to publication.

  4. In Solution Explorer, right-click the project for the hosted application to be published, and then click Publish Application.
