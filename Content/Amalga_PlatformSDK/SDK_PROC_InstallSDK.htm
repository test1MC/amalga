Install the

 

This topic describes the procedure to install the components from the command line, using the Setup utility.

The provides a set of tools, templates, controls, and other features used to develop applications. If not already installed, the following components are installed when installing the feature:

  • Data Access Runtime
  • Data Access Runtime 32
  • Hosting Runtime
  • SDK

The following procedure uses the Setup utility included with to check for and, as needed, install those components. The procedure assumes that all other software prerequisites have been installed. For more information about software prerequisites, see System requirements and prerequisites on page 1.

To install the components

  1. Copy the installation media to a folder on the target virtual or physical machine.

    This folder is referred to in subsequent steps as the installation folder.

  2. Open a Command Prompt window with administrative rights on the target virtual or physical machine.

    On Doors Server 2008, click Start, click All Programs, click Accessories, right-click Command Prompt, and then select Run as administrator.

    On other operating systems, click Start, click All Programs, click Accessories, and then click Command Prompt.

  3. Run the following command, replacing InstallationFolder with the path of the installation folder.

    cd InstallationFolder\Platform

  4. Run the following command, replacing the following parameters, where needed, with the appropriate values. (See the table after the command for descriptions of parameters.)

    Setup.exe /Feature SDK /Operation Install /Eula Accept [ PlatformClusterName PlatformClusterName ] [ PlatformPort PlatformPort ]

    Parameter Value
    PlatformClusterName

    Optional. The cluster name for the platform. If not specified, the default value is set to ComputerName.UserDNSDomain, replacing ComputerName with the name of the computer and UserDNSDomain with the fully qualified DNS domain of the currently logged-on user account.
    PlatformPort Optional. The port number used by the platform. If not specified, the default value is set to 8889.
