Data manifests

 

This topic describes the data manifest, an EDMX file which defines a virtual schema, a conceptual model that separates access to data from where the data is actually stored.

For hosted applications, data access is managed using a virtual schema. Based on the ADO.NET Entity Framework, the virtual schema provides an abstraction layer that allows hosted applications to use a portable, dynamic data model. The virtual schema is defined as a data manifest (.edmx) file, which stores the virtual schema, the definitions of the underlying data structures referenced by the virtual schema, and the mappings between the virtual schema and the underlying data structures.

The virtual schema for a hosted application is defined in the conceptual schema definition language (CSDL) section of the data manifest. The entities used by that virtual schema are defined in the store schema definition language (SSDL) section of the data manifest. The mappings that connect the conceptual model to the storage model are defined in the mapping definition language (MDL) section of the data manifest.

For more information about the structure and contents of a data manifest file, see .edmx File Overview, on Microsoft Developer Network at http://msdn.microsoft.com/en-us/library/cc982042.aspx.

To include a virtual schema for a hosted application, you can add a data manifest file to the Amalga application project in for the hosted application.

Only one data manifest can be added to the Amalga application project for a hosted application. If you add more than one data manifest, an error occurs when the hosted application is published or deployed.

The data manifest is edited in using the , a graphical tool that allows you to work with the virtual schema in a visual design surface.
