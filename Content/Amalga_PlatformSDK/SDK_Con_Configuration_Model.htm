Application configuration model

 

The application configuration model is contained in manifest files which can be read and modified through . Separate manifests are required for hosted applications, hosted WCF services, and hosted Doors services. The manifests contain information such as instance data, settings, dependencies, authorization, and performance counters.

Amalga hosted applications, Amalga hosted WCF services, and Amalga hosted Doors services all have manifest files, which are XML files used for configuration. Application manifests are a mandatory description of the application properties, and are required in order to host an application in Amalga v3. Manifests are designed by application developers using the Amalga development tools in Visual Studio 2010. The configuration data can be read and modified through .

Application mManifest

The application manifest contains the following information:

Top level properties

Information about the application instance, such as name, global identity, version, and publisher.

Settings

The application settings, which enable Amalga  fabric to automatically create a configuration store for an administrator to manage the settings of the application, and automatically create the page in to manage these settings. Examples of settings could include a “Welcome text” specific to an application instance, the default size of queries returned by an application instance, and the background color of an application, configurable by an administrator.

Dependencies

The application dependencies, which gives runtime access to services from other application instances deployed in an  Amalga topology. At application deployment time, Amalga automatically checks whether the other Amalga applications required for this application to work are preinstalled in the target topology. If multiple instances of these applications are present, Amalga asks the administrator to pick the instance that will support the dependent application.

Authorization

The application authorization model, which is a description of all resources that require authorization that are used by the application, as well as the access levels required for a user in order to be able to access one of these resources.

WCF sService mManifest

The WCF service manifest contains the following information:

Top level properties

Information about the service instance, such as name, global identity, and version.

Service mManifest

Information about the service, including clustering mode and exposed service contracts and their bindings supports federated, streaming, and queue binding.

Settings

The service settings, which enable to automatically create a configuration store for an administrator to manage the settings of the service, and automatically create the page in to manage these settings.

Performance Counter Information

The performance counters introduced by this service. By specifying the performance counters here, you can access them through the Doors Performance Monitoring Tools and from hosted applications and services.

Doors sService mManifest

The Doors service manifest contains the following information:

Top level properties

Information about the service instance, such as name, global identity, version, and publisher.

Service mManifest

Information about the service, including clustering mode.

Settings

The service settings, which enable to automatically create a configuration store for an administrator to manage the settings of the service, and automatically create the page in to manage these settings.

Performance Counter Information

The performance counters introduced by this service. By specifying the performance counters here, you can access them through the Doors Performance Monitoring Tools and from hosted applications and services.
