WcfServiceManifest element

 

This topic describes the WcfServiceManifest element.

Contains service-specific metadata required by a hosted .

Syntax

<DeploymentManifest>
   <WcfServiceManifest
      ClusteringMode = "string"
      AuthenticationMode = "string"
      AccessLevel = "string"
      [ ClusterGroup = "string" ]
      [ TestLogicalHostName = "string" ]
      [ LogicalHostName = "string" ] >
      <Contracts>...</Contracts>
   </WcfServiceManifest>
</DeploymentManifest>

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality

1-1: Required element that can occur once and only once.

This element is required only for manifests. Other deployment manifest types do not use this element.

Element relationships

Relationship Element
Parent elements DeploymentManifest
Child elements Contracts

Attributes

Attribute Description
ClusteringMode

(Required string.) The clustering mode supported by the hosted . The following values are allowed.

Value Description
None The hosted does not support clustering.
NLB The hosted supports Network Load Balancing (NLB) clustering.
FailOver The hosted supports failover clustering.
AuthenticationMode

(Required string.) The authentication mode supported by the hosted . The following values are allowed.

Value Description
None None
AuthenticatedUser AuthenticatedUser
TrustedCaller TrustedCaller
AccessLevel

(Required string.) The access level supported by the hosted . The following values are allowed.

Value Description
Public Public
Private Private
ClusterGroup

(Optional string.) The name of the cluster group on which the hosted should be deployed.
TestLogicalHostName

(Optional string.) The name of the logical host on which the hosted should be deployed while running in debug mode.
LogicalHostName

(Optional string.) The name of the logical host on which the hosted should be deployed.

Remarks

This element is required only if the parent DeploymentManifest element represents a hosted manifest. Otherwise, this element cannot be included. This element provides information on the clustering support, authorization support, access level, and the logical host for the hosted , as well as the service contracts and operations exposed by the service.
