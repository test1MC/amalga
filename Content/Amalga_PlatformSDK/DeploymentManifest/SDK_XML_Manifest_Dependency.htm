Dependency element

 

This topic describes the Dependency element.

Declares a dependency upon another hosted application.

Syntax

<Dependencies>
   [ <Dependency
      Name="string"
      Type="string"
      [ InstanceName="string" ] >
      <Id>...</Id>
      <MinVersion>...</MinVersion>
   </Dependency> ]
</Element>

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-n: Optional element that can occur one or more times.

Element relationships

Relationship Element
Parent elements Dependencies
Child elements Id, MinVersion

Attributes

Attribute Description
Name

(Required string.) The name of the dependency.

The value specified for this attribute must be unique within the set of Dependency elements contained by the parent Dependencies element.
Type

(Required string.) The type of the dependency. The following values are allowed.

Value Description
Application Dependency upon a hosted application.
InstanceName

(Optional string.) The instance name of the dependency.

Remarks

Application dependencies identify other hosted applications on which the hosted application defined by the deployment manifest depends. For example, a hosted application that contains a virtual schema must also contain a dependency on a data connector application. An instance of the specified data connector application must be present on the target topology to host this application.

Application dependencies are checked while debugging a hosted application and when resolving the application manifest while deploying the hosted application. For more information about resolving deployment manifests, see Deployment manifests on page 1.
