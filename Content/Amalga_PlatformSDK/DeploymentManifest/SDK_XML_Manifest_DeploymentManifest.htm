DeploymentManifest element

 

This topic describes the DeploymentManifest element.

Describes the topmost element of a deployment manifest.

Syntax

<DeploymentManifest
   xmlns="http://www.microsoft.com/2011/10/amalga/deployment/hosting"
   Type = "Application" | "WcfService" | "DoorsWorker"
   [ Name = "string" ]
   Id = "guid"
   Version = "string"
   [ TestInstanceName = "string" ]
   [ InstanceName = "string" ] >
   [ <Settings>...</Settings> ]
   (
      <ApplicationManifest>...</ApplicationManifest>
      [ <Dependencies>...</Dependencies> ]
      [ <TestConfiguration>...</TestConfiguration> ]
   ) |
   ( 
      (
         <WcfServiceManifest>...</WcfServiceManifest> | 
         <DoorsWorkerManifest>...</DoorsWorkerManifest>
      )
      <PerformanceCategories>...</PerformanceCategories>
   )
</DeploymentManifest>

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 1-1: Required element that can occur once and only once.

Element relationships

Relationship Element
Parent elements None
Child elements ApplicationManifest, Dependencies, PerformanceCategories, Settings, TestConfiguration, WcfServiceManifest, DoorsWorkerManifest

Attributes

Attribute Description
xmlns

(Required string.) The Uniform Resource Locator (URL) of the XML schema that supports the deployment manifest.
Name

(Optional string.) The name of the object represented by the deployment manifest. The value of this attribute depends on the value of the Type attribute, as described in the following table:

Type Value Description
Application The name of the hosted application.
WCFService The name of the hosted .
DoorsWorker The name of the hosted .

The value specified for this attribute cannot contain spaces.
Type

(Required string.) The type of object represented by the deployment manifest. The following values are allowed.

Value Description
Application Hosted application
WCFService Hosted
DoorsWorker Hosted
Id

(Required string.) The GUID used to uniquely identify the object represented by the deployment manifest.
Version

(Required string.) The string representation of a System.Version object, defining the version of the object represented by the deployment manifest.
TestInstanceName

(Optional string.) The fully qualified domain name (FQDN) of the instance name to be used when publishing the object represented by the deployment manifest for debugging. The value of this attribute is also used to remove any previously published test instances of the object during the publishing process.

The value specified for this attribute cannot contain spaces.
InstanceName

(Optional string.) The name of the instance used when resolving the deployment manifest for the specified object. This attribute is used to uniquely identify an instance of the object represented by the deployment manifest while deploying a hosted application.
For more information about resolving deployment manifests, see Deployment manifests on page 1.

The value specified for this attribute cannot contain spaces.

Remarks

The DeploymentManifest element is the root element of a deployment manifest. The type of deployment manifest depends on the value of the Type attribute, and determines the child elements contained by the DeploymentManifest element, as described in the following table.

Type value Description
Application

The DeploymentManifest element represents an application manifest. All hosted applications must have an application manifest, which provides the metadata necessary for to host the application. For example, if a hosted application represents a hosted , the hosted application must have both an application manifest and a manifest.

For an application manifest, the DeploymentManifest element must contain an ApplicationManifest element.
WCFService The DeploymentManifest element represents a service manifest for a hosted . The manifest defines the configuration settings and performance counters supported by the hosted , and also provides the metadata necessary to expose and grant access to the service operations for a hosted .
For a manifest, the DeploymentManifest element must contain a WcfServiceManifest element.
DoorsWorker The DeploymentManifest element represents a service manifest for a hosted . The manifest defines the configuration settings and performance counters supported by the hosted .
For a manifest, the DeploymentManifest element must contain a DoorsWorkerManifest element.
