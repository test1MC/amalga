Add an application dependency

 

This topic describes how to use the Manifest Editor to add an application dependency to an application manifest.

Hosted applications can depend on other applications for features or support, and require that those dependent applications be present on a target Amalga topology before deployment. For example, any hosted application that contains a data manifest and accesses data from the Amalga platform depends on the DataConnector hosted application included with the , and could not access data without having a valid instance of the DataConnector hosted application deployed on the target Amalga topology. The deployment tools provided with the Amalga platform check for application dependencies prior to deployment, stopping deployment if the requisite applications are not present.

You can explicitly specify such dependencies by using the Manifest Editor in to add Dependency elements to the Dependencies element of an application manifest for a hosted application. For more information about the Dependency element, see Dependency Element on page 1.

To add an application dependency to an application manifest in

  1. In Solution Explorer, right-click the application manifest to which an application dependency must be added, and then click Edit Manifest.
  2. If needed, expand Data.
  3. Select Dependencies, and then click the ellipsis button (...) to display the Dependency Collection Editor.
  4. Click Add.
  5. Set the attributes for the new Dependency element to the appropriate values.
  6. Click OK.
  7. Click OK.
