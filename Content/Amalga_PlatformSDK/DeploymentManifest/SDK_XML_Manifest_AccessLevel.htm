AccessLevel element

 

This topic describes the AccessLevel element.

Defines or identifies an access level exposed by the hosted application.

Syntax

<AccessLevels>
   <AccessLevel Name="string" >
      [ <Role>...</Role> ]
   </AccessLevel>
</AccessLevels>

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality

AccessLevels

1-n: Required element that can occur one or more times.

Resource

0-n: Optional element that can occur one or more times.

Element relationships

Relationship Element
Parent elements AccessLevels, Resource
Child elements Role

Attributes

Attribute Description
Name

(Required string.) The name of the access level. This value is displayed in .

Remarks

The usage of this element depends on the parent element.

AccessLevels

When included in a parent AccessLevels element, this element defines an access level exposed by the hosted application. The access level defined by this AccessLevel element is exposed to for managing access to the hosted application, and is used to specify for a user role the appropriate access level to the hosted application.

If one or more Role elements are included in this AccessLevel element, the built-in roles identified by those elements are automatically granted authorization to that access level for the hosted application. This automatic authorization cannot be modified by an administrator.

Resource

When included in a parent Resource element, the AccessLevel element identifies an access level that has been granted access to that resource. The Name attribute of the AccessLevel element for the parent Resource element must match the Name attribute of an existing AccessLevel element included in a parent AccessLevels element.

Role elements cannot be included in a parent AccessLevel element if the AccessLevel element is included in a parent Resource element.
