PerformanceCategory element

 

This topic describes the PerformanceCategory element.

Declares a single category of performance counters for the hosted or hosted .

Syntax

<PerformanceCategories>
   <PerformanceCategory
      Name = "string"
      Version = "string"
      CodeName = "string"
      Help = "string"
      MultiInstance = "boolean" >
      <PerformanceCounter>...</PerformanceCounter>
   </PerformanceCategory>
</PerformanceCategories>

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 1-n: Required element that can occur one or more times.

Element relationships

Relationship Element
Parent elements PerformanceCategories
Child elements PerformanceCounter

Attributes

Attribute Description
Name

(Required string.) The display name of the performance counter category.
Version

(Optional string.) The string representation of a System.Version object, defining the version of the performance counter category. If no value is specified, the default value is set to an empty string ("").

The value of this attribute should be incremented each time the definition of the PerformanceCategory element is changed, to allow different versions of the hosted or hosted to run side-by-side without conflicting performance counter categories.
CodeName

(Required string.) The name to be used when referring to the performance counter category in C# code.

The value of this attribute must conform to the C# naming convention.

Help

(Required string.) The help text for the performance counter category.
MultiInstance

(Required Boolean.) Indicates whether the performance counter category is multi-instance. If set to true, the performance counter category is multi-instance; otherwise, the performance counter category is single-instance.

Remarks

If the Version attribute is specified, the value is appended to the Name attribute to define the full name of the performance counter category. For example, if the Name attribute was set to "TestCategory" and the Version attribute was set to "1.0.0.0", then the full name for the resulting performance counter category would be "TestCategory - 1.0.0.0". The full name is shown in the Performance Object field of the Performance Viewer application's Add Counter dialog box. The Version attribute is appended to prevent different versions of the same hosted or hosted from overwriting the performance counter categories of each other.
