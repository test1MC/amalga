ApplicationAuthorization element

 

This topic describes the ApplicationAuthorization element.

Defines the resources exposed in the hosted application, as well as the access levels required to access the exposed resources.

Syntax

<ApplicationManifest>
   [ <ApplicationAuthorization>
      <AccessLevels>...</AccessLevels>
      <Resources>...</Resources>
   </ApplicationAuthorization> ]
</ApplicationManifest>

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements ApplicationManifest
Child elements AccessLevels, Resources

Remarks

The contents of the ApplicationAuthorization element are used in to manage access and authorization for exposed resources, such as s and service operations, provided by the object defined in the deployment manifest. The raw contents of this element are stored internally by .
