TestConfiguration element

 

This topic describes the TestConfiguration element.

Declares the services used while debugging the hosted application.

Syntax

<DeploymentManifest>
   <TestConfiguration
      [ VirtualSchemaDataHost="string" ] />
</DeploymentManifest>

      [ DiscoveryServiceUrl="string" ]

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DeploymentManifest
Child elements None

Attributes

Attribute Description
DiscoveryServiceUrl

(Optional string.) The Uniform Resource Locator (URL) of the discovery service for the target topology to use when publishing the hosted application for debugging.
VirtualSchemaDataHost

(Optional string.) The URL of the data host for the target topology to be used when publishing the virtual schema of the hosted application for debugging.

Remarks

This element can be included only in a parent DeploymentManifest element that represents an application manifest. The metadata included in this element is required when a hosted application is published from , providing the URLs needed for the discovery service and URL of the data host for the target topology to be used for debugging purposes.

This element is not included in the application manifest by default. Use the Manifest Editor to add, and then set the attributes for, this element prior to debugging a hosted application in .
