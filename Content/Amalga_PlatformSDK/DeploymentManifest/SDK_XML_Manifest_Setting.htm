Setting element

 

This topic describes the Setting element.

Declares a specific configuration setting that, when the hosted application is deployed, is stored in the configuration store for the object represented by the deployment manifest.

Syntax

<Settings>
   [ <Setting
   Name="string" Type="string" Description="string" >
      [ <DefaultValue>...</DefaultValue> ]
      [ <Validation>...</Validation> ]
   </Setting> ]
</Settings>

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-n: Optional element that can occur one or more times.

Element relationships

Relationship Element
Parent elements Settings
Child elements DefaultValue, Validation

Attributes

Attribute Description
Name

(Required string.) The name of the configuration setting.
Type

(Required string.) The supported data type of the setting. The following data types are supported:

  • System.String
  • System.Int32
  • System.Int64
  • System.Guid
  • System.DateTime
  • System.Boolean
  • System.Double
Description

(Optional string.) Description of the setting, displayed in Amalga Management Console.

This attribute supports localization.

Remarks

Once an instance of a hosted application is deployed, you can change the value initially specified by the deployment manifest for a configuration setting by using any of the following methods:

  • provides an administrative user interface for managing the configuration of a hosted application or hosted service. For more information about application management, see Application management on page 1.

  • cmdlets

    The Get-AmConfigurationSetting and Set-AmConfigurationSetting cmdlets, included with , can be used to get and set configuration settings for a hosted application or hosted service.

  • API

    The AmalgaConfigurationManager class can be used to programmatically manage the configuration settings of a hosted application or hosted service. For more information, see Application configuration and user settings on page 1.
