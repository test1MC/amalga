Dependencies element

 

This topic describes the Dependencies element.

Declares a set of application dependencies upon other hosted applications.

Syntax

<DeploymentManifest>
   [ <Dependencies>
      <Dependency>...</Dependency>
   </Dependencies> ]
</DeploymentManifest>

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-1: Optional element that can occur once and only once.

Element relationships

Relationship Element
Parent elements DeploymentManifest
Child elements Dependency

Remarks

Application dependencies identify other hosted applications on which the hosted application defined by the deployment manifest depends. For example, a hosted application that contains a virtual schema must also contain a dependency on a data connector application. An instance of the specified data connector application must be present on the target Amalga topology to host this application.

Application dependencies are checked while debugging a hosted application and when resolving the application manifest while deploying the hosted application. For more information about resolving deployment manifests, see Deployment manifests on page 1.
