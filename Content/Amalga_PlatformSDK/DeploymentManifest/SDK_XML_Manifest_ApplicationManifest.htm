ApplicationManifest element

 

This topic describes the ApplicationManifest element.

Contains application-specific metadata required by a hosted application.

Syntax

<DeploymentManifest>
   <ApplicationManifest
      Publisher="string"
      AmalgaMinVersion="string" >
      <ApplicationAuthorization>...</ApplicationAuthorization>
   </ApplicationManifest>
</DeploymentManifest>

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality

1-1: Required element that can occur once and only once.

This element is required only for application manifests. Other deployment manifest types do not use this element.

Element relationships

Relationship Element
Parent elements DeploymentManifest
Child elements ApplicationAuthorization

Attributes

Attribute Description
Publisher

(Required string.) The name of the publishing entity for the hosted application. This name is displayed in .

This attribute supports localization.
AmalgaMinVersion

(Required string.) The string representation of a System.Version object, representing the minimum version of required to run this hosted application.

Remarks

This element is required only if the parent DeploymentManifest element represents an application manifest. Otherwise, this element cannot be included. This element provides information on the publisher of a hosted application, as well as the authorization information for the hosted application, such as the resources exposed by the hosted application and the access levels needed to access those exposed resources.

The value of the AmalgaMinVersion attribute must be set to a valid string representation of the major, minor, build, and revision components of a System.Version object. This attribute is used during deployment to ensure that the hosted application can be supported by the topology on which the hosted application is being deployed.
