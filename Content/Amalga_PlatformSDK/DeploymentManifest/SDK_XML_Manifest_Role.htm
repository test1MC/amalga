Role element

 

This topic describes the Role element.

Identifies an built-in role to be automatically granted access to the access level for the hosted application, when the application is deployed.

Syntax

<AccessLevel>
   <Role Name="string" />
</AccessLevel>

Element characteristics

Characteristic Description
Data type and length None
Default value None
Cardinality 0-n: Optional element that can occur one or more times.

Element relationships

Relationship Element
Parent elements AccessLevel
Child elements None

Attributes

Attribute Description
Name

(Required string.) The name of the built-in role which will be automatically granted authorization for the access level. The following values are allowed.

Value Description
BUILTIN\AmalgaSysAdmin system administrator
BUILTIN\AmalgaService service
BUILTIN\AmalgaUsers users
BUILTIN\HVPatient patient

Remarks

If one or more Role elements are included in an AccessLevel element, the built-in roles identified by those elements are automatically granted authorization to that access level for the hosted application.

Once granted, this automatic authorization cannot be modified by an administrator.
