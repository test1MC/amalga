Prepare data manifest for deployment

This topic describes how to set properties for the virtual schema in a data manifest, as well as properties for the data manifest itself, to prepare the data manifest for deployment.

Once all of your views are complete, you must specify the publisher name for the virtual schema contained in the data manifest (.edmx) file, and then set properties on the data manifest file itself, to prepare the data manifest for deployment.

To prepare a data manifest for deployment

  1. In Solution Explorer, right-click the data manifest (.edmx) file for the hosted application, and then click Open.
  2. Right-click an empty area in the visual design surface of , and then click Properties.
  3. In the Properties window, set the value of the Publisher property to the desired name of the publisher.
  4. In Solution Explorer, select the data manifest (.edmx) file.

  5. In the Properties window, set the following properties to the appropriate values, as described.

    Property name Value
    Build Action Embedded Resource
    Copy to Output Copy Always
    Custom Tool Nothing
    Custom Tool Namespace Nothing

    The following screenshot shows the Properties window for a typical data manifest.

  6. Save the data manifest file.

The virtual schema and supporting objects will be deployed when the application is deployed.
