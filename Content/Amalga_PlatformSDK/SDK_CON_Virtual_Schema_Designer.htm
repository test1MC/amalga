This topic describes how to use the to create virtual schemas for hosted applications.

The is a tool, based on the ADO.NET Entity Data Model Designer , that enables point-and-click modification of a data manifest (.edmx) file in . You can use the to visually create and modify entities, associations, mappings, and inheritance relationships. You can also validate the contents of a data manifest. For more information about data manifests, see Data manifests and virtual schemas on page 1.

The works together with the Update Wizard to allow you to generate, edit, and update a data manifest.

Although available, the does not support the Generate Database Wizard.

The consists of the following components:

  • A visual design surface for editing the conceptual model, which defines the for a hosted application. You can create, modify, or delete entities and associations.
  • A Mapping Details window for viewing and editing mappings. You can map entity types or associations to the objects from underlying data sources, such as Amalga entities provided by the , that are declared in the storage model.
  • A Model Browser window that provides tree views of the conceptual model and storage model.
