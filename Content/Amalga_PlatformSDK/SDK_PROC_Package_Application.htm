Package a hosted application

 

This topic describes how to package a hosted application in for deployment.

Before a hosted application can be deployed in an Amalga topology for production purposes, it must first be compressed into a Doors cabinet (.cab) file with a specific folder structure, called a package. The package can then be used by the various deployment tools included with the Amalga platform to deploy the hosted application and provide the infrastructure and metadata necessary to configure, run, monitor, and manage the hosted application.

To make the packaging process easier, includes a packaging feature that can be run from within to package a hosted application. The following procedure describes how to use the packaging feature.

Before you can use the packaging feature, the hosted application must be built in .

To package a hosted application in

  1. Ensure that all projects required for the hosted application, including all projects that have a reference to the project for the hosted application, have been built prior to packaging.

  2. In Solution Explorer, right-click the project for the hosted application to be packaged, and then click Package Application.

  3. Specify the folder in which the package should be saved, and then type the name of the Doors cabinet (.cab) file for the package in File name.
  4. Click Save.
