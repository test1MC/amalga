Terminology

 

This topic describes the terminology platform application, a hosted application included with that provides a flexible mechanism by which relationships between concepts and clinical terminology can be modeled and used to standardize numerical and textual content.

Managing terminology is a challenge in any healthcare software system. Many healthcare concepts have more than one clinical term that is used to described them, and they often have a series of related terms that describe related concepts. provides a platform application, named TerminologyApplication, that gives you a way to access the entities that you need, without the terms employed by your application's user having to exactly match a specific term associated with that entity.

The TerminologyApplication application includes the TerminologyService service, which in turn provides two service contracts that can be accessed by hosted applications:

  • ITerminologyQueryService

    This service contract allows hosted applications to retrieve terminology content, such as terminologies, concepts, synonyms, relationships, and relationship types from the terminology database.

    You can also create application views for terminology entities, contained in the Microsoft.Terminology entity namespace, in your hosted application to directly retrieve terminology content. For more information about these terminology entities, see Microsoft.Terminology entity namespace on page 1.

    For example, your end user can enter a term, and your hosted application can use the operations provided by the ITerminologyQueryService contract to retrieve and display related terms from which your end user can choose.

  • ISemanticLabelTaggingService

    This service allows hosted applications to retrieve or add the source systems and data vectors that represent data assets, such as databases, tables, and columns. The service also allows hosted applications to assign a semantic tag to a data vector, tagging the data asset represented by that data vector with a semantic tag for later use in data operations that involve that data asset.

    This service is also used by Message Shredder and Data Parser parsers during the data ingestion process to register a data vector and, if possible, associate it with a semantic tag. A data vector contains a positional tag for each feed, its source system, and the physical location of the tag in the database. The VirtualSchemaService service, part of the Virtual Schema feature, uses this information to generate entity binding queries.

    For example, you can use the operations provided by the ISemanticLabelTaggingService to add a source system, data vectors, and semantic tags for the read-only application views of a third-party hosted application whose entity and attribute semantic labels used a different terminology.
