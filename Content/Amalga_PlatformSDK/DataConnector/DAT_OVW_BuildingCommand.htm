Working with commands and parameters

 

This topic describes how to work with the various objects included in to build and execute commands.

After establishing a connection to an instance of the DataConnector platform application, you can execute commands and return results from the instance using an AmalgaDbCommand object. You can create a command using one of the constructors for the AmalgaDbCommand object, specifying optional parameters such as the SQL statement to execute, an AmalgaDbConnection object, or an AmalgaDbTransaction object. You can also use the CreateCommand method of the AmalgaDbConnection object to create a command for a particular connection. The SQL statement being executed by the command can be configured using the CommandText property of the AmalgaDbCommand object. If the SQL statement contains parameters, you can use the AmalgaDbParameter object to pass parameter values during execution.
