schema collections

 

This topic describes the provider-specific schema collections, including schema restrictions, supported by .

The following provider-specific schema collections are supported by . You can query the .NET data provider included with , .NET Framework Data Provider for Amalga, to determine the list of supported schema collections by calling the GetSchema method of the AmalgaDbCollection object with no arguments.

Columns

This schema collection exposes information about all of the properties for entities and application views supported by .

Column name Data type Description
TABLE_CATALOG string? schema

The name of the platform feature or hosted application.

If the column represents an entity property for a public entity registered in the Entity Registry, the value of this column is set to "Amalga Entity Registry". Otherwise, the column represents an entity property for an application view.
TABLE_SCHEMA string If the column represents an entity property for a public entity registered in the Entity Registry, the value of this column is set to the namespace for that public entity. Otherwise, DBNull.Value is returned.
TABLE_NAME string The name of the entity or application view.
COLUMN_NAME string The name of the property for the entity or application view.
CHARACTER_MAXIMUM_LENGTH int Maximum length, in characters, for binary data, character data, or text and image data. Otherwise, DBNull.Value is returned.
ORDINAL_POSITION int The ordinal position of the property within the entity or application view.
COLUMN_DEFAULT string Default value of the property for the entity or application view.
IS_NULLABLE string Nullability of the property. If the property is nullable, the value of this column is set to "True". Otherwise, the value of this column is set to "False".
DATA_TYPE string System-supplied data type.
NUMERIC_PRECISION unsignedByte Precision of approximate numeric data, exact numeric data, integer data, or monetary data. Otherwise, DBNull.Value is returned.
NUMERIC_SCALE int Scale of approximate numeric data, exact numeric data, integer data, or monetary data. Otherwise, DBNull.Value is returned.
CODING_SYSTEM string Ignored.
DATA_FORMAT string Ignored.
OPTIONAL boolean The optionality of the property for the entity or application view.
CONCEPT string Semantic tag applied to the column. If no semantic tag has been applied to the column, DBNull.Value is returned.

Databases

This schema collection exposes information about all of the virtual schemas supported by .

Column name Data type Description
DATABASE_NAME string Name of the virtual schema.
PUBLISHER string Publisher of the virtual schema.
CREATE_DATE dateTime Creation date of the virtual schema.
TYPE string If set to "Public", the virtual schema can be accessed by platform features and hosted applications other than the one that published it. If set to "Internal", only the platform feature or hosted application that published the virtual schema can access it.
