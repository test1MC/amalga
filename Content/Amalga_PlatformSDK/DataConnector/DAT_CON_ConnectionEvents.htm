Connection events

 

This topic describes how to handle events for the AmalgaDbConnection object in .

The AmalgaDbConnection object implements the StateChange event, from the DbConnection object in ADO.NET, which you can use to determine if the state of the AmalgaDbConnection object has changed.

The AmalgaDbConnection object does not implement the InfoMessage event from the DbConnection object in ADO.NET.

Working with the StateChange event

The StateChange event occurs when the connection state of an AmalgaDbConnection object changes. The StateChange event receives a StateChangeEventArgs object that enables you to determine the change in the state of the AmalgaDbConnection object by comparing the OriginalState and CurrentState properties. The OriginalState property is set to a ConnectionState enumeration value that indicates the state of the object prior to the StateChange event. The CurrentState property is set to a ConnectionState enumeration value that indicates the state of the object at the time of the StateChange event.

Example

The following example uses an event handler for the StateChange event to write a message to the console when the connection state of the AmalgaDbConnection object changes.

protected static void OnStateChange(object sender, 
    StateChangeEventArgs args)
{
    Console.WriteLine(
        "The connection state has changed from {0} to {1}.",
        args.OriginalState,
        args.CurrentState);
}
