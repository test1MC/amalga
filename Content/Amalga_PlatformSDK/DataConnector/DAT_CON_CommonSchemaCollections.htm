Common schema collections

 

This topic describes the common schema collections, including schema restrictions, supported by .

The following common schema collections are supported by . You can query the .NET data provider included with , .NET Framework Data Provider for Amalga, to determine the list of supported schema collections by calling the GetSchema method of the AmalgaDbCollection object with no arguments.

DataSourceInformation

This schema collection exposes information about the data source to which is currently connected.

This schema collection has no schema restrictions or identifier parts.

Column name Data type Description
CompositeIdentifierSeparatorPattern string

The regular expression to match the composite separators in a composite identifier.

For , this column is set to "\.".

DataSourceProductName

 string

The name of the product accessed by the data provider.

For , this column is set to "Amalga".
DataSourceProductVersion string

The version of the product, in the native format used by the product, accessed by the data provider.

For , this column is set to "0.0".
DataSourceProductVersionNormalized string

The normalized version of the product, such that it can be compared with String.Compare(). The format of this column is consistent for all versions of the data provider, to prevent version 10 from sorting between version 1 and version 2.

For , this column is set to "0.0".
GroupByBehavior GroupByBehavior

A System.Data.Common.GroupByBehavior enumeration value that specifies the relationship between the columns in a GROUP BY clause and the non-aggregated columns in the select list.

For , this column is set to GroupByBehavior.Unrelated.
IdentifierPattern string A regular expression that matches an identifier and has a match value of the identifier. For example, "[A-Za-z0-9_#$]".
IdentifierCase IdentifierCase

A System.Data.Common.IdentifierCase enumeration value that indicates whether non-quoted identifiers are treated as case sensitive.

For , this column is set to IdentifierCase.Insensitive.
OrderByColumnsInSelect bool

Specifies whether columns in an ORDER BY clause must be in the selected list. A value of true indicates that they are required to be in the selected list;, a value of false indicates that they are not required to be in the selected list.

For , this column is set to false.
ParameterMarkerFormat string

A format string that represents how to format a parameter.

For , this column is set to "{0}".

By default, named parameter placeholders are supported by , and the contents of this column reflect that default option. However, also supports sequential parameter placeholders. For more information about parameter placeholders, see Configuring parameters and parameter data types on page 1.

ParameterMarkerPattern string

A regular expression that matches a parameter marker. It will have a match value of the parameter name, if any.
ParameterNameMaxLength int

The maximum length of a parameter name in characters.

For , this column is set to 128.

By default, named parameter placeholders are supported by , and the contents of this column reflect that default option. However, also supports sequential parameter placeholders. For more information about parameter placeholders, see Configuring parameters and parameter data types on page 1.

ParameterNamePattern string

A regular expression that matches the valid parameter names.

By default, named parameter placeholders are supported by , and the contents of this column reflect that default option. However, also supports sequential parameter placeholders. For more information about parameter placeholders, see Configuring parameters and parameter data types on page 1.

QuotedIdentifierPattern string

A regular expression that matches a quoted identifier and has a match value of the identifier itself without the quotes.

For , this column is set to "(([^\[]|\]\])*)".
QuotedIdentifierCase IdentifierCase

A System.Data.Common.IdentifierCase enumeration value that indicates whether quoted identifiers are treated as case sensitive.

For , this column is set to IdentifierCase.Insensitive
StatementSeparatorPattern string

A regular expression that matches the statement separator.

For , this column is set to ";".
StringLiteralPattern string

A regular expression that matches a string literal and has a match value of the literal itself.

For , this column is set to "'(([^']|'')*)'".
SupportedJoinOperators SupportedJoinOperators

A bitmask of System.Data.Common.SupportedJoinOperators enumeration values that specify what types of SQL JOINjoin statements are supported by the data source.

For , this column is set to a bitmask that includes all SupportedJoinOperators enumeration values other than SupportedJoinOperators.None.

DataTypes

This schema collection exposes information about the data types that are supported by .

This schema collection has no schema restrictions or identifier parts.

Column name Data type Description
TypeName string

The data type name.
ProviderDbType int The type value that should be used when specifying a parameter's type.
ColumnSize long

The length of a non-numeric column or parameter refers to either the maximum or the length defined for this type.

For character data, this is the maximum or defined length in units.

For date-time data types, this is the length of the string representation (assuming the maximum allowed precision of the fractional seconds component).

For numeric data types, this is the upper bound on the maximum precision of the data type.
CreateFormat string The format string that represents how to add this column to a data definition statement, such as CREATE TABLE. Each element in the array should be represented by a "parameter marker" in the format string.
CreateParameters string The creation parameters that must be specified when creating a column of this data type. Each creation parameter is listed in the string, separated by a comma in the order they are to be supplied.
DataType string The name of the .NET Framework type for the data type.
IsAutoIncrementable bool

If set to true, indicates that a value of this data type may be auto-incrementing.

This merely indicates whether a column of this data type may be auto-incrementing, not that all columns of this type are auto-incrementing.
IsBestMatch bool

If set to true, indicates that the data type is the best match between all data types in the data store and the .NET Framework type specified in the value of the DataType column.

For each set of rows in which the value of the DataType column is the same, the IsBestMatch column is set to true for only one row.
IsCaseSensitive bool If set to true, indicates that the data is a character type and is case-sensitive.
IsFixedLength bool If set to true, indicates that columns of this data type created by data definition language (DDL) statements will be of fixed length. If set to false, then columns of this data type will be of variable length. If set to DBNull.Value, it is not known whether will map this field with a fixed-length or variable-length column.
IsFixedPrecisionScale bool If set to true, indicates that the data type has a fixed precision and scale.
IsLong bool

If set to true, indicates that the data type contains very long data.

For , this column is set to true for the image, text, ntext, and xml data types.
IsNullable bool

If set to true, indicates that the data type is nullable. If set to false, indicates the data type is not nullable. If set to DBNull.Value, it is not known whether the data type is nullable.

For , this column is set to true for all data types except the timestamp data type.
IsSearchable bool

If set to true, the data type can be used in a WHERE clause with any operator except the LIKE predicate.

For , this column is set to true for all data types except the image, text, ntext, and xml data types.
IsSearchableWithLike bool

If set to true, the data type can be used with the LIKE predicate.
IsUnsigned bool If set to true, the data type is unsigned. If set to false, the data type is signed. If set to DBNull.Value, this does not apply to the data type.
MaximumScale short If the type indicator is a numeric type, this is the maximum number of digits allowed to the right of the decimal point. Otherwise, this column is set to DBNull.Value.
MinimumScale short If the type indicator is a numeric type, this is the minimum number of digits allowed to the right of the decimal point. Otherwise, this column is set to DBNull.Value.
IsConcurrencyType bool If set to true, the data type is updated every time the row is changed and the value of the column is different from all previous values. If set to false, the data type is not updated every time the row is changed. If set to DBNull.Value, this data type is not supported.
IsLiteralsSupported bool If set to true, the data type can be expressed as a literal.
LiteralPrefix string The prefix applied to a given literal.
LiteralSuffix string The suffix applied to a given literal.

MetaDataCollections

This schema collection exposes information about all of the schema collections supported by .

This schema collection has no schema restrictions or identifier parts.

Column name Data type Description
CollectionName string The name of the schema collection. This value is passed to the GetSchema method of the AmalgaDbConnection object to return the schema collection.
NumberOfRestriction int The number of schema restrictions that may be specified for the schema collection.
NumberOfIdentifierParts int

The number of parts in the composite identifier or object name referenced in the schema collection. For example, in , this column is set to 3 for the Tables schema collection, and 4 for the Columns schema collection.

For more information about schema collections, see schema collections on page 1.

ReservedWords

This schema collection exposes information about the words reserved by .

This schema collection has no schema restrictions or identifier parts.

Column name Data type Description
ReservedWords string The words reserved by .

Restrictions

This schema collection exposes information about the schema restrictions supported by .

This schema collection has no schema restrictions or identifier parts.

Column name Data type Description
CollectionName string The name of the schema collection to which these schema restrictions apply.
RestrictionName string The name of the schema restriction in the schema collection.
ParameterName string

The name of the parameter used for the schema restriction.

For , this column is typically set to "@RestrictionName", where RestrictionName is the value of the RestrictionName column.
RestrictonDefault string

Ignored.

RestrictionNumber int The ordinal position within the string array that represents the schema restrictions, passed to the GetSchema method of the AmalgaDbConnection object.
