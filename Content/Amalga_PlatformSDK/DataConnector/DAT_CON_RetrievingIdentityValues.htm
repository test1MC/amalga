Retrieving automatically generated values

 

This topic describes how to handle automatically generated values while invoking the Update method of the AmalgaDbDataAdapter object in .

Application views and entities in support columns that contain values automatically generated by the underlying storage schema. For example, the entity property used as the primary key for an application view, for which the StoreGeneratedPattern property is set to Identity, is mapped to an identity column used as the primary key column in the corresponding table in the application private database. The identity column contains an automatically generated unique value for each record in the table, set when the record was created, and cannot be changed once set. Knowing the value of the primary key column for a particular row allows you to locate the row that contains it.

During the invocation of the Update method for an AmalgaDbDataAdapter object, the underlying storage schema can send data back to your hosted application as output parameters or as the first returned record of the result set of a SELECT statement executed in the same batch as the INSERT statement. can retrieve these values and update the corresponding columns in the DataRow object being updated.

For more information about retrieving automatically generated values, see Retrieving Identity or Autonumber Values (ADO.NET), on Microsoft Developer Network at http://msdn.microsoft.com/library/ks9f57t0.aspx.
