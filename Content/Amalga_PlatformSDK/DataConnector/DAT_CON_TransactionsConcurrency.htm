Transactions and concurrency

 

This topic describes the support for local and distributed transactions included with .

provides support for local and distributed transactions, using an optimistic concurrency, to combine multiple operations into a single unit of work. If a failure occurs at any point in the transaction, all of the operations can be rolled back to their pre-transaction states.

The AmalgaDbTransaction object in implements the DbTransaction object in ADO.NET, to provide support for local transactions.

For more information about transactions and concurrency, see Transactions and Concurrency (ADO.NET), on Microsoft Developer Network at http://msdn.microsoft.com/library/777e5ebh.aspx.
