Add a new read-only application view

This topic describes how to use the Update Wizard to add read-only application views, based on existing Amalga entities from the , to a virtual schema.

If you have Amalga entities that already have been created in the , then you can automatically create read-only application views for those Amalga entities by using the Update Wizard.

Adding read-only application views to a virtual schema is typically the first action performed with a virtual schema.

The following procedure uses the Update Wizard to connect to the of an Amalga topology,

To add a new read-only application view using the Update Wizard

  1. In Solution Explorer, right-click the data manifest (.edmx) file for the hosted application, and then click Open.
  2. Right-click an empty area in the visual design surface of , and then click Update Model from Database.

  3. If a data connection is not yet associated with the data manifest, click New Connection.

  4. If Data source is not already set to Amalga Data Provider (Data Client), perform the following steps.

    1. Click Change.
    2. In Data source, select Amalga Data Provider.
    3. Click OK.
  5. In Amalga application server address, type the Uniform Resource Locator (URL), including port number, of the data host containing the Entity Registry.
  6. In Log on to the server, select Use Doors Authentication.
  7. In Connect to, select Entity Registry.
  8. Click OK.

  9. If you want to save the data connection in an application configuration (app.config) file for later use, select Save entity connection settings in App.Config as.

    If an app.config file does not exist in the application project, a new app.config file is created for the project and the connection string for the data connection is added to it. Otherwise, the connection string for the data connection is added to the existing app.config file for the application project.

    The username and password keywords are not supported in connection strings for .

  10. Click Next.

    Retrieving information from the entity registry may take some time.

  11. In the Add tab, expand Tables and select the entities from which you want to create the read-only application views.

  12. Click Finish.

    Depending on the number of selected entities, generating a virtual schema may take some time.
