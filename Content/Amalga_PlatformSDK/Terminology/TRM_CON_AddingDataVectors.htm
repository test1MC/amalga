Adding positional data vectors

 

This topic describes how to add positional data vectors for semantic tagging with the Terminology platform application.

Before semantic tags can be applied to data assets in the storage system, a positional data vector must be added for each data asset to be tagged using the Terminology platform application. The positional data vector represents a combination of a source system, a data feed from which data is received, and a data vector, a data asset to which data is stored.

You can use the AddPositionalDataVectors operation, included in the ISemanticLabelTaggingService service contract from the TerminologyService service, to add data vectors for semantic tagging. The operation accepts two parameters:

Parameter name Description
positionalDataVectors

An array of PositionalDataVector objects, representing the data vectors to be added. The operation only adds positional data vectors that do not already exist. If a PositionalDataVector object represents an existing positional data vector, the object is ignored by the operation.
autoAssignSemanticLabel

A Boolean value that, if set to True, indicate that the Terminology platform application should attempt to automatically assign a semantic tag to the new positional data vectors. If the SemanticLabelConceptId property is set to a value, then the operation does not attempt to automatically assign a semantic tag. If the SemanticLabelConceptId property is empty, the operation attempts to map the value of the PositionalTag property for each PositionalDataVector object to the appropriate concept. If a match is found, the value of the SemanticLabelConceptId property is then set to the identifier for the matching concept, and the appropriate semantic tag is automatically assigned to the new positional data vector by the operation.

If the autoAssignSemanticLabel property is set to False, the value of the PositionalTag is ignored and the value of the SemanticLabelConceptId is instead used to assign the appropriate semantic tag to the positional data vector. If the SemanticLabelConceptId property is empty, no semantic tag is set for the positional data vector.

The operation uses the following properties from the PositionaDataVector object to define a new positional data vector.

Property Description
DataVector A DataVector object that represents the data vector for which the positional data vector is created. This object, in combination with the source system specified in the SourceSystem property, identifies the positional data vector.
Description A human-readable description of the positional data vector.
PositionalTag

A string that represents the concept code of the concept to be applied to the positional data vector.

If the autoAssignSemanticLabel parameter is set to True when invokingthe operation, the value of this property is used to identify an appropriate concept. The value of the SemanticLabelConceptId property is then set to the identifier for that concept by the operation.
SemanticLabelConceptId

A string that represents the globally unique identifier (GUID) of the concept to be applied to the positional data vector.

If the autoAssignSemanticLabel parameter is set to True when invoking the operation, this property is set by the operation to the value of the concept identifier for the concept that matches the value of the PositionalTag property.

SourceSystem A SourceSystem object that represents the source system for which the positional data vector is created. This object, in combination with the data vector specified in the DataVector property, identifies the positional data vector.
