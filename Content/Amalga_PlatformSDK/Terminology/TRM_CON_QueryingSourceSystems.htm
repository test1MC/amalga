Querying source systems

 

This topic describes how to retrieve information for source systems stored by the Terminology platform application.

You can use the GetSourceSystems operation, included in the ISemanticLabelTaggingService service contract from the TerminologyService service, to retrieve an array of SourceSystem objects that represent the source systems already registered with the Terminology platform application. The SourceSystem object contains the following properties.

Property Description
Active

If set to True, the source system should be actively included for semantic tagging purposes during data ingestion. Otherwise, the source system is inactive and therefore ignored during data ingestion.
DataType

This property accepts the following SourceSystemDataType enumeration values, which indicate whether the source system allows updates:

  • InsertOnly

    The new source system only allows insert operations.

  • InsertUpdate

    The new source system allows both insert and update operations.
DefaultTimeZone The default time zone used by the source system, for conversion purposes.
Id The globally unique identifier (GUID) used to identify the source system.
Name The name of the data feed represented by the source system.
PreferenceRanking The preference ranking of the data feed, compared to other data feeds in the hospital.
PreferredCulture The preferred culture in which to find the semantic tags for the clinical terminology specified in the TerminologyName property.
Provider The provider of the data feed represented by the source system.
TerminologyName The name of the clinical terminology referenced by the source system.
Type The type of data feed represented by the source system.

Example

The following example uses the GetSourceSystems operation from the ISemanticLabelTaggingService service contract to retrieve an array of SourceSystem objects, representing existing source systems, from the Terminology platform application.

public SourceSystem[] GetSourceSystems() 
{
    // Use the GetSourceSystems operation from ISemanticLabelTaggingService
    // to retrieve an array of SourceSystem objects.

    // First, resolve the service endpoint and connect to the service.
    // Resolve the service endpoint.
    ServiceEndPoint terminologyEndPoint = HostedServiceResolver.ResolveServiceEndPoint(
       (ApplicationDependency)ApplicationConfiguration.Dependencies["TerminologyApplicationDependency"],
       new Guid("8c3be61d-021b-419a-ad9a-4c74b7b115ea"),
       "Microsoft.Amalga.Terminology.ServiceContracts.ISemanticLabelTaggingService");

    // Open the client channel and instantiate the WCF client.
    ISemanticLabelTaggingService terminologyServiceClient =
       AmalgaWcfChannelFactory<ISemanticLabelTaggingService>.CreateChannel(terminologyEndPoint);

    // Invoke the operation from the service.
    return terminologyServiceClient.GetSourceSystems();

}
