Data vectors

 

This topic describes data vectors and how the Terminology platform application interacts with them for semantic tagging purposes.

In data ingestion terms, a data vector identifies the physical location of a data asset in the storage system. In order of precedence, the following information in combination uniquely identifies a data vector:

  • The network address of the data host.
  • The instance name.
  • The database name.
  • The schema name.
  • The table or view name.
  • The column name.

Positional data vectors

The Terminology platform application uses positional data vectors to identify data assets for semantic tagging purposes. The positional data vector represents a combination of a source system, a data feed from which data is received, and a data vector, a data asset to which data is stored. For more information about source systems, see Source systems on page 1.

Once a positional data vector is identified and added to the Terminology platform application, a semantic tag can be applied to it for discoverability and organization purposes. You can use the module included with to manually apply semantic tags to data assets. For more information about adding semantic tags to data assets using , see dialog box on page 1.

You can programmatically retrieve information about existing positional data vectors, add new positional data vectors, and update semantic tags using operations provided by the ISemanticLabelTaggingService service contract from the TerminologyService service.
