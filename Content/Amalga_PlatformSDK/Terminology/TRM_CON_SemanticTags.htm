Semantic tags

 

This topic describes semantic tags, string representations of the concepts managed by the Terminology platform application.

In , a semantic tag is a fully qualified string representation of a concept in a clinical terminology included with the Terminology platform application. Semantic tags allow you to add concepts as metadata to data assets in Amalga, such as entities or entity properties. Semantic tags facilitate organization and search, as well as to improve performance. You can apply multiple semantic tags to the same data asset, as well. When multiple semantic tags are applied to a data asset, such as an entity or entity property, the definitive meaning of the data asset is the logical disjunction of all tags; that is, the set of semantic tags for the data asset are treated as a logical OR operation.

While semantic tags are not required for entities or entity properties in , you can take advantage of some of the more sophisticated functionality, such as merges or access inheritance, by applying the appropriate semantic tags. You can either use the module included with to manage semantic tags, or you can programmatically apply semantic tags using the ISemanticLabelTaggingService service contract, included in the TerminologyService service for the Terminology platform application. For more information about programmatically applying semantic tags, see Updating semantic tags on page 1.

Semantic tag components

An semantic tag is fully qualified when it contains the components indicated in the following image. Each component in a semantic tag is separated by a caret (^) character.

Components of fully qualified semantic tags in : 

Component Description
Terminology The human-readable description of the clinical terminology that contains the concept identified by the semantic tag.
Code The code for the concept identified by the semantic tag.
Meaning The human-readable description for the concept identified by the semantic tag.
Identifier The globally unique identifier (GUID) for the concept identified by the semantic tag.

A semantic tag does not identify a clinical concept, but instead identifies a particular concept that represents a clinical concept stored in a clinical terminology. Depending on the number of supported languages in a clinical terminology, a clinical concept can be represented by one concept per supported language. The semantic tag, by extension, identifies a concept that represents a clinical concept in a clinical terminology for a specific supported language.

Applying semantic tags

Semantic tags can be applied to source systems, which allows clinical concepts to be mapped to new columns at the point of data ingestion, or to data vectors, which allows clinical concepts to be mapped to existing columns in the storage system.

In data ingestion terms, a source system identifies a specific incoming data feed. The Terminology platform application can map a source system to a clinical terminology for data ingestion purposes, allowing clinical concepts in incoming messages to be consistently and accurately identified when messages from the source system are shredded. For more information about applying semantic tags to source systems, see Source systems on page 1.

Similarly, a data vector identifies a specific table or column in a database. The Terminology platform application can map a data vector to a clinical concept for discoverability and organization purposes, allowing clinical concepts in data assets to be consistently and accurately identified in the storage system. For more information about applying semantic tags to data vectors, see Source systems and data vectors on page 1.
