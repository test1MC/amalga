Concepts

 

This topic describes concepts, which represent clinical concepts within a terminology in the Terminology platform application.

A clinical concept is a clinical idea to which a code unique to the clinical terminology that contains the clinical concept, is associated. In , each clinical concept is represented by one or more concepts.

Each concept has the following properties:

  • Terminology

    Each concept is contained in a single clinical terminology. For more information about terminologies, see Terminologies on page 1.

  • Concept code

    Each concept has a concept code that identifies the clinical concept within a given clinical terminology. For example, the concept code PatientID is assigned to the concept that represents a patient identifier in the Microsoft Beta Org CS terminology. For more information about concept codes, see .

  • Concept code type

    Each concept has a concept code type, which can be used to further classify related concepts within a given clinical terminology.

  • Meaning

    Each concept has a meaning, a human-readable description of the clinical idea represented by the concept, specific to the language supported by the concept.

  • Supported language

    A single clinical terminology can support multiple languages. While the concept code for a given clinical concept does not change, the meaning can change depending on the languages supported by the clinical terminology. Multiple concepts, each in a different supported language, can represent the same, single clinical concept within a single clinical terminology. For example, if a single clinical terminology supports three languages, then there are three concepts, one for each supported language, that represent the same clinical concept. All three concepts have the same concept code, but different, language-specific meanings.

Concept codes

Within a given clinical terminology, each clinical concept has an associated concept code that uniquely identifies that particular clinical concept. For example, the clinical concept for benign essential hypertension, a diagnosis, is represented in the Systematized Nomenclature of Medicine - Clinical Terms (SNOMED CT) clinical terminology by the concept code, 1201005.

Each concept has a unique identifier associated with it that ensures the concept is unique in the terminology database. In turn, each concept also has a concept code associated with it that uniquely identifies the clinical concept within the clinical terminology to which it belongs. If a clinical concept supports multiple languages, there may be multiple concepts in the terminology database that support a single clinical concept for a single clinical terminology.

Concept codes may not be unique to a particular terminology. Two different clinical terminologies can use the same concept code value to represent different clinical concepts, depending on the coding system used by the clinical terminology.

Concept code types

A concept code type groups related clinical concepts. For example, all concepts for the RxNorm terminology are associated with the RXNORM Code concept code type. Concept code types are specific to the terminology that contains the concepts for which the concept code type is used.

While a single terminology can support multiple concept code types, most clinical terminologies use only a single concept code type. For example, all concepts contained by the SNOMED CT clinical terminology use the SNOMED Concept ID concept code type.

Root concepts

In a terminology that contains hierarchical relationships, a root concept is a top-level concept that represents a generalized clinical concept for a terminology hierarchy. For example, the SNOMED CT Concept root concept represents the top-level concept to which all other, more specialized concepts are related within the SNOMED CT terminology. For more information about relationships between concepts, see Relationships on page 1.

You can quickly query root concepts without negotiating the relationships between concepts for a given terminology, either by using the ITerminologyQueryService service contract or the Microsoft.Terminology.Beta.RootConcept entity. For more information about querying root concepts, see Querying concepts on page 1.
