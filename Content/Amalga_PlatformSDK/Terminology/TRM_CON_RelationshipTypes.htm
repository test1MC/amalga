Relationship types

 

This topic describes relationship types, which describe the relationships between concepts in the Terminology platform application.

In the Terminology platform application included with , relationships between concepts are described by a relationship type. The relationship type describes the characteristics of the relationship, such as whether the relationship is hierarchical, and provides a display name which can be used when graphing relationships between concepts.

Relationship types are specific to a given terminology. For example, the relationship types included with the SNOMED CT clinical terminology provide defining and qualifying characteristics, which allow concepts to be used as attributes for other concepts, or historical relationships, which allow concepts to be versioned or inactivated.

Relationship type attributes

Relationship type attributes define the characteristics of the relationship type. Multiple relationship type attributes can apply to a single relationship type. For example, the Resolved By relationship type, included with the HLI Core terminology, describes an immutable, reciprocal, ordered, displayable relationship.

The following relationship type attributes can be applied to a single relationship type:

  • Acyclic

    If set to True, this relationship type attribute indicates that the relationship type describes an acyclic relationship, such as a tree relationship, between two concepts. Otherwise, the relationship type can describe a cyclic relationship.

    For example, the "Next Version" relationship type, included with the "HLI Core" terminology, describes a typical acyclic relationship.

  • Displayable

    If set to True, this relationship type attribute indicates that this relationship type can be displayed. Otherwise, the relationship type should not be displayed.

    For example, all relationship types included with the "HLI Core" terminology are displayable.

  • Hierarchical

    If set to True, this relationship type attribute indicates that the relationship type describes a hierarchical relationship between two concepts. Otherwise, the relationship type describes a non-hierarchical relationship.

    For example, the "Subsumes" relationship type, included with the "HLI Core" terminology, describes a typical hierarchical relationship.

  • Immutable

    If set to True, this relationship type attribute indicates that the relationship type describes a relationship that, once referenced, cannot be changed. Otherwise, the relationship can be changed.

    For example, the "Dependency" relationship type, included with the "HLI Core" terminology, describes a typical immutable relationship.

  • InheritanceRelation

    If set to True, this relationship type attribute indicates that the relationship type describes an inheritance relationship.

    For example, the "RXNORM Constitutes" relationship type, included with the "HLI RXNORM" terminology, describes a typical inheritance relationship.

  • Ordered

    If set to True, this relationship type attribute indicates that the relationship type describes an ordered relationship.

    For example, the "Resolved By" relationship type, included with the "HLI Core" terminology, describes a typical ordered relationship.

  • Reciprocal

    If set to True, this relationship type attribute indicates that the relationship type describes a reciprocal relationship.

    For example, the "Validated By" relationship type, included with the "HLI Core" terminology, describes a typical reciprocal relationship.

  • Transitive

    If set to True, this relationship type attribute indicates that the relationship type describes a transitive relationship between concepts. If concept A has a transitive relationship with concept B, and concept B has a transitive relationship with concept C, then concept A has an implied transitive relationship with concept C.

    For example, the "Subsumes" relationship type, included with the "HLI Core" terminology, describes a typical transitive relationship.
