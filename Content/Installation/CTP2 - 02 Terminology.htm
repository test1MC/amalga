Terminology

The following are a list of terms commonly used during installation.

To be reviewed for clarity and completeness post tech review

Term Definition
Amalga hosted application

Applications that are packaged, deployed, monitored and managed by . Amalga hosted applications are all characterized by an application manifest that describes the necessary components of an application in order to host it in .
Amalga hosted service

supports service oriented architectural principles. In particular, hosted applications can contain a collection of hosted services on an Amalga platform, providing a set of different operations available to remote users, through direct access or a client.
application views The components of an application's that describe the application's required data model. An application view can be either read-only to the published entities in the , or read-write to the application's private data.
EDMX file An XML file that defines a conceptual model, a storage model, and the mapping between these models. An .edmx file also contains information that is used by the ADO.NET Entity Data Model Designer (Entity Designer) to render a model graphically.
entity An object that groups ontology elements (concepts) into meaningful structures (for example, Patient, LabResults) uniquely identified by a set of primary keys.
platform components

The basic unit of the platform, including databases, services, or tools.
platform features

Collection of services and a corresponding workflow process for orchestrating the interaction with each of its services, to combine multiple different basic [service] datasets into a more complex/valuable dataset.

platform service

Provides a specific set of functionality to read and/or write a dataset, as well as provide a set of functional APIs specific to that dataset.
roll-forward recovery The recovery model used by the Setup utility. Components that are successfully installed do not need to be rolled-back in order to rerun the failed Setup utility command and complete the installation.
Security Token System (STS) A web service that issues security tokens.
topology

The set of services, applications, features, logical hosts and physical hosts that are registered in an database (the main topology store).

The goal of the virtual schema is to empower application writers to create portable applications, which will be able to leverage semantic tools to assist the deployment of applications across customer sites using different data schemas.

An application’s is a map of concepts and their relationships to describe the data model of a specific application. An application virtual schema is defined by a developer as a set of application views and their associations, describing the data needed for the application to function.
