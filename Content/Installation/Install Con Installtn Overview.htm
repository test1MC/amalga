Installation overview

This section summarizes the installation tools, the installation procedures required, and their sequence.

The installation process was designed with the goals of serviceability and simplicity. Because the installation is enabled by the architecture and uses the same terminology to describe key installation concepts, we recommend that you thoroughly review Amalga Platform Development Architecture before planning for installation. For short descriptions of key terms, see Terminology on page 1. Serviceability capabilities include the following features.

Self-Describing System

is self-describing, meaning that all components of an implementation are catalogued in the system database. Features, services, databases, logical configuration and physical configuration are all tracked, not only during installation but also during subsequent management and upgrade activities. This self-describing capability enables administrators to focus on managing generic pools of services, applications and features rather than maintaining detailed records about the environment. It also facilitates a much simpler installation process as the system can provide many of the required parameters from its own repository rather than being dependent on detailed input from the administrator.

Automated Prerequisite Management

Prerequisite software and system settings can now be checked, and in some cases configured, by the Setup utility. During Setup execution, if a prerequisite is missing, a plain language error is displayed calling out the required action. Because the Setup utility is based on a roll-forward recovery model, the prerequisite can be provided and the Setup command run again. It will begin processing at the point of the prerequisite error and complete the installation process without starting over.

Platform and Application Installation Separation

Once the platform is in place, additional applications can be installed through the , which provides installation variable selections and a guided path through the application installation by referencing the topology database. Application deployment can also be automated through Doors PowerShell.

Installation process

The following procedures performed in sequence will install the platform.

Procedure Optional Description Page
Design the topology No Create an topology that will support enterprise response times and availability goals. 1
Confirm installation prerequisites No Install and configure the physical machines and virtual machines to meet the requirements of the topology design, hardware and software prerequisites. 1
Install the platform No Run Setup with appropriate operations and parameters appropriate for the target topology to install the core platform and application services. 1
Install applications No Run Doors PowerShell commands to install hosted applications on top of the platform. 1

Uninstallation

For , uninstallation of features is not supported. To uninstall an application instance, see Remove an application instance.

Setup utility

The platform is installed by using the Setup utility. The Setup utility is run from a command prompt and requires three types of input.

  1. Arguments: Primarily the features on which the operations will be performed. Also includes options to receive help and indicate silent acceptance of the end user license agreement.
  2. Operations: The actions taken for the specified feature.
  3. Parameters: Feature specific input required to perform the operations.

For detailed information on the Setup utility and the required input, see Setup Command Reference on page 1.

Setup is fault-tolerant and provides roll-forward recovery for a given feature or operation. If an error occurs while running Setup, you can resolve the cause of the error and run the same Setup command again to continue deployment. For more information about fault tolerance and roll-forward recovery, see the Setup Command Reference on page 1.
