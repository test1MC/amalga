Complete the Application Installation

Before doing the steps in this section, you must first install the following three applications: DataExplorerPrivate, DataExplorer, PatientChart. Use the steps found in the Install applications section to do this.

After finishing the installation steps above, continue the application installation by following the procedures in this section. For additional information on the commands used in this section, see Amalga Doors PowerShell reference.

To complete the installation

  1. Prepare the environment.
    1. Log on to the target server.
  2. Run the following cmdlet.

    Import-Module 'C:\Program Files\Microsoft Amalga\System 3.0\Platform\PowerShell\Microsoft.Amalga.PowerShell.dll'

    Add-AmRoleToAccessLevel -ApplicationOrFeatureInstanceName [DataExplorer InstantceName] -AccessLevelName DataExplorerUser -Roles BUILTIN\AmalgaUsers

    Add-AmRoleToAccessLevel -ApplicationOrFeatureInstanceName [DataExplorerPrivate InstanceName] -AccessLevelName DataExplorerPrivateUser -Roles BUILTIN\AmalgaUsers;

    Add-AmRoleToAccessLevel -ApplicationOrFeatureInstanceName [DataExplorerPrivate InstanceName] -AccessLevelName DataExplorerPrivateReportAuthor -Roles BUILTIN\AmalgaUsers;

    Add-AmRoleToAccessLevel -ApplicationOrFeatureInstanceName [DataExplorerPrivate InstanceName] -AccessLevelName DataExplorerPrivateAdmin -Roles BUILTIN\AmalgaUsers;

    Add-AmRoleToAccessLevel -ApplicationOrFeatureInstanceName [DataExplorerPrivate InstanceName] -AccessLevelName DataExplorerPrivateReportAuthor -Roles ReportAuthor

    Add-AmRoleToAccessLevel -ApplicationOrFeatureInstanceName [PatientChart InstanceName] -AccessLevelName PatientChartUser -Roles BUILTIN\AmalgaUsers;

    Add-AmRole -RoleName ReportAuthor -DisplayName "Report Author"

    Add-AmRoleMembership -RoleName ReportAuthor -UserAccount "domain\username" -DirectoryService MASTER

    Add-AmRoleMembership -RoleName BUILTIN\AmalgaService -UserAccount "domain\username" -DirectoryService MASTER

includes the following four entities, which must be bound. For information about how to do binding, see Entity binding overview.

  • DictationDocument
  • PathologyDocument
  • RadiologyDocument
  • VitalSign
