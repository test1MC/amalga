Create a PPE application

 

You need to create an application ID in the PPE to use for the testing and validation of your initial Connection installation and customizations. The following procedure creates that application ID.

As part of that procedure. you will need to upload a public key certificate file (.cer). However, in order to create the certificate, you must have the application ID provided by the Application Configuration Center (ACC). So, before you start the application ID process, make sure you are prepared to create the certificate at the same time.

The pre-production environment (PPE) is not secured at the same level as the production version of . The operations team recommends that de-identified patient data be used for testing and validation of the Connection in the PPE. If de-identified data is not available, see Install PRoc HVConnect RD.

To create a PPE application ID

  1. Open a browser and navigate to HealthVault Application Configuration Center (ACC) (http://go.microsoft.com/fwlink?LinkId=209977) to create and register a new application.

    A account is required to sign in. A Doors Live ID can also be used to create the account.

  2. On the ACC Welcome page, under Applications, click Create a New Application.
  3. On the ACC Application page, make a note of the application ID.

  4. Now, using that application ID, create your certificate. You should follow your established process of generating a certificate. If such a process does not exist, generate a self-signed certificate using the MakeCert.exe (http://go.microsoft.com/fwlink?LinkId=209978) tool. Run the MakeCert command on the machine where Connection will be deployed for development and test. Make sure to use the application ID noted in step 3.

    MakeCert syntax

    makecert.exe "<OutputPath>\<CertFileName>.cer" -a sha1 -n "CN=NewApp-<HVAppId>" -sr LocalMachine -ss My -sky signature -pe -len 2048

    MakeCert example

    makecert.exe "c:\temp\MyCert.cer" -a sha1 -n "CN=NewApp-6296418d-a6c7-418d-84ea-f4c04b9dd1b6" -sr LocalMachine -ss My -sky signature -pe -len 2048

  5. Back on the ACC Application page, enter an application name (for example, “My Hospital Patient Portal”). This name will be shown to the patient when they authorize Connection for access to .
  6. Select Web as the Application Type.
  7. Browse to the public key certificate file (.cer) that you just created and enter the file name in the Public Certificate text box.
  8. Review and accept “Microsoft HealthVault Developer Site Terms of Use” and click Create application.

The application is now registered in the PPE environment.
