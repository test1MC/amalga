Install the Front End

The Client Applications feature performs the following actions:

  • Uses current credentials for administration
  • Installs the WSPs to solution store (in this order)
    • Microsoft.Health.Amalga.AppCore.wsp
    • Microsoft.Amalga.AmalgaDataExplorer.wsp
    • Mcrosoft.Amalga.PatientChart.wsp
  • Deploys Microsoft.Amalga.Framework.wsp to the web app on port 8080 if it is not already there
  • Deploys the other Amalga WSPs to the web app on port 8080
  • Provisions the site at https://fqdn:8080/sites/Amalga

To install the application

  1. Verify that is installed and available on port 8080.

    8080 is the default port for .

    The installation site should be on: https://fqdn:8080/sites/amalga. (Replace fqdn with your fully qualified domain name.)

  2. Make sure the Amalga Physical Host service is started.
    1. From the Start menu enter Computer Management in the search dialog.
    2. Click Computer Management.
    3. In the Computer Management application, expand the Services and Applications branch.
    4. Click Services.
    5. Locate Amalga Physical Host Service and verify whether Status is set to Started.
      1. If the Amalga Physical Host service is not started, right-click it.
      2. Click Start.
  3. Create a root-level folder on the host machine titled c:\DataExplorerInstallation.
  4. Copy the installation files from your installation media into the folder.
  5. Open a Command Prompt with administrator level permissions.
  6. Change directories to the c:\DataExplorerInstallation\ created in step 3.
  7. Run setup.exe /Feature ClientApplications /install.

Setup.exe has two supported switches for the release:



/AmalgaDataFQDN (Optional) allows you to identify the fully qualified domain name of your server. (Default Value = '$env:ComputerName.$env:UserDnsDomain')



/AmalgaDataPort (Optional) allows you to specify the data connection port for your installation. (Default value = '8889')

If your Data server is on a different computer than , use the /AmalgaDataFQDN switch to specify the location of the Data server fully qualified domain name.

For example: setup.exe /Feature Client Applications/AmalgaDataFQDN fqdn/install. (Note that the fully qualified domain name in this example is the data server's FQDN.)

To create a site

Ensure you are logged in using a administrator account before using the following steps

  1. Open an supported internet browser
  2. Navigate to the default web page. For example:

    http://AmalgaPlatform.ContosoHospital.com:8080/sites/default.aspx

  3. When the web site loads, click on the carrot to the right of the administrator account name at the top of the screen.
  4. Click 'Site Actions' and choose 'New Site'
  5. From 'All categories' select the 'Amalga' category.
  6. Choose 'Internal Provider Site' from the site definitions.
  7. Enter a title and URL name for the new site. For these instructions we will use 'DataExplorerSite'.
  8. Click 'Create'
  9. You should be redirected to the newly created site. In our example the URL for the site would be:
  10. You should be redirected to the newly created site. In our exmplple the URL for the site would be:

    http://AmalgaPlatform.ContosoHospital.com:8080/sites/DataExplorerSite/default.aspx

  11. Click the carrort to the right of the adminstrator account name at the top of the screen.
  12. Click 'Site Actions' and choose 'Site Settings'
  13. Click 'Manage Site Features' under the 'Site Actions' icon.
  14. Scroll down to 'Microsoft Amalga solution Root feature.
  15. Click 'Deactivate'
  16. On the confirmation page, click 'Deactiveate this feature'.
  17. On the 'Manage site features' page click confirm.
  18. On the features page, click 'Sample Site'.
  19. Click 'Sample Site.
  20. Click 'Patient Lists' to be redirected to the Application Binding' page.
  21. From the drop down box on the 'Application Binding page, choose 'DataExplorerPrivate'
  22. On the confirmation page, click 'Change'.
  23. Click 'Click here' to be redirected to the Patients List page.
  24. The newly created site is now ready for functionality.

 

 

To verify the installation

  • Ensure your installation was completed successfully by opening the following URL, https://fqdn:8080/sites/Amalga, to verify that the site is running.

    You can ignore any certificate warnings you might see.
