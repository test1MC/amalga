Create a RD application

 

Organizations that do not have de-identified data for testing purposes can request access to a Rapid Deployment (RD) environment. The RD environment is a secure environment that is accessible only by the requesting organization.

To request a RD environment

An RD environment is hosted by the operations team and hosting fees will apply while the RD environment is maintained.

  • Consult with the Hosting Team to obtain the latest hosting fee schedule.
  • Contact Operations via HSG Help (http://go.microsoft.com/fwlink?LinkId=209970) to request a RD environment for your organization.
  • The Operations Team will provide a private URL for the installation’s RD environment.
  • Once the URL is received, you can log on to the Action Control Center (ACC) associated with the installation’s private RD Environment.

As part of the following procedure you will have to upload a public key certificate file (.cer). However, to create the certificate, you must have the application ID provided by the ACC. Before you start the application ID process, make sure you are prepared to create the certificate at the same time.

To create a rapid deployment application ID

A account is required to sign in to the ACC for the RD environment.

  1. Go to the private URL provided by the operations team and log on to ACC to create and register a new application.

    A account is required to sign in. A Doors Live ID can also be used to create the account.

  2. On the ACC Welcome page, under Applications, click Create a New Application.
  3. On the ACC Application page, make a note of the Application ID.

  4. Now, using that application ID, create your certificate. You should follow your established process of generating a certificate. If such a process does not exist, generate a self-signed certificate using the MakeCert.exe tool. Run the MakeCert command on the machine where Connection will be deployed for development and test. Make sure to use the application ID noted in step 3.

    MakeCert syntax

    makecert.exe "<OutputPath>\<CertFileName>.cer" -a sha1 -n "CN=NewApp-<HVAppId>" -sr LocalMachine -ss My -sky signature -pe -len 2048

    MakeCert example

    makecert.exe "c:\temp\MyCert.cer" -a sha1 -n "CN=NewApp-6296418d-a6c7-418d-84ea-f4c04b9dd1b6" -sr LocalMachine -ss My -sky signature -pe -len 2048

  5. Back on the ACC Application page, enter an application name (for instance, “My Hospital Patient Portal”). This name will be shown to the patient when they authorize Connection for access to .
  6. Select Web as the Application Type.
  7. Browse to the public key certificate file (.cer) that you just created and enter it in the Public Certificate text box.
  8. Review and accept “Microsoft HealthVault Developer Site Terms of Use” and click Create application.

The application is now registered in the RD environment. Use ACC to configure offline rules for the application to work with standard configurations. Allergy, Blood Glucose, Blood Pressure, Continuity of Care Record (CCR), File, and Medication should be selected.
