Prepare database host

Perform each of these steps in order to successfully create an Database Server. It is not necessary to repeat these steps on any other server.

Configure Doors Firewall

  1. Open Doors Server Manager
  2. Expand 'Configuration'
  3. Expand 'Doors Firewall with Advanced Security'
  4. Right click 'Inbound Rules'
  5. Click 'New Rule'
  6. Select 'Port' and click next
  7. Ensure 'TCP' is selected
  8. Select 'Specific local ports'
  9. Enter '8889, 443, 9999, 1433' in the text box and click next
  10. Ensure 'Allow the connection' is selected and click next
  11. Ensure 'Domain', 'Private', and 'Public' are all selected and click next
  12. In the 'Name' box type 'Amalga Inbound Exceptions' and click Finish

Install

This is the only logical host on which must be installed.

 

  1. Follow the instructions associated with to complete this step.
  2. Ensure SQL Server is using Doors Authentication
  3. Both the administrator and 'Network Service' must have administrative rights

Attach the HLI database

  1. Navigate to http://connect.microsoft.com and navigate to the Amalga download page.
  2. Download the Health Language International (HLI) database .zip file
  3. Extract the zip file to a folder on the database logical host
  4. Open
  5. Log in to the Database logical host
  6. Right click 'Databases'
  7. Click 'Attach
  8. Click 'Add'
  9. Navigate to the folder in which you have copied the HLI
  10. Highlight the HLI.mdf file and click 'OK'
  11. Delete the log file entry (*.ldf)
  12. Click OK
