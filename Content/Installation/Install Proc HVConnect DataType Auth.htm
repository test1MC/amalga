Connection authorization configuration

 

This step is optional.

Use Application Configuration Center (ACC) to configure offline rules for the application to work with standard configurations. Enable the data types that the application will be supporting.

  • Application authors must decide which data types are required and which are optional.
  • As part of the "go-live" process, application authors must provide a "why" string for each of the data types for the application to authorize. This information gives the user a better understanding of the benefit that they get by granting access.
  • Applications cannot assume that they have access to a specific data type in a specific user's record—they need to query the platform to see what access has been granted.

For more information and the procedures to set authorization parameters, see Optional Authorization (http://msdn.microsoft.com/en-us/healthvault/cc539985.aspx).
