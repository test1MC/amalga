Connection prerequisites

 

A P application ID and a certificate for communication with must be obtained prior to deploying Connection. The application should be registered in the pre-production environment (PPE) for development and test purposes. If Personally Identifiable Information (PII) is involved, please request a Rapid Deploy (RD) environment. 

Similar to HealthVault PPE, Rapid Deploy applications are created and registered through Application Configuration Center (ACC). There is a separate instance of ACC that can be pointed to RD for this purpose. The steps are the same for both PPE and RD environments.

To prepare for Connection installation

  1. Perform one of the following. PPE in most cases; RD if personally identifiable patient data will be used.
    1. Create a PPE application
    2. Create a RD application
  2. Connection authorization configuration (optional)

  3. If you want to digitally sign the data before pushing it to , you will need a signing certificate. Obtain a signing certificate from one of the commercial third party certificate providers.
