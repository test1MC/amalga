Add-AmRoleToAccessLevel

 

The Add-AmRoleToAccessLevel cmdlet grants permissions for an application and access level to one or more roles.

Amalga built-in role membership required: AmalgaAppAdmin or AmalgaSysAdmin

For more information, see Built-in roles.

Syntax

Add-AmRoleToAccessLevel -ApplicationOrFeatureInstanceName <String> -AccessLevelName <String> -Roles <String>

Parameter descriptions

Example

Add-AmRoleToAccessLevel -ApplicationorFeatureInstanceName LabApp1 -AccessLevelName Writer -Roles DomainName\LabAdmin,DomainName\Manager
