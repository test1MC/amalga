Binding element

 

This topic describes the Binding element.

Represents an entity binding for a single entity.

Syntax

<Bindings>
   <Binding
      EntityName = 'string"
      EntityNamespace = "string"
      [ Materialize = boolean ] >
BindingSQLStatement
   </Binding>
</Bindings>

Element characteristics

Characteristic Description
Data type and length CDATA section
Default value None
Cardinality 1-n: Required element that can occur one or more times.

Element relationships

Relationship Element
Parent elements Bindings
Child elements None

Attributes

Attribute Description
EntityName

(Required string.) The name of the entity to be bound.
EntityNamespace

(Required string.) The name of the entity namespace containing the entity specified in the EntityName attribute.
Materialize

(Required Boolean.) Value = true.

Materialization is required for the current version of . This attribute must be used, and it must be set to true.

Remarks

The Binding element identifies a single entity and contains the binding SQL statement, a Transact-SQL SELECT statement, used to bind an entity to the underlying databases, tables, and columns in the storage schema.

BindingSQLStatement must be replaced with a CDATA section containing a valid Transact-SQL SELECT statement, with fully qualified table and view names, conforming to the definition of the entity specified in the EntityName attribute.

Example

The following example illustrates a Binding element for an entity, named Entity1, in the entity namespace. The Entity1 entity binding is configured for materialization.

<Binding 
   EntityName='Entity1' 
   EntityNamespace='Common.Beta2'
   Materialize=true>
   <![CDATA[
      SELECT [EID]
         ,[FName]
         ,[LName]
         ,[SSN]
         ,[DOB]
         ,[DOB_o]
      FROM [AzADT].[dbo].[HIS601]
   ]]>
</Binding>
