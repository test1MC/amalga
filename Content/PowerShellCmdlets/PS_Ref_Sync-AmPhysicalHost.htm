Sync-AmPhysicalHost

 

The Sync-AmPhysicalHost cmdlet queries the topology service, downloads all the new changes to the physical host and applies them. These changes include the following actions.

  • Downloading new services and hosting them
  • Uninstalling services from the physical host

For Doors Worker services, if Sync-AmPhysical Host has new changes for the service, the service needs to be in a Stopped state before running Sync-AmPhysicalHost.

Amalga built-in role membership required: AmalgaSysAdmin

For more information, see Built-in roles.

Syntax

Sync-AmPhysicalHost [-ServiceInstanceName <String>] [-Force]

Parameter descriptions

[-ServiceInstanceName] <string>

Takes the unique instance name as recorded in the topology database and included in the resolved application or service manifest used during the Publish-AmApplicationInstance process.

Characteristic Value
Required? No
Position Named
Default value None
Accepted values Instance name as recorded in the AmalgaSystem database
Accept pipeline input? Yes
Accept wildcards? No

[-force]

This parameter takes no values. When this parameter is specified, services are forced to synchronize on the physical host, regardless of the requirement.

Characteristic Value
Required? No
Position Named
Default value None
Accepted values NA
Accept pipeline input? No
