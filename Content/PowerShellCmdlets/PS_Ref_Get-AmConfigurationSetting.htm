Get-AmConfigurationSetting

 

Get-AmConfigurationSetting returns the list of configuration settings for an application, service, feature or component.

Amalga built-in role membership required: AmalgaSysAdmin

For more information, see Built-in roles.

Syntax

Get-AmConfigurationSetting -InstanceType <ConfigurationSettingInstanceType> -InstanceName <String> [-SettingName <String>]

Parameter descriptions

-InstanceType <string>

Takes the type of the instance.

Characteristic Value
Required? Yes
Position Named
Default value None
Accepted values ApplicationInstance

ServiceInstance

FeatureInstance

ComponentInstance
Accept pipeline input? False
Accept wildcards? No

-InstanceName <string>

Takes the name of the instance of the type specified by the InstanceType parameter.

Characteristic Value
Required? Yes
Position Named
Default value None
Accepted values Instance names, such as Terminology_Inst1 (application instance), or externalregistry (feature instance).
Accept pipeline input? False
Accept wildcards? No

[-SettingName] <string>

Takes the setting name to return for the instance. Only one name can be specified. If omitted, all settings are returned.

Characteristic Value
Required? No
Position Named
Default value None
Accepted values Setting value as a string.
Accept pipeline input? False
Accept wildcards? No
