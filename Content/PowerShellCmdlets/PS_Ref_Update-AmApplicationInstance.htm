Update-AmApplicationInstance

 

The Update-ApplicationInstance cmdlet updates an application instance with the the contents of an application cab file. The TopologyNode, PropertyData and PropertyDataBinary properties are updated with new values.

Amalga built-in role membership required: AmalgaSysAdmin, AmalgaAppAdmin

For more information, see Built-in roles.

Syntax

Update-AmApplicationInstance -ApplicationCabFile <String> -UpdateApplicationInstance <String> [-Rollback] [-SkipSigningCheck]

Parameter descriptions

-ApplicationCabFile <String>

The Local or UNC path for the application cab file containing the application update.

Accept pipeline input? False

Characteristic Value

The application instance name to be updated as a string.
Accept pipeline input? False
Accept wildcards? No

[-Rollback]

This parameters is used when you want to roll back an application to a previous version. Rollback logic ensures that the application in the provided cab file is an earlier (lower) version than the installed version.

To be used as the source for an application rollback, an application cab file must meet the following requirements.

  • Only non-manifest changes
  • No dependency changes or new configuration settings
  • No virtual schema changes
  • No new services, or removal of previous services

Characteristic Value
Required? No
Position Named
Default value None
Accepted values None
Accept pipeline input? False
Accept wildcards? No

[-SkipSigningCheck]

This option allows applications to be updated without checking the cab signature. This option is useful for testing purposes.
