Get-AmServiceEncryptionCertificate

 

The Get-AmServiceEncryptionCertificate cmdlet gets the service encryption certificate associated with the endpoints of an service or component. This certificate is also used as the token encryption certificate in .

The SSL certificate of the logical host to which a service is assigned is used as the service and token encryption certificate.

Syntax

Get-AmServiceEncryptionCertificate -InstanceName <String> -InstanceType <String>

Parameter descriptions

[-InstanceName] <string>

Takes the name of the service or component instance.

Required? Yes
Position Named
Default value None
Accepted values

Name of an service or component instance.
Accept pipeline input? False
Accept wildcards? No

[-InstanceType] <string>

Takes the instance type of the item.

Required? Yes
Position Named
Default value None
Accepted values

ServiceInstance

ComponentInstance
Accept pipeline input? False
Accept wildcards? No

Examples

Get-AmServiceEncryptionCertificate -InstanceName TopologyService -InstanceType ComponentInstance

Get-AmServiceEncryptionCertificate -InstanceName DataConnectorService -InstanceType ServiceInstance
