Unregister-AmDataHostLabel

 

In the context of , a data host is a instance which is meant to hold Application private databases. A data host can have multiple labels assigned so that administrators have a more user friendly way to identify the data host. For example, a data host instance might have the operational name "SQLHost91". That same host could have multiple labels such as "AllergyApplicationPrivateData", or "VistingDoctorApplicationData" so that the instance's use would be more apparent to an administrator. This cmdlet unregisters the specified label from the associated data host.

Amalga built-in role membership required: AmalgaSysAdmin

For more information, see Built-in roles.

Syntax

Unregister-AmDataHostLabel -ServerInstance <String> -Label <String>

Parameter descriptions

-ServerInstance <String>

The instance which has the label being unregistered.

Characteristic Value
Required? Yes
Position

Named
Default value None
Accepted values machine name, or a machine name and instance name combination: SQLMachine2, or SQLMachine2\Instance1.
Accept pipeline input? False
Accept wildcards? No

-Label <String>

The string label to be unregistered from the specified instance.

Characteristic Value
Required? Yes
Position

Named
Default value None
Accepted values Any string that describes a data host in terms that will assist an administrator in making assignments to that data host. For example, a data host that was intended for application's private databases could be labeled "PrivateApplicationData".
Accept pipeline input? False
Accept wildcards? No
