Grant-AmCertificateReadAccess

 

The Grant-AmCertificateReadAccess grants the specified user read access to the private key of the SSL certificate.

After setting the SSL certificate on a logical host, run this cmdlet on all the physical hosts in the logical host. If the logical host is set up as an NLB cluster, specify the service account as the Username; otherwise, specify "NetworkServiceSid" as the Username.

Syntax

Grant-AmCertificateReadAccess -Username <String> -CertificateThumbprint <String>

Parameter descriptions

[-Username] <string>

Takes the user account to be granted read access to the private key of the SSL certificate.

Required? Yes
Position Named
Default value None
Accepted values

NetworkServiceSid

or a service account of the form "domain\username".
Accept pipeline input? False
Accept wildcards? No

[-CertificateThumbprint] <string>

Takes the thumbprint of the SSL certificate.

Required? Yes
Position Named
Default value None
Accepted values Thumbprint of the SSL certificate (can contain spaces).
Accept pipeline input? False
Accept wildcards? No

Examples

Grant-AmCertificateReadAccess -Username NetworkServiceSid -CertificateThumbprint "40 a1 d2 62 2b fb da c8 0a 38 85 8a d8 00 1e 09 45 47 36 9b"

Grant-AmCertificateReadAccess -Username contoso\adfsservice -CertificateThumbprint ddb0c572f0e0aaaf0d83a5f656584ad27e5deb55
