Publish-AmApplicationInstance

 

The Publish-AmApplicationInstance cmdlet uploads the application package to the database.

The Publish-AmApplicationInstance cmdlet does not process manual changes to the ApplicationAuthorization element of the deployment manifest. This is by design. Application authorization is hardcoded by the application developer and is not available for adjustment by administrators.

Amalga built-in role membership required: AmalgaAppAdmin, AmalgaSysAdmin

For more information, see Built-in roles.

Syntax

Publish-AmApplicationInstance -ApplicationCabFile <String> -ApplicationAuxiliaryFiles <String>

Parameter descriptions

[-ApplicationCabFile] <string>

Takes the local or Universal Naming Convention (UNC) path for the application .cab file to be uploaded to database.

Characteristic Value
Required? Yes
Position Named
Default value None
Accepted values Local or UNC path. Examples: \\sharename\TestApp.cab,
c:\Temp\TestApp.cab, .\TestApp.cab
Accept pipeline input? No
Accept wildcards? No

[-ApplicationAuxiliaryFiles] <string[]>

Defines the semicolon-separated UNC paths to all the resolved application and service manifest files.

Characteristic Value
Required?

Yes
Position Named
Default value None
Accepted values

Semicolon-separated UNC paths to the resolved application and service manifests.

  • The manifest file names should be exactly the same as the application and service names.
  • There should be one resolved manifest file for the application and one resolved manifest file for each service that is part of the application.

  • Manifest file names are case-insensitive.

  • Examples: C:\Manifest\LabApp.xml, .\LabApp.xml,
    \\sharename\LabApp.xml
Accept pipeline input? No
Accept wildcards? No
