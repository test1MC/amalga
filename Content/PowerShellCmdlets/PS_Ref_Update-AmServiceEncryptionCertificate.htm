Update-AmServiceEncryptionCertificate

 

The Update-AmServiceEncryptionCertificate cmdlet updates the service encryption certificate associated with the endpoints of an service or component. This certificate is also used as the token encryption certificate in .

The SSL certificate of the logical host to which a service is assigned is used as the service and token encryption certificate.

Syntax

Update-AmServiceEncryptionCertificate -InstanceName <String> -InstanceType <String> -EncryptionCertificate <String>

Parameter descriptions

[-InstanceName] <string>

Takes the name of an service or component instance.

Required? Yes
Position Named
Default value None
Accepted values

Name of an service or component instance.
Accept pipeline input? False
Accept wildcards? No

[-InstanceType] <string>

Takes the type of instance.

Required? Yes
Position Named
Default value None
Accepted values

ServiceInstance

ComponentInstance
Accept pipeline input? False
Accept wildcards? No

[-EncryptionCertificate] <string>

Takes the encryption certificate, encoded as a base64 string.

Required? Yes
Position Named
Default value None
Accepted values

Encryption certificate encoded as a base64 string. Example: $cert64
Accept pipeline input? False
Accept wildcards? No

Example

Update-AmServiceEncryptionCertificate -InstanceName TopologyService -InstanceType ComponentInstance -EncryptionCertificate $cert64
