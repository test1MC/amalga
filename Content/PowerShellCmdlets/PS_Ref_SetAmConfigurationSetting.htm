Set-AmConfigurationSetting

 

Set-AmConfigurationSetting sets the values for the application, service feature, or component settings.

Amalga built-in role membership required: AmalgaSysAdmin

For more information, see Built-in roles.

Syntax

Set-AmConfigurationSetting -InstanceType <ConfigurationSettingInstanceType> -InstanceName <String> -SettingName <String> [-SettingValue <String>] [-SettingValueFile <String>]

Parameter descriptions

-InstanceType <string>

Takes the type of the instance.

Characteristic Value
Required? Yes
Position Named
Default value None
Accepted values ApplicationInstance

ServiceInstance

FeatureInstance

ComponentInstance
Accept pipeline input? False
Accept wildcards? No

-InstanceName <string>

Takes the name of the instance.

Characteristic Value
Required? Yes
Position Named
Default value None
Accepted values Instance names, such as Terminology_Inst1 (application instance), or externalregistry (feature instance).
Accept pipeline input? False
Accept wildcards? No

[-SettingName] <string>

Takes the new value for the setting.

Characteristic Value
Required? No
Position Named
Default value None
Accepted values Setting value as a string.
Accept pipeline input? False
Accept wildcards? No

[-SettingValue] <string>

Takes the new value for the setting.

-SettingFile and -SettingValue are mutually exclusive.

Characteristic Value
Required? No
Position Named
Default value None
Accepted values Setting value as a string.
Accept pipeline input? False
Accept wildcards? No

[-SettingValueFile] <string>

Takes the path to a file containing settings for the instance.

-SettingFile and -SettingValue are mutually exclusive.

Characteristic Value
Required? No
Position Named
Default value None
Accepted values Path to the file containing the setting values. For example, "c:\ValueFile.doc".
Accept pipeline input? False
Accept wildcards? No
