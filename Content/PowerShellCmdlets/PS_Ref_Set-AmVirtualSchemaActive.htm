Set-AmVirtualSchemaActive

 

The Set-AmVirtualSchemaActive cmdlet activates the virtual schema associated with a specified application instance. The virtual schema instance is activated when

it is deployed unless additional entity bindings are required. In that case the virtual schema will inactivated. This cmdLet can be used to activate the instance after the bindings are set

Amalga built-in role membership required: AmalgaAppAdmin, AmalgaSysAdmin

For more information, see Built-in roles.

Syntax

Set-AmVirtualSchemaActive -ApplicationInstanceName <String>

Parameter descriptions

-ApplicationInstanceName <String>

Takes the name of the application instance whose virtual schema is to be activated.

Characteristic Value
Required? Yes
Position Named
Default value None
Accepted values The name of the application instance as specified in the topology.
Accept pipeline input? False
Accept wildcards? No
