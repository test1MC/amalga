Configure binding: add and edit joins

 

This topic describes how to edit joins, or relationships, between tables for an entity and binding source.

You can add and edit relationships (joins) between a bound column's parent table or view and the anchor table (the table or view that is bound to the entity's primary key property). Joins allow you to bind properties of an entity to columns from multiple tables and views. For example, you can bind one property to the Observation column from the entity's anchor table and bind another property to an Order column from another table or view, as long as you specify a join between the tables or views.

Each entity property binding can have a unique join for its bound column. A join maps the primary key column of the bound column's parent table or view to a selected column of the anchor table.

Self-joins are also possible; that is, if the bound column is in the anchor table, you can add or edit a join for the anchor table.


Required Role: AmalgaDataAdmin or AmalgaSysAdmin

To add a relationship (join) for an entity property binding

  • Bind the entity property to a column from a table or view other than the entity's anchor table and then follow the prompts to select the desired anchor table column. See Configure binding: selected column ( tab).
