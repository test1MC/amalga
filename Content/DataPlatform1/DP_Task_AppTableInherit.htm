Configure access to an application's private database tables

 

This topic describes how to configure access to application-private database tables.

applications can contain application-private databases, which include database tables. You can configure read-write and read-only access for data elements in an application-private database by inheriting access rules from entities in the .

Access inheritance is suggested in the virtual schema for the application, with zero or one suggested parent entity for each private database table. You can enable or disable suggested access inheritance for each table.

Only those entities that a virtual schema suggests for a database table can be selected for access inheritance. This configuration cannot be overwritten by an administrator.

When suggested access inheritance is enabled for a table, the access policy from the indicated public entity is applied to the table as a child, which causes roles authorized for the parent entity to automatically receive access to the table at the level indicated by the access rules (read-write or read-only).

For example, if a rule gives the role Doctors read-write access to the entity Contoso.Patient where Patient.VIPFlag != ‘1’, and an application containing a private database table Note is configured to inherit access rules from the entity Patient, members of the role Doctors can read and write to the table Notes only for patients where Contoso.Patient.VIPFlag != ‘1’.

When suggested access inheritance is disabled for a table, no access policy is applied; all authorized application users are able to access the table according to access level permissions. Users of the application instance who are members of roles authorized to an access level that allows writing data can read and write data from the private table unconditionally. Application access levels are defined in the application manifest for the application; see Application manifests.

In the current version of , access rules cannot be configured for the private database tables directly.

  3. In the Application Configuration Tasks pane, click Manage Private Data Access.

    The Access Inheritance dialog box appears, listing all application-private database tables for the selected application instance along with any public entities suggested for access inheritance. Suggestions are derived from the virtual schema for the selected application instance.

  4. To opt out of access inheritance for a private table, clear the check box for that table.
  5. To apply suggested access inheritance from the indicated public entity to a private table, select the check box for that table.

  6. Click Save to save changes and close the dialog box.
