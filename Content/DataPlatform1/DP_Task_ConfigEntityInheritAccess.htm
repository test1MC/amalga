Configure access inheritance for an entity

 

This topic describes how to inherit access rules from peer or parent entities.

You can reuse an access policy in other entities by configuring the other entities to inherit from the entity on which the access policy was defined. For example, you can configure the Visit entity to inherit the access policy (containing its access rules and sensitive label permissions for all authorized roles) from the Patient entity. Inheritance can be between peers or from parent to child.


Required Role: AmalgaDataAdmin or AmalgaSysAdmin

To inherit access rules from a peer or parent entity

  5. Select Parent Entity or Peer Entity and click Next.

    Ensure that each role represented in the child entity's access policy is also represented in the parent entity's access policy. Parent rules can be set at the rule access level for "authorized children only."

    By default, the dialog box lists entities that have the same semantic tags (as applied to their primary key properties).

  6. To browse the complete rather than just those with matching semantic tags, select Entire Registry.

  7. Select the desired entity and click Next.

    The primary key property of the selected parent or peer entity is listed.

  8. Select the property of the current entity that you want to map to the related entity's primary key property.

    If the related entity is a parent, this mapping explicitly defines a primary key-foreign key relationship. If the related entity is a peer, this mapping explicitly identifies and matches the primary key properties of both entities.

  9. Click Finish to save inheritance settings and close the dialog box.

    The changes are saved and the pane is redisplayed.
