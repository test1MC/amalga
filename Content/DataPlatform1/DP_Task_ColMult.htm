Configure binding: multiple binding sources per entity

 

This topic describes how to bind entities to multiple binding sources.

You can bind entities to multiple binding sources. Each added binding source is automatically given the next lower precedence. For example, if two binding sources exist for an entity and you add a third, then the third binding source is given the precedence of number three in the sequence of binding sources for that entity.

Use multiple binding sources to make data from different storage schema available through a single entity. For example, inpatient data might be stored separately from outpatient data; therefore, each type of data would have unique storage schema. To bind the patient entity to both storage schemas, you can create a binding source specific to each storage schema.

Precedence of binding sources cannot be changed except by removing and re-adding binding sources in the sequence that matches the desired precedence.


Required Role: AmalgaDataAdmin or AmalgaSysAdmin
