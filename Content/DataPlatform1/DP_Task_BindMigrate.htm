Migrate binding configuration

This topic describes how to migrate entity binding information across installations.

You can migrate entity binding information from one installation to another. This procedure is useful when migrating applications.

Exporting binding information results in a file with a .bnd extension. Importing binding information involves loading the .bnd file to the in the destination site. At the time of import, the system validates that all referenced storage schema (databases, schemas, tables, views, and columns) exist in the destination environment (although the server name can differ) and that source system definitions and semantic tags (concept GUIDs) are the same as the source environment.

Before beginning this procedure, ensure that the source and destination environments match exactly: tables, views, data catalog, and must be the same.
