Roles and data (entities)

 

This section describes how roles interact with data via entities.

In , data permissions are managed by defining access policies for the corresponding entities. An access policy defines to a unique entity.

Each access policy defines one or more role-specific access rules and allows configuration of role-specific access to any entity properties that are tagged with sensitive labels. Each entity can also inherit access policies from parent or peer entities.

Each rule in an access policy gives the role a specific privilege (read or read-write) for the entity, as well as specifying the rule access level and any SQL conditions.
