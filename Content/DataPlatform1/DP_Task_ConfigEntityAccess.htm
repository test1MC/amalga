Grant permissions to an entity (add an access rule)

 

This topic describes how to enable users to access data by authorizing their roles for the related entities through access rules.

You can enable users to access data by granting permissions for the related entities to the users' roles.

Required Role: AmalgaDataAdmin or AmalgaSysAdmin

To grant permissions to an entity (add an access rule)

  4. In the pane, click Authorize Role.

    If the role you want is already authorized, you can alternatively select the role in the Authorized Roles section and then click Add New Rule.

    The dialog box appears.

  5. Select the role and click Next.
  6. Select the rule access level and click Next.
  7. Select the privilege (read or read/write) and click Next.

  8. Define a SQL condition if needed.

    For instructions on adding SQL conditions to access rules, see Add a SQL condition to an access rule.

  9. Enter a name for the rule and then click Finish.

    The rule is saved and the pane is redisplayed.

    To authorize the role to access data from entity properties that are tagged with sensitive labels, select the check boxes for the sensitive labels listed under . See Grant permissions for a sensitive label for instructions.
