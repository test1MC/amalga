Copy an access rule

 

This topic describes how to copy access rules for use by other roles in access policies for entities.

You can copy access rules from one role to another role in the same entity-specific access policy.

For information on reusing access policies in other entities (access inheritance), see Configure access inheritance for an entity.


Required Role: AmalgaDataAdmin or AmalgaSysAdmin

To copy an access rule

  4. In the Authorized Roles section, click the role that has the access rule you want to copy.
  5. In the Rules applying to Role section, expand the group that contains the access rule you want to copy, right-click the access rule, and then select Copy Rule from the right-click menu.
  6. In the Authorized Roles section, click the role that you want to use the copied rule.

  7. In the Rules applying to Role section, right-click a group and then select Paste Rule from the right-click menu.

    You can click any rule group for purposes of pasting the rule.

    The copied rule appears in the Rules applying to Role section for the selected role.
