Edit an access rule

 

This topic describes how to edit access rules in access policies for entities.

You can edit access rules for a selected role. The following access rule properties are editable: 

  • Rule access level
  • Rule privilege (read or read-write)
  • SQL conditions
  • Name of the rule


Required Role: AmalgaDataAdmin or AmalgaSysAdmin

To edit an access rule

  4. In the Authorized Roles section, click a role to display its access rules.

  5. In the Rules applying to Role section, expand the group that contains the access rule you want, click the access rule, and then click Edit Rule.

    The dialog box appears, listing the current definition of the access rule.

  6. In each page of the dialog box, make changes as needed and then click Next or Finish.

    To remove a SQL condition from the current rule, clear the check box for where custom condition is true in the Add Custom Condition page. For instructions on adding SQL conditions to access rules, see Add a SQL condition to an access rule.

    When you click Finish on the last page of the dialog box, the rule is saved and the pane is redisplayed.
