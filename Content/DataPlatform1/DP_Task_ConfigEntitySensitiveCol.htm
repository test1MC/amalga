Restrict access to an entity property (sensitive labels)

 

This topic describes how to limit access to data using sensitive labels.

You can hide columns of data from unauthorized roles by applying sensitive labels to the corresponding entity properties.

When a column is hidden, the data for that column is redacted. For information about working with redacted data as an application developer, see Working with restricted data.

Hiding the data involves creating a sensitive label, applying that label to the entity property mapped to the column that contains the data, and granting access to the sensitive label for select roles.

Granting roles access to a sensitive label involves modifying the access policy for the related entity.

For example, to hide test result status stored in a particular column (corresponding to the Result property in the Patient entity) from administrators but show it to doctors, define a sensitive label TestResult, grant the Doctors role access to this label, and bind the label to the Result property. Now doctors can see these test results but administrators cannot.

Sensitive labels available for application are those defined for the current entity, its peers, and its parents. Multiple properties can make use of a sensitive label. Role-specific permissions are determined by the union of all rules, including sensitive labels.
