Debug access rules

 

This topic describes how to debug access rules for roles.

You can debug performance of user access to data and troubleshoot errors by temporarily disabling access rules.


Required Role: AmalgaDataAdmin or AmalgaSysAdmin

To debug performance of user access to data

  4. Select a role to view the corresponding SQL SELECT statement.

    The SQL SELECT statement determines the underlying data for the current entity that the role can access.

  5. To see how the statement changes if an access rule for the role is disabled, do the following.
    1. Expand the role, expand the rule category, and select the rule.
    2. Click Disable Rule.
    3. Reselect the role.
  6. To copy the SQL SELECT statement for further debugging in or other tools, select the statement and press CTRL+C.
  7. For , use to assess performance of SQL SELECT statements.
  8. When you are finished debugging, ensure that all rules are re-enabled.


Required Role: AmalgaDataAdmin or AmalgaSysAdmin

To troubleshoot access rule errors

  4. Select a role to view its access rules.
  5. To see if disabling an access rule removes an error displayed in the pane, do the following.
    1. Expand the rule category and select the rule so that it is highlighted.
    2. Right-click the rule and then select Disable Rule.
  6. To enable a rule, right-click the rule and then select Enable Rule.
  7. When you are finished debugging, ensure that all rules are enabled.
