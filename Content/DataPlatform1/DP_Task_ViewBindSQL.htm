View the binding SQL statement for an entity

 

This topic describes how to view the binding SQL statement for an entity.

You can view the binding SQL statement (TRANSACT-SQL SELECT statement) that is used to select data for an entity.

Example of a simple binding SQL statement selecting patient data for an entity: 

      SELECT [EID]
         ,[FName]
         ,[LName]
         ,[SSN]
         ,[DOB]
         ,[DOB_o]
      FROM [AzADT].[dbo].[HIS601]

In , the TRANSACT-SQL SELECT statement is generated using each binding source in sequence, with a UNION statement to combine all binding sources for the entity.


Required Role: AmalgaAppAdmin, AmalgaDataAdmin, or AmalgaSysAdmin

To view the binding SQL statement for an entity

  4. Click the pane.

    The pane expands to show the TRANSACT-SQL SELECT statement used to select data for the current entity.
