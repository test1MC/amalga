Performance counters for the

 

This topic describes performance counters for the application ()

The logs the following types of events:

  • Web access events
  • Data access events
  • Capacity-related events

Web access events

If the encounters a web access error while accessing a web service (such as the topology web service), an event will be created based on the following table.

Event title HTTP error code Event ID
Request unauthorized 401 35002
Request timeout 408 35008
Service unavailable 503 35021

Database access events

The provides an interface for the library to create an event log in case of a database access error.

Capacity-related events

If the encounters an event that pertains to capacity of a used resource, appropriate events be created based on the following table:

Event title Event ID
Disk is full 31001
Disk utilization above high water mark 31002
Critical resource starved 31003
Critical resource above high water mark 31004
Capacity trend predicts failure in N days 31005
Restart if load fails 31008
Component is hung, backlog above threshold 31009
