Bind entities to the underlying SQL view

You can use a cmdlets to bind entities to the underlying SQL views.

For more information on the cmdlet, see Set-AmEntityBinding on page 1.

To bind entities to SQL views

  1. From Doors Start, click All Program, click Accessories, click Doors PowerShell, and then click Doors PowerShell. You can run Doors PowerShell from any computer in the topology.
  2. Type:

    Import-module 'C:\Program Files\Microsoft Amalga\System 3.0\Platform\PowerShell\Microsoft.Amalga.Powershell.dll'

  3. Type:

    Set-AmEntityBinding -FilePath ImagingBindingFilePath

    Placeholder Value
    ImagingBindingFilePath The path of the binding configuration file that maps entities to SQL views. The name of the sample file present on the installation media is ImagingEntitiesBinding.xml. For example: c:\install\ImagingEntitiesBinding.xml.
