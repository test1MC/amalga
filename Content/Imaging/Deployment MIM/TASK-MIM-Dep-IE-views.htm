Create underlying SQL views

 

Copy the configuration files from the media to the data host, prepare the configuration file, and run a script to create the underlying SQL views.

To create the underlying SQL views

  1. Copy the following files from the V3\Release\Samples\Imaging folder on the installation media into a single folder on the data host. The data host is the server that hosts the AzAEID and ClinicalData databases.
    Copy these files:
    • CreateImagingViews_V2.ps1
    • ServerConfiguration.xml
    • create_imaging_correlation_view.sql
    • create_linked_server.sql
    • ImagingEntitiesBinding.xml

    • create_dicommetadata_views.sql

    • create_IRDD_views.sql

  2. Modify the server configuration file. For more information, see Modify the Server Configuration File on page 1.
  3. You can launch on any computer in the topology, for example, AppsLogicalHost2. Run the following command, supplying the placeholders as indicated in the table:

    .\CreateImagingViews_V2.ps1 ConfigurationFile DataHost userId userPassword ImagingDataHost ImagingUserId ImagingUserPassword

    Placeholder Value
    ConfigurationFileName of the configuration file. The name of the sample file present on the installation media is “ServerConfiguration.xml.” This file should be modified before creating the Imaging views.
    DataHostName of the server that hosts the Amalga V3 clinical database
    userId

    User ID for the data host (optional)

    If the optional parameters are not provided, Doors authentication will be used.

    userPasswordUser password for the data host (optional)
    ImagingDataHostImaging data host is the server where the imaging databases are located, i.e. in centralized (single site) topology this would be the Imaging data server with the database, in the distributed (multi-site) topology this is the IRDD.
    ImagingUserIdUser ID for the Imaging data host (optional).
    ImagingUserPasswordUser password for the Imaging data host (optional).
