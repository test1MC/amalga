application operation

 

This topic describes the availability of the application, process flow, communication, and data access for the application ().

Process flow

The provides patient, study, series, and instance-level metadata information and returns the corresponding application (IDA) instance name for a given site ID.

After the client retrieves the metadata, including the site identifier, the client may communicate with the again to retrieve the IDA instance name so that it can ask for the BLOB retrieval.

The facilitates the following process flow.

  1. A control or application sends the service (IRS) a unique patient ID.
  2. The IRS requests data from the database through the application.
  3. The IRS returns the available information for the client application.

Application input and output

The API takes a unique patient ID and returns study information. For more information, see Structure on page 1.

For more information about APIs, see Reference Library on page 1.

Communication

Controls communicate with the via web APIs exposed on the IRS.

The IRDA communicates with the via web services.
