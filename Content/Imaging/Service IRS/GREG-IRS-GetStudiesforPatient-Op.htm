Get studies for patient

 

The service () can provide study information using the person ID.

The person ID corresponds to a patient. The application passes a person ID and the IRS will return the studies.

GetDicomStudiesForPatient

[OperationContract]

StudyInfoEntry[] GetDicomStudiesForPatient(string patientID);

Parameter Description
Patient ID The patient ID is an internal ID of the patient.

For the method, GetDICOMStudiesForPatient, see Syntax on page 1. For all members of the IRS see Constructors on page 1.

Operation

  1. The client, or application, passes the person ID to the .

    If the person ID parameter is null, an ArgumentNullException will be thrown.

  2. The IRS returns a list of the study information entries.

Summary

The IRS returns a list of the study information entries. These entries include the basic study, series, and instance metadata for populating a study list on the client. Each study contains a collection of series, which contains basic series information, location information, collection of instance IDs.

databases

For a configuration that uses databases, the configuration parameter EIDColumnContent determines how the OID is mapped to an EID internally.

The information is stored as a string in the EidColumnContent column. The following configuration entry is part of the ImagingRegistryDocumentService.xml service manifest:

  <Settings>

      <Setting Name="EidColumnContent" Type="System.String" >

        <DefaultValue>EID</DefaultValue>

      </Setting>

  </Settings>

The values are supplied during the deployment process. There is one configuration entry for this functionality. The EidColumnContent determines whether the provided person ID is patient-level matching (EIDfor SOID) or visit-level matching (EID).
