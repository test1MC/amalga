Dynamic views

 

This topic describes how dynamic views allow you to access patient list data using the .

Patient lists cannot be accessed directly using the . Dynamic views are abstractions of patient lists and provide that access. They are created and updated automatically whenever a patient list is created or changed.

Using dynamic views, an application can access data contained in a patient list using tools like Excel PowerPivot and to create graphical views of the data. For more information on creating reports, see Create a report from a patient list on page 1.

Dynamic views have the following naming convention:

  • Username_listtitle_listID

All characters in the list title that are not alpha-numeric are replaced with an underscore (_). If the overall length of the Username_listtitle_listID exceeds 128 characters, then the listtitle portion of the string will be truncated.

To find the name of the dynamic view to use in your application, see Locate name of dynamic view on page 1.

Dynamic views are in the public store. They get erased when you uninstall and reinstall the instance. To rebuild the dynamic views, you can run a command on the platform server. The format of the command is:

RebuildListViews.exe FQDN of platform server DataExplorer instance name DataExplorerPrivate instance name
