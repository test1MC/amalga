Solution Picker control

 

The Solution Picker control lets users navigate between solutions.

The Solution Picker control displays a drop-down list of all solutions for which the authenticated user has been granted permissions to view in . Solutions for which the user does not have permissions are not shown in the list. When the user makes a selection, the default page for the selected solution automatically appears.

In order for the Solution Picker control to work properly, users must have root-level read access on the site. This applies to all site navigation features.

Adding the Solution Picker control to a project

You can add a Solution Picker control to a project in by following the instructions in Toolbox controls.

The Solution Picker control is present by default in the header of the master page template for the provider and staff portal pages. You would only need to explicitly add this control in situations where the master page template was not being used.

Customizing a Solution Picker control instance

You can change the appearance and behavior of each Solution picker control instance by editing the control properties in the Properties Window. For more information about properties of the Solution Picker control, see SolutionPicker.

Managing Solution Picker list contents

The visibility of solutions in the Solution Picker is managed by permissions. There is no explicit way to add solutions to, or exclude solutions from the list. To expose a solution to a particular user or group, the administrator needs to grant the appropriate permissions. Likewise, to keep from exposing a solution to a particular user or group, the administrator should configure permissions so that the user or group does not have permissions for the site containing the solution. For more information, see Roadmap: Grant permissions for a site.

Control behavior in nested hierarchies

The Solution Picker control displays only the solutions within its child hierarchy, and will not list any solution above that hierarchy. For example, if ProviderPortal-2 is a child of ProviderPortal-1, ProviderPortal-2 will appear in the solution picker of ProviderPortal-1. However, ProviderPortal-1 will not appear within the solution picker of ProviderPortal-2. This has a potential impact on navigation, because once a user has navigated to ProviderPortal-2 the only way to get back to the parent ProviderPortal-1 is by using the back button.
