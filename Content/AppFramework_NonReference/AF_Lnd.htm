Developing a user interface using the

This topic describes how to create a -based user interface for an application using the features of the .

The is a set of features that enable the rapid development of clinical applications in a environment.

The foundation of the is a runtime environment. The includes both client and server side APIs that streamline authentication and data access, enhance communication within and among applications, and integrate features such as logging and configuration into the environment.

The also includes a set of reusable components built on those APIs. The reusable components begin with a set of extensible web controls and WebParts, then use those parts to construct the Connected Health Portals, an extensible set of basic but fully functional clinical Web sites. Both the websites and the elements that comprise them are available for reuse and extension in your own healthcare applications.

Finally, the includes a set of design tools. The provides several site and project templates for a head start on application creation. The templates provide the design surface for quick assembly of clinical applications, using both reusable controls from the and additional item templates.
