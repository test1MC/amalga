Participant

 

This topic describes the structure of the Participant object and the role of a participant in the Page Messaging System.

A Page Messaging System participant is a server control, WebPart, or other web page element that uses the Page Messaging API to exchange state information with other participating page elements. The Participant object provides the entry point to the Page Messaging API.

This topic contains the following subsections:

Create a participant

To add a participant to the messaging system, simply call the static create a Participant object with a locally unique ID and call its join method. The participant ID must be unique within the scope of the local messaging object, which is usually the containing web page.

Manage groups

To add a participant to the messaging system, simply call the static create a Participant object with a locally unique ID and call its join method. The participant ID must be unique within the scope of the local messaging object, which is usually the containing web page.

The join method both registers the Participant with the [group, and registers two callbacks] .

CommonContext

[no, think it joins the CC group later]

If a joining participant indicates that it will join the CommonContext, then its context updates will automatically be merged with the common context and synchronized with the web service. (It is not necessary to join the common context in order to read its contents.) A participant that does not join the common context is limited to L1 participation, and can only send messages to other messaging participants that explicitly subscribe to its messages.

Surveyable

If a joining participant indicates that it is Surveyable, it must subsequently create a surveyCallback callback method that indicates whether or not the participant is in a state that makes it unsafe to update the common context. When it is safe to proceed, the method should return null. When it is not safe to update the context, the survey callback method should return a text message that includes the following information:

  • The identity of the surveyed participant
  • The reason that the context change may be unsafe to apply.

The message will be displayed in a dialog box, allowing the user to decide whether or not to make the change.
