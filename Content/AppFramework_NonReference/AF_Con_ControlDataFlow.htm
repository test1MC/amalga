Features of Framework controls

 

This topic describes the features common to all controls.

The following features are defining characteristics of a control. Unlike other features described in Amalga Framework Controls, which are available to any page element that can discover the API assemblies, these features must be correctly implemented for a control to participate in a -based environment.

Any control derived from a generic or production control will support these features by default.

This topic contains the following subsections:

  • Client API management on page 1
  • Page messaging and context passing on page 1
  • Configuration and user settings on page 1
  • Application identity on page 1

Client API management

The primary feature that distinguishes server controls from standard server controls, and that distinguishes Web Parts from standard Web Parts, is their support for the client-side APIs.

Implementation details may differ from control to control, but all controls must do the following:

  • Load the client side JavaScript APIs into the Document Object Model (DOM)of the local browser.
  • Initialize the client-side APIs, if they have not already been initialized.
  • Register the control with the appropriate client-side API managers.

Page messaging and context passing

Page messaging is a primarily client-side messaging system used to synchronize otherwise independent elements of a single page within a application. It also uses a wWeb service to synchronize some application states as a "common context" across page boundaries that the client side messaging API cannot cross.

Configuration and user settings

The Configuration system allows applications to store and retrieve strongly typed values on a per-application basis. The system also allows those values to be overridden for each user, and for the user setting values to persist across multiple sessions.

Application identity

controls can resolve the identity of their own application instance, and can identify themselves to other controls.

This includes, but is not limited to, an implementation of the IAmalgaApplicationIdentity interface.
