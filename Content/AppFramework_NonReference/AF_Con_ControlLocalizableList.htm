Localizable List Control

 

The Localizable List control is a drop-down control that gets its content from a resource (.RESX) file.

Unlike typical drop-down controls, the Localizable List control can be localized, meaning that all content is stored in a .RESX file. This unique characteristic makes it easy to edit the contents of the control, and adapt the contents of the control to multiple languages, as no recompilation or database changes are required.

Updating a Localizable List control instance

You can make changes to a Localizable List control instance by using to create a new resource file to which you can add new keys and values as well as override existing ones. To do this, create a new file named CustomLists.resx, and deploy it to \inetpub\wwwroot\wss\VirtualDirectories\[port]\App_GlobalResources\.

The resource file for all default instances of the control is located at C:\Program Files\Common Files\Microsoft Shared\Web Server Extensions\14\Resources\AmalgaLists.resx. Do not directly edit this file.

Adding a Localizable List control to a project

You can add a Localizable List control to a project in by following the instructions in Toolbox controls. Add the resource strings for all custom controls to a new custom resource file named CustomLists.resx. Each list has a key that is mapped in the ResxKey property of the control. In turn, the value of this key is comprised of a list of semi-colon separated values that should appear in the dropdown list. The order that they appear in the list is the order they'll show in the dropdown list. For example:

Customizing the appearance of a Localizable List control instance

You can change the appearance and behavior of each list control instance by editing the control properties in the Properties Window. For more information about properties of the Localizable List control, see LocalizableList.
