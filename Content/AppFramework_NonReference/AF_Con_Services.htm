service endpoints

 

This topic describes the default service endpoints for the web services.

 

This topic contains the following subsections:

  • Generic service endpoint
  • Parts of a service endpoint
  • service values
  • Example

Generic service endpoint

The service endpoints of the web services all conform to the following template:

https://ServerFQDN:ServicePort/ServiceInstanceName/ServiceAddress

Parts of a service endpoint

Each endpoint has the following parts:

  • ServerFQDN - the fully qualified domain name of the server that hosts the web service.
  • ServicePort - the TCP port assigned to the web services. The default service port for web services is 8801.
  • ServiceInstanceName - The instance name assigned to the web service when it was deployed. Instance names are site-specific and are usually specified in the service manifest. In practice, many instance names are constructed by adding a standard postfix (such as "_Ins1") to each service name. The names of the web services are listed below.
  • ServiceAddress - the path to the .svc WCF implementation file for each service. The default path for each service is listed below.

Example

For example, if the Task Flow service were deployed to Webserver.Example.com and its instance name was derived from its service name as described above, its default service endpoint would be:

https://Webserver.Example.com:8801/TaskFlowService_Ins1/Microsoft.Amalga.TaskFlow.Service.TaskFlowService.svc
