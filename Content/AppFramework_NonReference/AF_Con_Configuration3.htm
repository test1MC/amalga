Store and retrieve the value of a user setting

This topic describes how to store and retrieve the value of a user setting.

The configuration system allows controls to store and retrieve strongly typed values on a per-application basis. The system also allows per-user changes to those values to persist across multiple sessions.

The Configuration API cannot be used to set the default value of a configuration setting. Whenever a configuration value is stored using SaveConfigurationItem, it is automatically associated with the current user.

The IWriteableConfigurationItemStore interface implements IConfigurationItemStore and adds the SaveConfigurationItem method, which allows configuration items to be written back to the store. IWriteableConfigurationItemStore does not include type-specific helper methods, so new configuration values must be converted to strings before they can be saved.

Example

The following code sets the value of the NumberOfTabs setting for the current user.

// Name of a configuration setting
String tabSetting = "NumberOfTabs";

// Get the configuration store for the current application instance
IConfigurationItemStore appSettings =
AmalgaConfigurationManager.Current.GetApplicationConfiguration((BaseServerControl)this);

// Retrieve the NumberOfTabs configuration setting.
Int32 numberOfTabs = appSettings.GetConfigurationValueInt32(tabSetting);
