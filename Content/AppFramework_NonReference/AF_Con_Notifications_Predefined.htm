Predefined notification type reference

This topic describes the predefined notification types available for use in applications.

The Notification service includes the following predefined notification types. Their names correspond to the key value of their message resource in the Notifications resource file (FrameworkNotifications.resx):

  • PatientMatchAutolinkFailedLastChance on page 1
  • PatientMatchAutoLinkFailedMultipleMatches on page 1
  • PatientMatchAutoLinkFailedNoMoreChances on page 1
  • PatientMatchAutolinkFailedTryAgain on page 1
  • PatientMatchAutoLinkProcessing on page 1
  • PatientMatchLinkRevokedByStaff on page 1
  • PatientMatchManualRequestFailed on page 1
  • PatientMatchManualRequestFailedMultipleTimes on page 1
  • PatientMatchManualRequestPending on page 1
  • PatientMatchManualRequestRejectedBlockedAccount on page 1
  • PatientMatchManualRequestRejectedByStaff on page 1

  • PatientMatchSuccessful on page 1

 

PatientMatchAutolinkFailedLastChance

An auto-link attempt failed. The user is trying again using the patient link form. If the next attempt fails, it will exceed the threshold of allowed attempts.

The maximum number of chances is determined by the AutoLinkMaxAttemptedMaxCount setting of the hosted application, which can be set using the . For more information on changing application settings with the , see configure application settings.

NotificationTypeId

f67afea0-41e4-43fd-83d8-d237cbeee51d

Severity

4 - Warning

Arguments

Argument Description
{0} Hospital name
{1} Record nickname or full name
{2} Hospital phone number
{3} URL or JavaScript for page refresh or redirect

 

 

 

 

PatientMatchAutoLinkFailedMultipleMatches

 

An auto Link attempt failed with more than one match found. The user is being informed that their request is pending approval; it was transferred to the Staff queue, since there were multiple exact matches found.

NotificationTypeId

6d8b34cf-3c18-41a3-9abf-60eeda5b7b41

Severity

5 - Info

Arguments

Argument Description
{0} Patient first name
{1} Record full name

 

PatientMatchAutoLinkFailedNoMoreChances

There are no more chances in auto-link mode, as the threshold has been reached. The request is automatically switched to manual mode.

The maximum number of chances is determined by the AutoLinkMaxAttemptedMaxCount setting of the hosted application, which can be set using the . For more information on changing application settings with the , see configure application settings.

NotificationTypeId

2cfbe58a-0131-4317-a4a3-7a4eaeea1786

Severity

4 - Warning

Arguments

Argument Description
{0} Record nickname or full name
{1} URL or LightBox script that takesthe user to Edit or Cancel request page

PatientMatchManualRequestFailed

The user’s manual link request was rejected by the hospital staff because a match could not be found. The staff has selected a statement to show to the end user through the Reject Match UI in the Staff Console.

Severity

3- Strong Warning

Arguments

Argument Description

{1} Hospital name
{2} Patient Request Details Lightbox URL or JavaScript
{3} URL to try again

A manual link has been requested, but was rejected by staff due to a blocked account. The user is not given an opportunity to submit another request; their ability to issue link requests is blocked.
