Provider Portal site

 

The Provider Portal site is used by providers to view patient data.

The Provider Portal site provides system integrators with a starting point for creating sites which enable providers (doctors, nurses, and other medical staff) to view patient data.

In this topic

  • Provider Portal main page (default.aspx)
  • Customizing the Provider Portal site
  • Adding a new page to the Provider Portal site

Provider Portal main page (default.aspx)

By default, the Provider Portal main page contains the global header and footer. You can customize this page by adding web parts, controls, and content as needed.

Customizing the Provider Portal site

The following modifications are specific to the Provider Portal site page.

  • Customize the Provider Portal header by adding, removing, resizing, and repositioning links, assets, and controls.
  • Add a custom background image to the Provider Portal header.

You can also add assets and controls to the page, and modify its appearance by updating the style sheets and using built-in customization options. For more general information about customizing the Portal sites, see Portal Page administration and customization.

Adding a new page to the Provider Portal site

You can use the Empty Site Page template to create additional down-level pages for the Provider Portal site. The following steps describe how to add a new page.

  1. Open the project in .
  2. Right-click the project name, click Add, then click New item.
  3. Under Installed Templates, click .
  4. Select the Empty Site Page template.
  5. Name the page, and click OK to create the page.
  6. Deploy the project.
