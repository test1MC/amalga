This topic describes the server controls available in the and provides guidance for adding them to projects.

Many of the server controls that are used in the foundational portals are also available for reuse as part of the SDK. Most of these controls are available in the .

Using controls

The displays icons for controls and other items that you can add to projects. To open the , click Toolbox on the View menu.

The organizes the available controls into tabs. The controls provided with the are found in the "Amalga Controls Library" tab, which should be available whenever a project created from a project template is open in .

Adding a control to a project

To add a visual control (that is, a control with its own UI) to a project, click on its icon in the and drag it into place on any design surface within the project.

Setting property values of a control

Selecting the control on the design surface will cause its public properties to appear in the Properties pane in . Changes made to those values will persist when the design surface is saved (Ctrl+S).

Many of the controls can be customized by changing their settable properties. For instance, the following markup text creates a MessageLightBoxHyperLink control that links to a localizable message box. (The display content of the message box is specified in a separate resource file, MyResources.resx.) 

<Amalga:MessageLightBoxHyperLink ID="InfoMsg" runat="server"
    LightBoxTitle="<%$Resources:MyResxFile,MyControl_MsgLightBox_Title%>"
    LightBoxResourceFileName="MyResources" 
    LightBoxBodyTextResourceKey="MyControl_MsgLightBox_Body_Text" 
    LightBoxOkButtonTextResourceKey="MyControl_MsgLightBox_Button_Text" 
    LightBoxOKButtonToolTipTextResourceKey="MyControl_MsgLightBox_Button_ToolTip"
    LightBoxMode="OkButton"
    />

Edit the visual appearance of controls

You can edit the visual appearance of the control by overriding the default styles. For more information, see Base structural styles. To discover which styles are in use in a given control, navigate to \14\TEMPLATE\CONTROLTEMPLATES\ and view the ASCX file source code. The following example shows the source code of the main content for the Patient Banner control (PatientBanner.ascx).

<div class="patientBanner">

<Amalga:PatientBannerProfileImage id="PatientBannerProfileImage" runat="server" />

<div class="row1">

<Amalga:PatientBannerIdentification id="PatientBannerIdentification" runat="server" />

</div>

<div class="row2">

<Amalga:PatientBannerClinicalModules id="PatientBannerClinicalModules" runat="server" />

</div>

<div class="clear"></div>

</div>

<div class="headerFade"></div>

Once you find the class for the element you want to restyle, you can find the style(s) in Base.css.

CarouselNavigationBar Carousel Navigation

Provides page level navigation based on the content of the SPNavigationNode structure of the current solution.

In the staff and internal provider portals, the carousel works with the SolutionPicker control, and the default node collection contains the pages of the selected solution. In the patient portal there is only one solution, so the default collection contains all the pages in the site.

A admin can reorder the pages in the node collection, which automatically reorders the links in the carousel. If the structure contains more pages than can fit in the header, an overflow UI will start to show with left and right arrows allowing the user to navigate to all of the available pages. The control also allows for the first page to be "pinned" to the far left of the navigation area if it should always be visible.
