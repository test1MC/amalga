Portal page administration and configuration

 

This topic describes the administration and configuration options for portal pages.

The portal pages expose the standard ribbon, enabling staff and administrators to customize and configure the page. The ribbon exposes standard customization options by default. Access to the ribbon is controlled exclusively by permissions. Ribbon functionality is exposed for custom pages as well, via the Ribbon Access control available in the toolbox.

In this topic

  • Using the ribbon
  • Managing central configuration settings

Using the ribbon

The ribbon allows administrators and system integrators to make a variety of changes to a page. You can toggle the ribbon on any of the default portal pages by clicking the caret ("^") symbol in the upper-right corner of the page. The caret is only visible to users who have the appropriate permissions. For more information, see Ribbon Toggle control.

Managing central configuration settings

There are a number of tasks that must be performed using central configuration. These tasks include deploying packages, retracting packages, deploying servers to a farm, etc. For more information, see Operations for Server 2010.
