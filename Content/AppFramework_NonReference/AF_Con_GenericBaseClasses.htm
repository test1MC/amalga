Base classes and control templates

 

This topic discusses the base classes and control templates that are available for developers to use.

The SDK provides several base classes for creating generic controls. Any control that is derived from one of these classes will automatically support the basic features required by the client infrastructure, but will require additional design consideration in order to smoothly integrate their additional functionality into an portal.

The base classes

BaseServerControl

The BaseServerControl is an ASP.NET composite control with built-in support for the page messaging and logging APIs. Its constructor also loads and initializes the client-side APIs into the DOM of the local browser, if another control has not already done so.

As a CompositeControl, the base server control has no default UI. It is generally used as a container for other, visible controls.

VisualServerControl

The VisualServerControl is a BaseServerControl that is designed to host a single user control.

Controls derived from VisualServerControl must override its DefaultUserControlPath property to specify the path of the user control that should be loaded along with the server control. The derived control also has the option of overriding the PreAddChildControl method to assign values to settable properties of the user control before it is rendered, and can override the RequiresPatientLink property to indicate that the server control requires access to patient linking in order to run.

TemplateControl

The TemplateControl is a VisualServerControl that is populated with child controls according to a predefined template.

Controls derived from TemplateControl may override its ItemTemplate property to provide a default template. Otherwise, a template should be assigned before the TemplateControl is rendered.

For more information about using templates to configure a server control, see Templated Server Control Example.

BaseWebPart

The BaseWebPart is a WebPart. Like the BaseServerControl, it provides built-in support for page messaging and logging, and loads and initializes the client-side APIs if they are not already present. Like other web parts, it does provide a default UI, but its appearance is mostly determined by the settings of its  host.

BaseUserControl

The BaseUserControl is a standard user control, with one added feature.

Controls derived from BaseUserControl can call its GetContainer(T) method. GetContainer(T) returns the closest containing control that implements the INamingContainer interface, which is usually its parent server control. The parent container control is useful for applying templates.

Unlike the server controls, the base user control does not manage the client-side APIs.

The control templates

The includes several item templates that allow controls to be quickly added to or other projects in .

Name Path Description
Amalga Server Control Visual C# Items/Amalga Creates a control derived from BaseServerControl, with additional safeguards for handling design time and runtime experience in SharePoint 2010.
Amalga User Control Visual C# Items/Amalga Creates a control derived from BaseUserControl, with additional safeguards for handling design-time and run-time experience in SharePoint 2010. It includes all references and register directives required to drag and drop into a host control or Web Part.
Amalga WebPart Visual C# Items/Amalga Creates a Web Part derived from BaseWebPart, with additional safeguards for handling design-time and run-time experience in SharePoint 2010.
Amalga Visual WebPart Visual C# Items/Amalga Combines the Amalga Web Part and Amalga User Control templates to create a user control hosted in a Web Part.
Amalga Empty Site Page Visual C# Items/Amalga Creates a site page derived from BasePage.
