Patient Portal site

 

The Patient Portal site is used by the institution to communicate to patients, family, and caregivers.

The Patient Portal site provides system integrators with a starting point for creating sites to meet the needs of patients, family, and caregivers. By default it contains two pages: the Patient Portal login page, and the Patient Portal home page.

Patient Portal login page (PatientSignIn.aspx)

The Patient Portal Login page is the first page patients see when they access the Patient Portal site. The initial page is non-authenticated, and presents a set of controls which allow new users to create a account (Patient Portal sign up control), and registered users to sign in to the site (Patient Portal sign in control). Other controls on the page include a "What is ?" lightbox control, containing a brief description of the service, and a user control that contains all components of the patient public header:

  • Ribbon Toggle control
  • Patient Portal Logo control
  • Clickable Healthvault Logo

The Ribbon Toggle control is never visible to end-users, but can be accessed by the site administrator and anybody with the appropriate permissions. For more information see Ribbon Toggle control.

Patient Sign Up Link Request page (PatientSignUpLinkRequest.aspx)

The Patient Sign Up Link Request page is where new patients can create a account, and request health records for family members. The patient first fills out the " information" section in the top half of the form, and the name and details for a family member in the "Patient Information" section on this page. They also have the option to add additional family members to their account after signing up.

Patient Portal home page (default.aspx)

The Patient Portal home page is where users are directed upon signing in. On this page, the patient header user control contains all components of the signed in patient header:

  • Ribbon Toggle control
  • Patient Portal Logo
  • HealthVault Record Selector control
  • The Carousel Navigation Bar control
  • The Patient Account Context control
  • The Notification Banner control

The main content placeholder of the Patient Portal home page contains two web part zones which enable administrators to add, edit, and delete web parts in page edit mode. There are two additional pages at the same level as this page

Customizing the Patient Portal site

The following modifications are specific to the Patient Portal site pages.

  • Customize the Patient Portal header by adding, removing, resizing, and repositioning links, assets, and controls.
  • Add a custom background image to the Patient Portal header.

You can also add assets and controls to the pages themselves, and modify the appearance by updating the style sheets and using built-in customization options. For more general information about customizing the Portal sites, see Portal Page administration and customization.

Adding a new page to the Patient Portal site

You can use the Empty Site Page template to create additional down-level pages for the Patient Portal site. The following steps describe how to add a new page.

  1. Open the project in .
  2. Right-click the project name, click Add, then click New item.
  3. Under Installed Templates, click .
  4. Select the Empty Site Page template.
  5. Name the page, and click OK to create the page.
  6. Deploy the project.
