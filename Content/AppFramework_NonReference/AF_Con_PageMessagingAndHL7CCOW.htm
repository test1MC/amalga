This topic describes the HL7 clinical context.

Clinical Context

Clinical context is information about the state of a user’s interaction with healthcare applications in the context of a specific clinical process or event. A common context allows multiple participants to coordinate their selection and presentation of data, so that choices made by the user while interacting with one application can be applied immediately by other applications.

Clinical context can include any information that is relevant to the clinical task at hand. Common examples include:

  • The ID of the patient who is the focus of the current task
  • The ID of the clinical event or encounter with which the user is engaged
  • A specific time or range of times that specify a period for which clinical data is relevant to the user
  • A grouping of patients, personnel, or clinical resources that the user will manage or inspect
  • The ID of the individual who is using the application

The Amalga Application Framework supports clinical context management with the Page Messaging System, a primarily client-side messaging system that also uses a Web service to synchronize application states across process boundaries that the client side messaging API cannot cross.

Context Management and HL7 CCOW

The Amalga Application Framework context management system is designed to conform to the Context Management Standard specifications published by the Health Level Seven Clinical Context Object Workgroup, collectively called the HL7 CCOW.

For more information about the CCOW standard and best practices, see the HL7 web site.
