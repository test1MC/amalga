Using the HealthVault Record Selector

The HealthVault Record Selector is a Toolbox control that allows a user to select and view a record that the user is authorized to see.

The HealthVaultRecordSelector depends on the HealthVault Connector to establish a connection to the service, and the STS for user authentication and authorization. If those applications are installed correctly, the HealthVaultRecordSelector can be dropped from the into a custom control and work with no additional configuration.

Add a new record

By default, selecting "Add a Person" in the Record Selector causes the user to navigate away from the page containing the Record Selector and to a page that begins the Patient Matching workflow. It is possible to modify this behavior in two ways.

Assigning a value to the AddNewRecordUrl property will cause the Record Selector to navigate to the specified page, instead of the default Patient Match page.

Using the SDK

Any application can connect to the shell, allow the user to log in, and use the web service to send and receive data. The HealthVault Connector makes connecting to the shell easier by providing connection information based on a common Application ID and certificate that are assigned to each local implementation.

For more information, see Getting Started with the SDK.

Sending data to

The HealthVault Connector application supports two ways to send clinical data to .

The HealthVault Connector can also send data directly to an authorized record, either as a standard clinical document, or as a native data type.

A Direct to Clinical connection allows healthcare information to be sent to a record without waiting for the connection to be authorized. The sent information is written to a temporary store, and is not added to the record until the connection is authorized by the record owner.

For more information, see HealthVault Connector.
