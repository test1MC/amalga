Patient Search control

 

The Patient Search control enables staff and providers to search for patients using various criteria.

The Patient Search control appears by default on the global header for the sStaff pPortal and pProvider pPortal sites. The control initially appears as a hyperlink showing the text "Search for patient", and a magnifier icon. When a user clicks the link, a lightbox-style control presents advanced search options, allowing the user to search for a patient by using one or more of the following fields:

  • First Name
  • Last Name
  • Date of Birth
  • SSN
  • MRN

The search results are displayed in a grid, with each row representing a single search result. When a patient is selected, the user is then returned to the page that they were already on, with the resulting patient switched into the current page context.

Adding the Patient Search control to a project

The Patient Search control has a large number of potential uses across a site. You can add a local Patient Search control instance to a project in by following the instructions in Toolbox controls. Once you add an instance of the control, it will automatically hook into the page messaging system, and will render according to the page CSS. Customizing the appearance of a Patient Search control instance

You can change the appearance and behavior of a Patient Search control instance by editing the control properties in the Properties Window. For a complete list of properties, see PatientSearch class. Each customized instance needs to be deployed to the site as a new control.
