Staff Portal site

 

The Staff Portal site is used by institution staff to view patient lists and respond to link requests.

The Staff Portal site provides system integrators with a starting point for creating sites which enable institution staff (e.g. billing staff, registration staff, etc.) to respond to patient and provider inquiries, view patient lists, and to manage interactions originating from the Patient Site and the Provider Site.

Staff Portal main page (default.aspx)

By default, the Staff Portal main page contains the global header and footer. You can customize this page by adding web parts, controls, and content as needed.

Patient Portal Management solution

To open the Patient Portal Management main page, click the down arrow next to "Staff" in the header, and select PatientPortalManagement. By default the header of the main Patient Portal Management solution contains the following links: Link Request List, and Manage Linked Accounts. You can customize this page by adding web parts, controls, and content as needed.

Link Request List page

The Link Request List page displays a list of users who have requested to link a record to an patient record. When a staff member selects a record, they are taken to the Patient Link Request page, where the link request can be matched with an existing account. The list displayed here includes all link attempts for the patient account in Amalga (both successful and failed). The staff is also provided with options to escalate or reject the link request, as well as a field to search by Medical Record Number.

Manage Linked Accounts page

The Manage Linked Accounts page contains a complete list of all hospital patient accounts with a history of linked records. This list can include patients whose records are not actively linked, but may have a pending link request, or some history of link requests. For each entry, the list includes at minimum:

  • Patient name
  • MRN
  • Patient gender
  • Patient DOB
  • account link
  • Date of link creation

This list can be sorted by any column, and is searchable by using the Patient Search Control. From this list you can link an account to a account, and unlink an account from a account.
