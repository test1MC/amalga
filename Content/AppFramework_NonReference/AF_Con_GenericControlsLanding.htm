Generic Controls

 

The Generic controls serve as a basis for developers to adapt existing controls and develop new ones.

The Generic controls expose functionality that is necessary for an application to interact with the Application Framework. There are two primary situations in which a developer would use the Generic controls.

  • Adapting an existing control to work with the Application Framework.
  • Creating a new control not based on an Application Framework production control.

Think of the Amalga Generic controls as a set of building blocks that are pre-configured for the . Controls that extend the generic controls can subscribe to the Context Manager to exchange context messages, hook into authentication and authorization functions, and are able to confirm whether the current user has provided consent.

All controls begin with a server control.

The server control can be an Web server control or a Web Part, but its role is the same:

  • It situates the control in the context of its host page.
  • It establishes connections between the control and a core set of server-side APIs.
  • It ensures that the client-side APIs are loaded and initialized in the DOM of the local browser.

The base classes are not visible in the toolbox.
